## 实体描述

CC Switch 是 [[Harness]] 技术栈中一个轻量级的配置工具，用于动态切换 Agent 的底层语言模型（LLM）。在自动化知识讲解视频生产流水线中，核心 Agent 通常默认使用某种高性能模型（如 Claude Code），但不同生产阶段对模型的准确性、推理速度、成本以及多模态支持有不同的需求。CC Switch 允许开发者或流水线脚本通过简单的配置参数，在保持 Agent 上下文和工具链不变的前提下，将底层模型替换为其他选项（如 MiniMax 2.7、Cursor 内置模型或 Codex）。这种设计极大地提升了流水线的灵活性和经济性：例如在内容编写阶段可以使用高精度模型以保证稿件质量，而在并行开发章节时则可切换至性价比更高的模型以降低开销。CC Switch 本身不参与逻辑执行，仅作为桥梁负责模型 API 的适配与路由，其核心价值在于“零侵入”的切换机制——不影响 Agent 的状态、记忆、工具调用以及编排逻辑。此外，CC Switch 还支持按任务类型预设模型映射表，实现自动化的上下文感知切换，进一步减少人工干预，是 [[驾驭工程]] 思想在模型层的重要体现。

## 在本视频中的角色

在《[[Harness 实践：Agent 全自动知识讲解视频生成技术简报]]》视频中，CC Switch 作为技术栈的核心配置工具出现，负责在视频生产的四个关键阶段（内容编写、网页开发、语音合成、录制）中按需切换 Agent 所调用的底层模型。它使得同一个 Agent 可以同时利用不同模型的优势：例如在口播稿改写时调用推理能力更强的模型，而在批量生成网页代码时切换至速度更快的模型。视频通过演示 CC Switch 的配置界面和自动化映射功能，展示了如何在不中断 Agent 工作流的情况下完成模型切换，从而实现了流水线的弹性与成本优化。
