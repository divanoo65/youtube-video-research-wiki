# OpenAI

## 基本定位
OpenAI 是全球领先的人工智能研究实验室，以 GPT 系列和 Codex 模型闻名。在本报告中，OpenAI 在构建百万行代码级别的 Agent 应用中，总结出三项关键 Harness Engineering 实践：**环境设计者**、**模块化指令集**和**自动化治理系统**，成为指导 Agent 工程化的重要参考。

## 核心特征/能力

1. **环境设计者角色**：OpenAI 重新定义工程师为“环境设计者”——他们不再手写 agent 逻辑，而是设计 Agent 运行的环境（包括工具、反馈链路、约束条件）。当 Agent 出错时，修复方案是优化环境结构（如增加能力或反馈链路），而非让模型“更努力”。
2. **模块化指令集（Index 化指令）**：将巨大的指令文档（如 ent.md）改为目录索引页，按需调取子文档（设计规范、安全规则等），从而节省模型稀缺的上下文窗口。类似“索引驱动”的信息暴露策略。
3. **自动化治理系统**：将资深工程师的经验转化为可执行的系统规则（如模块依赖拦截、代码质量检查），并将修复方案自动反馈至下一轮对话，实现代码质量的自动闭环。
4. **失败归因转向环境缺失**：核心经验是，Agent 失败几乎从来不是因为模型不聪明，而是因为环境缺失了某些结构性能力（如缺少反馈、工具不可用、规则不明确）。
5. **大规模代码仓库治理**：在百万行代码的应用中，OpenAI 通过上述实践实现高度自动化的代码治理，减少人工干预。

## 应用场景

- **大型代码仓库维护**：自动检测模块依赖冲突、生成修复建议并自动应用到后续开发对话中。
- **多工具协同 Agent**：Agent 需要调用多个 API 和工具时，通过环境设计明确每个工具的调用时机和结果加工方式。
- **长文档程序化处理**：使用模块化指令集，Agent 在处理复杂文档时只加载需要的部分，避免上下文过载。
