# OpenAI

**基本定位**
OpenAI 是全球领先的人工智能研究机构，推出了 GPT 系列模型、ChatGPT 等产品。在 Harness Engineering 体系中，其强调工程师角色的转变——从写代码到“环境设计师”，并提出了目录化 Skills 和自动化治理等系统设计方法。

**核心特征/能力**
1. **环境设计理念**：工程师不再只是写代码，而是设计 Agent 运行的环境。当 Agent 失败时，首要思考环境中缺了什么结构性能力，而非让模型“再努力一点”。
2. **目录化 Skills**：放弃将所有规范塞进一个巨大的 `.md` 文件，而是将其变为目录索引，Agent 根据需要动态调取子文档（如设计规范、安全规则），解决上下文窗口稀缺问题。
3. **自动化治理**：将资深工程师的经验（如模块分层规则）写成系统拦截规则，不仅报错，还会将修复建议反馈给 Agent，进入下一轮迭代。
4. **细粒度工具暴露**：按需暴露工具和技能，避免在上下文中塞入过多无关能力导致 Agent 误用。
5. **动态上下文加载**：根据任务阶段动态加载相关 SOP 和参数定义，实现“进阶式”信息暴露。

**应用场景**
- **大型代码库维护**：Agent 需要理解多层架构时，通过目录化 Skills 按需加载模块文档，避免上下文爆炸。
- **安全合规审查**：自动化治理系统可拦截违规行为（如跨层依赖），并给出修复建议，适合企业级 AI 应用。
- **复杂多步任务**：通过环境设计中的结构性工具（如数据库查询、API 调用）让 Agent 在有限信息下稳定执行。
