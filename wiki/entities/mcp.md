**Model Context Protocol**（简称 MCP）是由 [[Anthropic]] 提出并推广的一种开放协议，旨在为大型语言模型（LLM）提供标准化的上下文感知工具调用机制。其核心设计理念是解决传统 Prompt 工程中工具集成碎片化、安全性差、上下文管理复杂等问题。MCP 定义了一套轻量级的 JSON-RPC 接口，允许模型在运行时动态发现、调用外部工具、数据库、API 甚至本地文件系统，同时保证调用过程的可审计、可回滚。该协议将“模型能力”与“工具执行”解耦，使 LLM 能够专注于推理与生成，而工具的具体实现由独立的 Worker 节点或沙盒环境负责。

在实际应用中，MCP 的引入标志着智能体架构从“硬编码工具链”向“动态工具市场”的跃迁。开发者只需按照 MCP 规范发布工具描述文件（Schema），模型即可根据当前任务上下文自动选择合适的工具，无需提前在 Prompt 中列举所有可能。这种机制极大地提升了智能体的通用性和弹性，尤其在多步骤、多工具的复杂任务场景中，MCP 能够避免工具冲突、减少 Token 浪费，并支持工具链的实时热更新。此外，MCP 还内置了安全隔离机制——所有工具调用均在受控的沙盒（如 [[Docker]] 容器或 [[Redis]] 会话存储）中执行，防止模型直接访问宿主机敏感资源，从而满足企业级部署的合规要求。

## 在本视频中的角色

在本视频（或关联的分析报告）中，MCP 被定位为 **“四层解耦架构”的第四层核心组件**，与第一层 Worker 节点、第二层 Session Store、第三层静态编排器共同构成完整的智能体运行基座。视频强调，开发者应优先复刻这一四层结构，而 MCP 的作用是 **“以标准协议扩展智能体的工具集”**，使协调者（Coordinator Agent）能够通过统一的接口调用任意外部服务，包括数据库查询、代码执行、文件操作、第三方 API 等。视频中还以 [[Claude]] 的实际部署为例，展示了如何通过 MCP 将模型与 Palantir AIP、腾讯 Cube Sandbox 等平台对接，实现企业级数据管道与 AI 推理的无缝整合。此外，MCP 的引入也直接支撑了 [[宠物模式]] 与 [[牛马模式]] 两种不同粒度的任务执行策略——在需要高安全性的“宠物模式”下，MCP 会限制工具权限；而在追求效率的“牛马模式”下，MCP 则允许更宽松的工具链组合。
