# Codex Chrome

## 基本定位

Codex Chrome 是由 Open AI 前沿研究团队开发（依托 Deep Research 和 Computer Use 技术）的 AI 代理浏览器插件。它以 Chrome 扩展的形式存在，集成 GPT-5.5 模型，使 AI 能够直接操作浏览器标签页，完成真实世界的数字工作任务。

## 核心特征/能力

1. **多标签页并行处理**：可以同时打开多个独立标签页，每个标签页运行一个 AI 代理，代理之间互不干扰，并行执行任务。
2. **后台静默运行**：AI 代理在后台标签页运行时，不打断用户的当前浏览操作，用户可以继续使用浏览器做其他事。
3. **子代理协同工作**：支持创建多个子代理，它们可以在不同标签页中协同完成复杂任务（例如四个代理在画图房间根据统一提示绘制不同作品）。
4. **CDP 底层控制**：通过 Chrome DevTools Protocol 直接控制浏览器的 DOM、网络、鼠标键盘事件，实现精准的模拟操作。
5. **智能 DOM 数据获取**：当传统 HTTP 抓取被站点防火墙拦截时，自动切换为从浏览器渲染后的 DOM 提取数据。
6. **Fast/Medium 双模式**：提供快速模式和中等思考深度模式，Fast 模式处理速度极快，在同类工具中表现突出。
7. **文件上传权限控制**：支持手动开启“Allow access to file URLs”权限，以执行涉及本地文件的操作。
8. **MCP 协议支持**：支持模型上下文协议，可连接特定客户端以规避网页安全拦截。

## 应用场景

1. **自动化办公流程**：从 Gmail 中检索食物相关邮件，匹配本地 PDF 收据，自动提取日期、金额、商户信息，登录报销系统完成分类选择、附件上传和表单提交。
2. **社交媒体调研**：在 Twitter（X）上大规模采集帖子，自动更换关键词、筛选过滤无关信息，汇总结构化报告。
3. **内容创作与发布**：自动生成图文内容（使用 GPT Image 2 生成配图），编写文案，上传附件，模拟点击发布到小红书等平台。

## 关键事件/里程碑

- **2026年5月实测**：首次在公开评测中展示了全自动报销流程、多代理协同绘图、5标签页并行图像生成等突破性能力。
- **安全挑战**：在淘宝等平台遭遇站点安全策略拦截，验证了当前 AI 浏览器插件与大型电商之间的冲突。
- **手机 APP 支持**：支持手机端 APP 操作，用户可在户外下达指令，让 Codex 在后台执行耗时的多标签任务。
