## 实体描述

Docker 是一种基于操作系统级虚拟化的容器引擎，通过镜像封装应用及其依赖，实现“构建一次，随处运行”的轻量级隔离环境。在智能体（Agent）蜂群架构中，Docker 被广泛用作 Worker 节点的运行时基础：每个 Agent 独立运行在一个容器中，环境相互解耦，支持快速扩缩容。然而，传统 Docker 容器的启动（通常需要数百毫秒至数秒）和销毁仍有不可忽略的耗时，当 Agent 数量达到千级甚至万级时，这种延迟会显著拖慢任务调度与重创恢复速度。因此，业界开始关注更极致的沙盒技术（如腾讯 Cube Sandbox），追求毫秒级的冷启动能力，以支撑毫秒级动态创建 Agent 蜂群的需求。Docker 的另一个核心价值在于它结合了混合模型策略：高层决策使用大模型（如 [[Claude]]），低层执行使用小模型，不同模型可运行在不同容器中，通过编排器统一调度，从而显著提升 Token 利用效率。此外，Docker 与 [[Redis]]、[[Model Context Protocol]] 等组件共同构成了“四层解耦架构”中的 Worker 层，是当前多数 Agent 工程实践的首选方案，但也正面临被更轻量沙盒技术逐步替代的趋势。

## 在本视频中的角色

在《智能体编排与蜂群架构分析报告》中，Docker 被定位为传统基础设施的代表，展示了容器化方案的优势与局限。视频指出，虽然 Docker 能够让开发者快速搭建解耦的 Worker 节点，但其启动延迟已成为规模化 Agent 部署的“木桶短板”。报告特别对比了 Docker 与新兴的腾讯 Cube Sandbox：后者在毫秒级启动方面具备明显优势，因此 Docker 的地位正从“默认选项”转变为“过渡方案”。同时，视频强调，开发者应当首先复刻以 Docker 为基础的 Worker 层（配合 [[Redis]] Session Store 和静态编排器），再逐步演进到更轻量的沙盒环境。这一路径反映了从“工具思维”到“基础设施思维”的转变，也与 [[前瞻部署工程师]] 这一新兴角色的实践密切相关。
