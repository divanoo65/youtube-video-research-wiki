# OpenClaw

## 基本定位

OpenClaw 是早期定义自托管智能体赛道的开源项目，采用中心化 Gateway 架构，通过后台守护进程统一管控会话、路由、工具调用和状态。其设计哲学强调稳定性、可审计性与行为可控性，适合对安全性和可预测性要求较高的场景。

## 核心特征/能力

1. **中心化 Gateway 控制**：Gateway 作为后台守护进程，统一管控所有模块，确保系统行为可预测、可审计。
2. **SOUL.md 工作区绑定**：身份定义文件 SOUL.md 绑定特定工作区，切换工作区即切换身份，适合多角色管理。
3. **模块化插件系统**：功能扩展通过人类编写的模块化插件实现，结构清晰但缺乏自动进化能力。
4. **Markdown 文件记忆体系**：记忆以 Markdown 文件为核心，便于人工编辑和审查，但动态自我优化能力较弱。
5. **高稳定性**：中心化控制架构在复杂任务中表现出极高的稳定性，适合生产环境。

## 应用场景

1. **企业级自动化**：需要严格审计和稳定性的业务流程自动化。
2. **多角色管理**：通过切换工作区实现不同身份（如个人、团队）的隔离管理。
3. **安全敏感任务**：中心化控制便于权限管理和行为监控。
