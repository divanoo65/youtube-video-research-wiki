# Model Context Protocol (MCP)

## 基本定位

Model Context Protocol（MCP）是一种通信协议，旨在让 AI 代理能够脱离浏览器环境，直接与桌面或移动端的客户端应用程序（如电商 APP、办公软件）建立连接。通过 MCP，代理可以绕过网站对浏览器插件的安全策略限制，获得更稳定、权限更高的操作环境。

## 核心特征/能力

1. **跨应用连接**：允许代理通过标准化接口与本地或远程客户端应用通信。
2. **安全策略规避**：当网页端禁用浏览器插件访问时（如淘宝），可通过 MCP 连接到该平台的专用客户端继续任务。
3. **扩展能力**：支持开发者自定义 MCP 适配器，连接任意支持该协议的应用。

## 应用场景

- **电商比价/下单**：在电商网站（如淘宝）因安全策略禁止 Codex 接入时，通过 MCP 连接至其移动客户端或桌面客户端完成商品查询与表格生成。
- **办公软件集成**：未来可连接本地 Office、微信、飞书等客户端，实现跨应用自动化。

## 关系网络

- **被 [[Codex-Chrome插件]] 支持**：插件提供 MCP 配置选项，作为浏览器自动化的补充。
- **解决站点安全拦截问题**：MCP 是绕过网站安全策略拦截的技术手段，作为浏览器自动化的补充方案。
- **类似于 [[Chrome调试协议]] 但面向外部客户端**：CDP 控制浏览器内部，MCP 控制外部应用。

## 关键事件/里程碑

- **2026年**：Codex 插件实测中提及 MCP 可用于绕过淘宝安全策略，但尚未在实验中实际验证。
- **OpenAI 推动**：MCP 作为开放协议，有潜力成为 AI 代理与各类客户端交互的标准。
