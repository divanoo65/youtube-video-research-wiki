连接地址（connection address）是计算机网络中的一个基本概念，它作为网络上特定位置的唯一标识符，用于定位和访问服务或资源。它可以表现为多种形式，包括互联网协议（IP）地址（例如，192.168.1.1，2001:0db8::1）、域名（例如，example.com）或统一资源定位符（URL）（例如，https://api.example.com/v1）。其主要目的是使设备或应用程序能够跨局域网（LAN）或互联网定位并建立与其他设备、服务器或服务的通信。在软件配置的语境中，连接地址通常用于指定应用程序查找外部依赖项的位置，例如数据库、API端点或云服务。正确配置连接地址对于确保分布式系统的不同组件能够有效通信以及服务按预期可访问至关重要。它决定了用于数据交换的网络路径和协议，在系统集成和功能中发挥着关键作用。

### 在本视频中的角色
在[[赫美斯 Agent (Hermes Agent) 高级应用与部署简报]]中，"connection address" 作为一个重要的配置参数被提及，尤其是在配置赫美斯 Agent 与外部服务（如Ollama云端模型或本地部署模型）进行通信时。它通常用于指定 [[API service]] 的入口点，例如一个 [[Base URL]]，使得赫美斯 Agent 能够知道如何连接到模型服务并发送请求。正确设置此地址是确保 Agent 能够成功调用模型、执行任务的关键。例如，用户可能需要在 `[[config.toml]]` 文件中配置此地址，以指向其Ollama服务或自定义的API端点，从而实现零成本上手或高级部署策略。
