# Codex Hooks 自动化音频通知系统简报

## 执行摘要

在执行长时间的 Codex 任务时（例如耗时超过一小时的任务），用户往往无法持续盯着屏幕。如果任务中途需要用户授权而用户不在现场，任务将会停滞，导致严重的时间浪费。通过配置 Codex Hooks，用户可以在任务的关键节点（如开始、需要授权、结束）触发自动化脚本播放特定音频。这不仅提升了人机交互的效率，也为更复杂的自动化工作流（如日志检查、自动提交等）提供了入口。

---

## 核心要点

### 1. Codex Hooks 的生命周期节点
Codex 任务运行过程中存在六个关键的生命周期节点，通过配置这些节点，可以自动运行特定的脚本。本方案主要使用了其中的三个阶段：

- **`session_start`**：开启新窗口（本方案未使用）。
- **`user_prompt_submit`**（对应“1号位”）：任务开始时播放音频。
- **`permission_request`**（对应“2号位”）：需要删除文件等操作时提醒用户授权。
- **`tool_use`**：工具类使用（本方案未使用）。
- **`stop`**（对应“3号位”）：整个任务完成后播放结束音频。

### 2. 技术实现架构
实现该自动化方案需要在项目结构中建立特定的文件体系：

- **目录结构**：在项目根目录下新建 `.codex` 文件夹，其下包含 `audio`（存放 MP3 音频）和 `scripts`（存放执行脚本）两个子文件夹。
- **配置文件**：
  - `hooks.json`：定义生命周期名称、类型（如 `command`）以及具体运行的命令和超时时间。
  - `confirm`：具体的逻辑配置文件（用于确认权限）。
- **脚本支持**：根据操作系统不同（macOS、Windows、Linux），需编写对应的脚本文件。例如，在 macOS 下，需为脚本赋予可执行权限。

### 3. 安全性确认机制
为了确保系统安全，Codex 不会自动运行用户配置的 Hooks。

- **人工审核**：配置完成后，Codex App 会提示 Hooks 已配置但需人工确认。
- **信任授权**：用户必须点击 “Review Hooks” 并对具体的项目级 Hooks 点击 “Trust”（信任）按钮，自动化脚本方可正常工作。

### 4. 自动化能力的扩展
音频通知只是 Codex Hooks 自动化的一个初步应用（入门入口）。基于脚本的可扩展性，该功能可延伸至：

- **推送通知**：将任务状态发送至移动端。
- **日志检查**：自动扫描任务运行日志。
- **Git 自动化**：实现代码的 `auto commit`（自动提交）。
- **全局应用**：将配置文件从“项目级”（当前目录）迁移至“用户级”（用户根目录的 `.codex` 文件夹），即可使 Hooks 对该用户下的所有项目生效。

### 5. 关键引言及上下文

> “比如它运行了一个小时23分……如果你恰巧没看见这个授权，过了两个小时回来一看，好家伙它只干了10分钟的活，剩下的时间都在那等。最简单的解决方法就是在需要授权的时候，让它自己播放一个声音。”

**上下文**：这段话道出了开发者的核心痛点：在长时任务中，由于缺乏反馈机制，微小的授权请求往往会导致数小时的进度停滞。

> “我们做的视频是音频播放，是为了帮助大家快速的理解和入门。但实质上这是自动化的一个入口，因为它是用脚本来的，脚本你就可以实现无限多你想要的内容。”

**上下文**：作者强调音频提醒只是一个教学案例，其背后的 Hook + 脚本机制才是实现无限自动化可能的关键。

## 执行建议与行动指南

1. **环境准备**：
   - 访问指定技术文档页面下载所需的五个脚本文件和三个音频文件（`start`、`permission`、`stop`）。
   - 在目标项目的根目录创建 `.codex` 目录及其子文件夹。
2. **配置映射**：
   - 将 `hooks.json` 内容导入，确保 `command` 路径指向正确的脚本位置。
   - 根据使用的操作系统（Windows/Mac/Linux）选择对应的脚本内容。
3. **权限授予**：
   - 对于 macOS/Linux 用户，确保执行脚本具有可运行权限。
   - **关键步骤**：进入 Codex App，完成 "Review Hooks" 并勾选 "Trust"，否则脚本将被拦截。
4. **功能验证**：
   - 尝试运行一个涉及文件删除或修改的任务，测试在提交请求、弹出授权窗口和任务结束时，系统是否能按预期播放对应的音频提醒。
5. **进阶应用**：
   - 验证成功后，可考虑将 `.codex` 配置提升至用户根目录，以实现跨项目的全局自动化通知。

通过以上步骤，您可以快速建立基于 [[Codex Hooks]] 的音频通知系统，并以此为起点探索更广泛的 [[自动化工作流]] 方案。
