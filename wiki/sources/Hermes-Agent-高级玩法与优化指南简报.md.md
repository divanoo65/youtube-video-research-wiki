## 执行摘要

本报告旨在总结 [[Hermes Agent 高级玩法与优化指南简报]]（中文官方名：[[赫美斯]]）的高级应用技巧与配置优化方案。Hermes Agent 作为一个高性能 AI Agent 框架，在稳定性上显著优于 OpenCloud 等同类项目，尤其在避免更新引入 Bug 方面表现出色。本简报核心内容涵盖了通过 Ollama 使用云端免费模型、集成 Open WebUI 提升交互体验，以及通过“主副模型”配置实现 Token 成本大幅压缩的三大隐藏技能。

## 核心要点

本简报深入探讨了 [[Hermes Agent 高级玩法与优化指南简报]]（官方中文名：[[赫美斯]]）的三大核心高级玩法与优化策略，旨在帮助用户最大化其性能并有效控制运营成本。

### 1. Ollama 集成与云端免费模型调用

Hermes Agent 已深度内置于 Ollama 平台，为用户提供了便捷的云端模型调用方案，尤其适合希望避免占用本地资源或寻求免费额度的用户。通过在 Ollama 官网下载并安装对应版本，用户只需简单的终端命令即可激活模型选项，例如利用 MXM 2.7 及其云端版本，无需本地计算资源即可运行 Agent。这种“傻瓜化”的配置流程，通过简单的账号登录和设备连接，便能激活 Hermes Agent 的 Gateway，极大地简化了部署难度。

### 2. Open WebUI 交互美化与功能增强

尽管许多用户习惯于在微信等聊天工具中使用 Agent，但这些工具在 Markdown 解析、会话记录管理和复杂交互方面存在明显局限。Hermes Agent 原生支持 Open WebUI，提供了一种类似 ChatGPT 的卓越交互体验。Open WebUI 的核心功能提升包括：
*   **会话管理：** 侧边栏自动保存历史记录，支持搜索过往对话，便于回顾和管理。
*   **代码执行：** 支持 Python 代码块的流式输出及直接运行，为开发者提供了强大的调试和测试环境。
*   **移动端访问：** 通过局域网 IP 和端口（8080），用户可以在手机浏览器上直接与电脑端的 Agent 交互，实现跨设备操作。
*   **富媒体支持：** 相比聊天软件，Open WebUI 提供更好的文件上传、截图引用、网页及笔记库引用功能，极大地丰富了交互形式。
配置上，用户需在 Hermes Agent 配置文件中启用 API 服务（`enable_api`）并设置自定义密码，随后在 Open WebUI 的管理员设置中连接到本地端口（8642/v1）。正如报告所强调：“我们通过 Open WebUI 与 MIS Agent 进行交互，就能够达到更好的使用体验……会比我们直接在聊天软件中交互要方便非常非常多。”

### 3. “主副模型”配置：Token 成本优化策略

针对 Agent 消耗 Token 过快的问题，Hermes Agent 引入了创新的主副模型协作机制，旨在大幅节省 Token 成本。其核心思想是职能分工：
*   **主模型：** 负责处理复杂的逻辑任务和最终决策，通常选用性能最强但价格较高的模型。
*   **副模型（辅助模型）：** 负责处理批准（Approval）、压缩（Compression）、记忆重刷、MCP 调用、Session 搜索、技能相关、视觉解析及网页工具等简单、重复性任务。
视频作者强烈建议将所有辅助任务指派给性价比极高的模型，例如 Google 的 **Gemini 1.5 Flash**。实现这一策略，用户需通过编辑配置文件，为不同的子任务（如 `tasks.compression`, `tasks.memory` 等）分别指定 API Key、Base URL 及模型 ID。这种分流策略能够显著降低运行成本，正如报告所言：“由主模型来完成复杂任务，由附模型来完成比较简单的任务……这样能大幅节省 token。”

### 实践指南与功能测试

部署与连接方面，用户可快速启动 Ollama 并运行相关命令激活 Hermes 环境。界面配置则涉及安装 Open WebUI，修改 Agent 配置文件启用 API 服务并设置密码，然后在 Open WebUI 中添加连接指向 `http://localhost:8642/v1`。在性能与成本优化上，除了上述多模型分流，若需在室外访问，可结合内网穿透工具将本地 8080 端口映射至公网。成功部署后，可通过提问“你可以调用哪些技能”来验证 Agent 是否正确加载了所有相关的 Scale/Tools（通常超过 100 个）。
