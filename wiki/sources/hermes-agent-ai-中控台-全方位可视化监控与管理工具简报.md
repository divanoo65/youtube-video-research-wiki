## 执行摘要

本报告详细介绍了针对 GitHub 热门项目 [[Hermes Agent]]（拥有 8 万颗星）开发的一款可视化 Web 中控台。由于 Hermes Agent 在后台运行时，用户往往难以直观地观察其处理逻辑、资金消耗及记忆准确性，该中控台通过 13 个功能标签页（Tabs），将 [[AI Agent]] 的内部运作完全透明化。该工具不仅提供了详尽的数据看板、成本分析和记忆编辑功能，还集成了任务调度管理、多项目扫描及浏览器内直接对话等高级功能，极大地提升了开发者管理 AI Agent 的效率与深度。

## 核心要点

### 1. 全透明的数据看板与性能监控 (Dashboard & Growth)

中控台将 AI 的运行状态比作“航天船控制台”，通过本地数据库 `state.db`（AI 的“大脑”）实时反映其知识储备。用户可以监控对话次数、消息总量、工具调用频率以及技能掌握情况（包含 AI 自学技能）。平台分布功能清晰记录 CLI 和 Telegram 等不同平台的交互次数。“Growth Delta”功能通过每日快照对比，以绿色增长箭头直观展示 AI 在会话和消息量上的成长趋势。此外，24 小时活动折线图帮助用户识别 AI 使用的高峰时段。

### 2. 记忆与纠错机制 (Memory & Correction)

该中控台解决了 AI 记忆不可见、不可控的问题，将“工作记忆”与“用户画像”分离管理。用户无需翻阅底层的 Markdown 文件，即可直接在网页端修改或删除错误记忆，且删除操作需两次确认以确保数据安全。“Correction”标签页自动记录 16 条纠正记录，按严重程度（致命、主要、次要）分类，并追溯至具体的对话会话，相当于 AI 的“错题本”。

### 3. 成本控制与资源管理 (Cost & Health)

针对 API 消耗提供精细化审计，防止超额支出。例如，详细列出不同模型（如 Claude Opus）的费用占比，在 30 个会话中处理 130 万个 Token 的总费用可精准核算至 20.06 美元。“Health”标签页一眼扫描 API 密钥配置状态（Missing 或 Active）以及 Telegram Gateway 等服务的运行状态，确保系统健康运行。

### 4. 自动化任务与项目扫描 (Cron & Project Step)

中控台将 AI 从单一的聊天机器人提升为系统级助理。用户可以管理如“每日开发摘要”、“GitHub 状态检查”、“依赖安全性扫描”等后台自动运行任务。它还能自动侦测本地 9 个项目文件夹，显示分支信息、编程语言及 Git 状态（如提示“Dirty”代表有修改未提交），实现对开发环境的全面监控。

### 5. 交互式控制与快捷操作 (Wchat & UI)

“Wchat”功能允许用户直接在网页端调用 `hermes chat` CLI 指令，支持工具调用（如查询日期、纪念日等）。中控台还支持主题切换（T 键）和快速跳转（Ctrl + K 搜索标签页），极大地提升了操作效率和用户体验。

### 7. 核心语录与背景

> **“是不是有种打开太空船控制台的感觉？”**
*   **背景：** 描述初次启动中控台时，顶排 13 个标签页带来的视觉冲击力与掌控感。

> **“这块区域等于你 AI 的成绩单。”**
*   **背景：** 指代 Dashboard 中汇总的对话数、消息数、工具调用数等核心表现指标。

> **“这就是你 AI 的错题本。”**
*   **背景：** 形容 Correction 标签页的功能，它能从记忆和对话中挖出 AI 被纠正的记录，帮助用户优化 AI 表现。

### 8. 操作指南与实践洞察

要使用此中控台，前提是计算机上已安装并运行过 Hermes Agent 本体。部署流程包括克隆项目、运行 `install` 指令进行自动化安装（创建虚拟环境、安装套件、构建前端），然后点击生成的链接即可进入控制界面。

在行为模式分析方面，系统会自动将所有会话分成六大类，帮助用户了解自己最常要求 AI 执行的任务。当系统检测到某个 Prompt 重复出现 3 次以上时，会标记“闪电”图标，提示用户应考虑将其转化为自动化脚本（Skill），实现高效的任务自动化。

高级交互技巧包括支持在数百个会话中进行全文本搜索，明确标识关键词所在位置；以及在 Cron 标签页中，用户可以随时对后台任务执行恢复（Resume）或删除操作，实现对自动化任务的精细化管理。
