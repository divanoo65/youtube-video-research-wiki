## 执行摘要

Hermes Agent 作为一款高效 AI 代理工具，其纯后台运行模式存在显著的“黑盒”弊端——用户无法实时感知记忆状态、逻辑错误、成本消耗及资源占用。为了突破这一瓶颈，开发团队推出了一套基于 Web 的 13 标签页中控台，将 AI 的“大脑”容量（state.db 大小）、对话统计、工具调用等物理指标全面可视化，并创新地融入了直接编辑记忆、精细化成本拆分、任务聚类热力图、高频 Prompt 自动化建议等功能。借助该中控台，用户得以从被动观察者转变为主动管理者，不仅可监控 128 万 token 处理量、80 个技能点的运行实况，还能直接干预 AI 记忆、优化模型选择、自动化重复工作流，实现了对 AI 代理的深度治理与透明化运营。

## 核心要点

- **Dashboard 提供 AI 综合“成绩单”**：以物理指标（state.db 大小）衡量记忆深度，统计对话数、消息量、工具调用次数（187 次）及多平台来源（CLI 28 次 / Telegram 6 次），并通过“Growth Delta”每日快照直观展示增长趋势。
- **双维度动态记忆管理**：将记忆分为“工作记忆”与“用户画像”，用户可直接在中控台编辑、新增或删除记忆条目（需二次确认），无需翻阅底层 MD 文件，实现快速纠偏。
- **错题本功能与错误分级**：Correction Tab 记录所有纠正记录，按严重、主要、次要分级归类，并附上具体的逻辑缺陷描述（如 fetch 缺少错误处理），同时链接至对应会话便于溯源。
- **精细化成本监控**：实时计算当前/累计 session 费用，按模型拆分开销（例如 Claude Opus 消耗 19.7 美元），帮助用户评估是否切换更具性价比的模型，实现财务透明。
- **行为模式分析与自动化推荐**：通过任务聚类（6 个集群）和 24 小时活动热力图识别用户习惯；当同一 Prompt 被重复输入 3 次以上时，系统自动标记闪电图标，提示可转化为永久 skill 或自动脚本。
- **Cron 任务排程与项目扫描**：支持设置 GitHub 状态检查、每日摘要、内存备份等自动化后台任务；Projects Tab 自动侦测本地 Git 仓库，显示脏/活跃状态，方便项目管理。
- **多代理进程监控与 Web Chat 交互**：Agents Tab 监控 Hermes、Codex、Cursor 等所有活跃代理；Web Chat 集成 Terminal 工具，支持全键盘快捷键（T 键换肤、Ctrl+K 搜索），降低操作门槛。
- **部署简便，先决条件明确**：需先运行 Hermes Agent 本体以生成本地数据文件夹，通过 Git 克隆、自动安装脚本启动，即可通过本地链接访问中控台。
- **推荐优化路径**：定期检查 Cost Tab 控制高模型开销、利用 Correction Tab 修正严重错误、将 Patterns 中的闪电提示转化为 Cron 自动任务，并熟练使用键盘快捷键提升效率。
