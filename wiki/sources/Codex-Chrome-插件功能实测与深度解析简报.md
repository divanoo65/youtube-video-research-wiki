# Codex Chrome 插件功能实测与深度解析简报

**视频URL**: https://www.youtube.com/watch?v=9MCjT-eUrTs
**脑图文件**: [[raw/notebooklm-mindmaps/Codex-Chrome-插件功能实测与深度解析简报.json]]

## 执行摘要

Codex Chrome 插件是 AI 代理从文本交互迈向“计算机使用”(Computer Use) 的重要里程碑。它依托 GPT-5.5 模型，实现了多标签并行处理与后台静默运行，能够在已登录浏览器环境中完成跨平台数据抓取、结构化总结、自动化报销填报以及多代理协作绘图等复杂任务。实测显示其处理速度与准确率显著优于同类工具，但在面对淘宝等站点安全策略时仍存在限制。

## 核心要点

1. **多标签并行与后台协同**：允许 AI 在独立标签页中运行，不干扰用户当前操作；支持多个子代理同时在不同标签页执行任务（如四个代理协同绘画）。
2. **端到端自动化办公**：从 Gmail 检索邮件、匹配本地 PDF 收据、提取金额日期，直接登录报销系统完成表单填报与附件上传。
3. **深度内容提取**：通过浏览器渲染后的 DOM 获取数据，即使服务器抓取被拦截也能绕过；自动打开正文页面读取上下文以生成精准摘要。
4. **高效调研**：在 Twitter 搜索中自动更换关键词，采集89条帖子并筛选汇总50条相关样本；在开发者社区浏览20+帖子后整理出结构化 Excel。
5. **多代理协作**：四个代理同时加入在线画图房间，根据同一提示词绘制出不同作品，体现任务隔离与并行处理能力。
6. **速度优势**：Fast 模式下处理速度远超以往 AI Agent，实测社区调研仅3分41秒，电商比价1分10秒。
7. **GPT-5.5 模型集成**：提供 Medium（中等思考）与 Fast（快速）两种模式，支持深度推理与快速执行。
8. **CDP 底层控制**：通过 Chrome DevTools Protocol 连接，实现对浏览器标签页、DOM、网络请求的精细操控。
9. **MCP 支持**：可连接特定客户端（如电商客户端）以规避网页端安全拦截。
10. **安全策略限制**：淘宝等平台明确禁止插件访问，触发站点安全策略拦截；插件需手动开启 file:// 权限才能上传文件。

## 关键引言

> “Chrome 的插件呢，就允许在不干扰你的情况下运行自己的标签页，并且你可以让多个 agent 同时运行独立的标签。”
> **背景**：强调 Codex 解决以往 AI 工具独占屏幕的根本问题。

> “我之前用过的所有的 AI Agent 的处理速度是没有目前 Codex 的处理速度特别快的，尤其是当你选择它的 fast 模式的时候，就会感觉速度更快更爽。”
> **背景**：实测者在大规模检索任务中对速度的高度评价。

> “它这里说直接服务器抓取站点防护页拦截，但是页面本身是可以访问的，所以 Codex 会从浏览器渲染后的 DOM 获取数据。”
> **背景**：展示 Codex 灵活的反爬虫绕过能力。

> “某宝无法集成 Chrome 插件，对某宝触发了站点安全策略拦截，明确禁止 Codex 去使用。”
> **背景**：揭示当前 AI 浏览器插件的主要挑战。
