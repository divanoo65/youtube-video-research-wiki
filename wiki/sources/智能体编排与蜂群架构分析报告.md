## 执行摘要

当前 Agent 领域正经历从单一工具向企业级基础设施的范式转移。核心痛点在于模型进化速度远超手动修复代码的速度，以及模型记忆与执行环境耦合导致的系统脆弱性。Anthropic 提出的 Managed Agents 架构通过“脑手解耦”提供了标准答案：将大模型的规划决策（大脑）与执行代码的沙盒环境（双手）彻底分离。通过将 Agent 视为无状态的、可随时销毁与重建的“牛马”而非需要精心维护的“宠物”，该架构实现了高度的可扩展性、任务并行化以及优化的 Token 效率。

## 核心要点

### 1. 从“宠物”到“牛马”：Agent 理念的根本变革
传统的 Agent 开发模式倾向于将模型记忆与本地环境深度绑定，形成所谓的 [[宠物模式]]。这种模式具有极高的脆弱性：一旦配置出错、记忆堆积或环境污染，整个系统就会崩溃且难以修复。为弥补模型缺陷而编写的数千行补丁代码，在模型升级后往往会变成毫无价值的垃圾。解决之道是转向 [[牛马模式]]——Agent 应该是无状态的、独立的沙盒。如果某个沙盒出错，直接销毁并重新拉起一个干净的环境。这一转变彻底解决了维护困境，使系统能够随着模型进化而自动受益。

### 2. Managed Agents 的四层解耦架构
通过对相关 SDK 的逆向分析，该架构可以抽象为层层递进的四层解耦模型，这是 [[四层解耦架构]] 的核心设计：

- **第一层：定义层 (Agent)**：静态模板，定义“我是谁”，包括模型选择、系统提示词（Prompt）、工具集和技能（Skills）。
- **第二层：执行层 (Session)**：动态实体，定义“我在干什么”，包含对话线程、记忆挂载文件。同一个 Agent 模板可以派生出多个相互独立的 Session。
- **第三层：编排层 (Coordinator)**：充当不干活的“指挥官”，负责拆解任务、派发活计，管理多达 20 个具有独立工具和 MCP 服务器的子 Agent。这一层体现了 [[脑手解耦]] 的关键思想——大脑（协调者）与双手（Worker）彻底分离。
- **第四层：持久层 (Session Store)**：将上下文从本地搬到云端（如 [[Redis]]），支持记忆的 Fork（分叉）、回滚和克隆，使 Agent 彻底实现[[无状态化]]。

### 3. 编排者（Coordinator）的任务调度逻辑
在长程任务中，单一 Agent 常因上下文不足需要不断压缩记忆，导致信息丢失。Managed Agents 引入了专门的 [[协调者]] 角色。人类不再充当手动协调者，而是由一个专门的 Agent 担任协调者。协调者能识别 Agent 的“最聪明状态”，并在需要时执行[[状态回滚与复制]]。通过预定义的[[任务图]]（如：编码 -> Review -> 测试），实现多个 Worker 节点的并发协作，极大提升复杂任务的执行效率。

### 4. 关键技术与实践洞察
底层 SDK 的 Beta 目录揭示了 Managed Agents 的核心构建块：Agents & Environments 实现物理隔离；Sessions & Skills 分离动态过程与静态技能；Memory Stores & Vaults 提供安全可伸缩的记忆存储；每个子 Agent 通过独立的 MCP 服务器接入工具集。在实践层面，[[混合模型策略]]（高性能模型+轻量级模型）能显著提升令牌效率；采用容器化与毫秒级沙盒技术（如 [[腾讯Cube Sandbox]]）可实现大规模蜂群的动态创建；开发者应优先复刻四层解耦结构，从工具思维转向[[基础设施思维]]，关注[[前瞻部署工程师]]等新职业的兴起。海外领先厂商如 [[Palantir AIP]] 与 [[OpenAI Frontier]] 已将 Agent 提升至企业级基础设施维度，印证了[[蜂群架构]]的未来方向。
