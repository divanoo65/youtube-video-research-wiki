## 执行摘要

本报告基于 Harness 实践，详细解析了利用 Agent 全自动制作知识讲解视频的技术实现方案。当前 AI 视频生成领域在精细化、可控性和成本方面仍存在挑战，该方案避开了直接使用视频生成模型或 NotebookLM 等工具，转而采用 **Web Coding（网页编码）** 的方式生成视频内容。通过在 Harness 框架下设计的“Web Video Presentation”技能，开发者可以指挥 AI Agent（如 Claude Code）完成口播稿改写、开发大纲制定、网页动态效果编写及语音合成。整个流程的核心价值在于通过工程化手段（流程控制、状态管理、自检修复）驾驭模型能力，确保不同主题、不同背景下的视频产出具有高度的稳定性和一致性。关键技术栈包括 Claude Code、MiniMax 模型、CC Switch、MiniMax CLI 和 TMUX 环境管理工具。

## 核心要点

本方案是一套完整的 **Harness（驾驭）工程系统**，而非简单的 Prompt 工程。一个成熟的 Agent 技能（Skill）被拆解为六个核心部分：上下文管理、工具系统、执行与编排、状态与记忆、评估与观测、约束与恢复。其中，上下文管理将文档拆分为多个子文档（口播规范、开发指引等），仅在对应阶段读取，避免注意力稀释；执行与编排将流程切分为四个阶段并设置人工检查点；状态与记忆通过 `outline.md` 保持长链路开发的连续性，记录用户修改偏好；评估与观测引入独立的 Reviewer Agent 进行硬性自检；约束与恢复遵循“最小切片修复”原则，出错时仅修改受影响的代码层级，不重作整个章节。

视频生产过程包含四个关键阶段：**第一阶段（内容编写与规划）** 完成口播稿改写和开发大纲制定，需经过人工审核确认风格与节奏；**第二阶段（网页开发与验证）** 先开发基准章节，经人工验证视觉气质与动效后，再利用 Subagents 并行开发后续章节，并通过物理隔离（独立文件夹和 CSS 类名）保证风格一致；**第三阶段（语音合成）** 从口播稿提取音频清单，调用 MiniMax CLI 批量生成音频文件；**第四阶段（录制与产出）** 支持手动模式、音频模式和全自动模式（根据音频节奏自动翻页动效），实现“一键录屏”。

本方案选择 Web Coding 路线的根本原因在于 **可控性**：字体配色、每一步停留时间、帧内精确数字等细节，通过修改几行代码即可直接调整，远比视频模型“抽卡”稳定。如报告核心语录所述：“做 Harness 不是非得从零去做一个 Agent，能用一个 Skill 约束 Agent 把一个垂直的工作做好，也是在做 Harness。” 其核心价值在于把模型能力、工具能力和人的判断组织成一条稳定可控的生产流程。

行动建议方面，环境搭建需安装 `claude-code`、配置 `cc-switch`、注册 MiniMax 并获取 API Key、下载 “Web Video Presentation” 技能包。执行策略强调“从小样开始”，严格执行“第一章验收”制度，对10章节以上长视频建议启动 `agent-teams` 配合 `tmux` 同时运行3个左右 Agent。模式选择上，追求极致效率时使用全自动模式 (`out2=1`)，追求表达感染力时建议使用音频模式 (`audio=1`) 配合真人配音或手动节奏控制。

本报告涉及的核心概念包括 [[最小切片修复]]（修复时仅修改受影响代码层级，保持已有正确部分不变）、[[Agent团队]]（利用 Subagents 并行开发章节，通过 TMUX 可视化协作）以及 [[Agent技能]]（将复杂任务封装为可复用的标准化技能流程）。
