## 执行摘要

本简报基于对 [[Hermes Agent]] 辅助中控台（Dashboard）的功能及应用分析。Hermes Agent 作为在 [[GitHub]] 上拥有 8 万星的热门 [[AI]] 项目，其运行逻辑通常在后台执行，导致用户难以直观监控其运行成本、记忆准确性及后台动态。本文档详细介绍了新推出的 13 标签页（tab）Web 中控台，该工具为用户提供了一个如“太空船控制台”般的界面，实现了对 AI 大脑状态、对话记录、成本消耗及自动化技能的全方位可视化与直接管理。

## 核心要点

### 1. 全透明的系统观察与状态监控
中控台通过多个维度打破了 [[AI]] 运行的“黑盒”状态：
*   **概览仪表盘（Dashboard）：** 提供 AI 的“成绩单”，包括运行天数、使用的模型后端、对话次数（如 34 个对话）、消息总数（505 则）及工具调用频次（187 次）。
*   **大脑容量监测：** 通过监控本地数据库文件 `state.db` 的大小，直观反映 AI 学习到的知识量。
*   **健康检查（Health）：** 实时显示 9 种 API 密钥的配置状态及 [[Telegram]] 网关等服务的运行状态，确保系统链路畅通。
*   **多智能体追踪（Active Agents）：** 能够识别并显示电脑上正在运行的所有 AI 智能体（如 Hermes Cloud, Codex, Cursor 等），并记录它们的活跃时间轴。

### 2. 深度记忆管理与纠错机制
针对 AI 记忆可能出错的问题，中控台提供了比手动修改 Markdown 文件更高效的方案：
*   **记忆编辑（Memory）：** 将记忆分为“智能体工作记忆”与“用户画像（User Profile）”。支持直接在网页端进行增加、编辑或删除（需二次确认以防误删）操作。
*   **错题本功能（Correction）：** 自动从记忆和对话中提取错误并分类（Critical, Major, Minor）。例如，纠正 AI 在未处理错误捕获的情况下建议使用 `fetch` 的行为，并链接回具体的对话会话。

### 3. 成本控制与消耗分析
中控台内置了详尽的财务统计功能（Cost Tab）：
*   **精细化计费：** 统计不同模型（如 Claude Opus）的 Token 消耗及对应美元费用。
*   **会话级追踪：** 列出单个会话的成本，帮助用户识别哪些高频对话导致了高额支出。

### 4. 行为模式分析与自动化提升
通过对使用习惯的挖掘，工具协助用户将重复劳动转化为自动化技能：
*   **任务聚类（Pattern）：** 自动将对话分为多个类别（如 6 类），并生成 24 小时活动热力图，识别使用高峰。
*   **提示词优化：** 自动识别重复出现 3 次以上的 Prompt，并标注“闪电”图标，提示用户可将其转化为自动化脚本（Sale 自动化）。
*   **排程管理（Cron）：** 集中管理 AI 在后台自动运行的任务，如每日开发摘要、GitHub 状态检查、依赖安全性扫描等。

## 核心功能模块概览

| 标签页 (Tab) | 主要功能描述 | 关键数据/特点 |
| :--- | :--- | :--- |
| **Dashboard** | 系统概览与 AI 成长曲线 | 显示 Token 处理量（如 128 万）、增长增量（Growth Delta）快照 |
| **Memory** | 工作记忆与用户信息管理 | 分类清晰，支持网页端直接增删改 |
| **Sessions** | 完整对话记录查看 | 支持全文检索（标题及内文），区分 Telegram 与 CLI 来源 |
| **Cost** | 成本与 Token 消耗统计 | 按模型拆分费用，详细到单次会话金额 |
| **Pattern** | 使用习惯与重复 Prompt 分析 | 任务聚类、24 小时活跃图、自动化建议 |
| **Skills** | 技能库管理 | 展示 80 个技能及其分类，支持查看自定义技能 |
| **Cron** | 后台排程任务管理 | 控制任务启动、恢复与删除 |
| **Project** | 项目文件夹扫描 | 自动侦测 Git 状态（Active, Dirty, Stale）与代码语言 |
| **Wchat** | 浏览器内置对话接口 | 调用真实终端工具返回结果，支持快捷键切换主题（T）与跳转（Ctrl+K） |

*   **上下文：** 强调中控台在用户体验上的改进，将繁琐的底层文件操作转化为直观的 UI 交互。

## 行动指南与操作建议

### 1. 系统部署步骤
*   **前提条件：** 电脑必须先安装并运行过 Hermes Agent 本体。
*   **安装流程：**
    1.  克隆中控台项目仓库。
    2.  在终端执行 `install` 指令。
    3.  系统将自动创建虚拟环境、安装依赖并构建前端。
    4.  安装成功后点击生成的链接即可访问。

### 2. 效率优化建议
*   **利用快捷键：** 使用 `T` 键快速切换 4 种主题风格；使用 `Ctrl + K` 呼出搜索框在不同标签页间快速跳转。
*   **定期清理记忆：** 通过 Correction 标签页定期审查 AI 的“错题本”，删除过期或错误的记忆条目，以维持 AI 回答的准确性。
*   **识别自动化机会：** 关注 Pattern 页面中带闪电图标的高频 Prompt，通过 Cron 标签页将其配置为后台自动运行的任务。

### 3. 成本管理策略
*   **监控高溢价模型：** 鉴于 Claude Opus 等模型成本较高（如 130 万 Token 消耗约 20 美元），应定期在 Cost 标签页检查各模型消耗占比，必要时切换到更经济的模型后端。
