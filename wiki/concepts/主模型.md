## 概念定义

[[主模型]]是[[赫美斯 (Hermes) Agent 高级应用与部署指南：云端集成、界面优化与 Token 成本控制]]中提出的一种模型配置策略的核心组成部分，旨在优化[[AI Agent]]运行时的[[Token 消耗]]。在[[主副模型分离配置]]机制中，主模型特指那些性能强大但计算成本相对较高的[[AI模型]]。它的主要职责是处理Agent运行过程中遇到的复杂任务和核心逻辑，这些任务通常需要更强的推理能力、更广的知识储备或更精细的理解。与负责简单、辅助性任务的[[副模型]]形成对比，主模型是Agent智能决策和复杂问题解决的关键。这种分离配置的目的是通过将大部分简单任务分流给廉价的副模型，从而大幅降低整体运行成本，同时确保复杂任务仍能由高性能模型高质量完成。

## 技术细节

在[[赫美斯 (Hermes) Agent 高级应用与部署指南：云端集成、界面优化与 Token 成本控制]]的“主副模型配置与 Token 节约策略”中，主模型被定位为处理“复杂任务”和“核心逻辑”的“昂贵的高性能”模型。这意味着主模型通常具备更高的参数量、更强的上下文理解能力和更优秀的生成质量。当Agent需要进行深度推理、多步骤规划、复杂数据分析或生成高质量长文本时，会调用主模型。其技术特点在于能够处理更长的上下文窗口、更复杂的指令集，并提供更准确、更富有洞察力的输出。然而，这种高性能也伴随着更高的[[Token 消耗]]和计算资源需求，因此其使用需要被精心管理，以避免不必要的成本开销。

## 应用场景

主模型的应用场景主要集中在需要高级认知能力和复杂处理流程的任务中，例如：

*   **复杂问题解决：** 当[[AI Agent]]需要对用户提出的复杂问题进行深入分析、多方面权衡并给出详细解决方案时。
*   **多步骤任务规划：** 在执行需要分解为多个子任务并按特定逻辑顺序执行的复杂流程时。
*   **高质量内容生成：** 生成专业报告、创意文案、代码或进行复杂的数据总结和提炼时。
*   **深度语义理解：** 处理模糊、歧义较多的用户输入，或需要从大量非结构化数据中提取深层含义时。
*   **核心业务逻辑处理：** 在Agent系统中，涉及关键决策、策略制定或需要高度准确性的核心业务流程。

通过将这些复杂任务分配给主模型，而将[[批准]]、[[压缩]]、[[记忆刷新]]、[[MCP 调用]]、[[Session 搜索]]以及[[技能工具]]、[[视觉工具]]、[[网页工具]]等辅助性任务交由[[副模型]]处理，可以实现效率与成本的最佳平衡。
