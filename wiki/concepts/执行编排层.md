# 执行编排层

## 定义
执行编排层解决 Agent “下一步该做什么”的问题，它将任务拆解为子目标，按逻辑顺序串联执行步骤，并在每步后检查结果，必要时触发修正或重试，形成闭环流程。

## 在本库的具体例子
- [[OpenAI公司]] 强调的闭环流程被明确定义：理解目标 → 判断信息 → 补全信息 → 执行 → 分析结果 → 检查输出 → 修正/重试。这正是执行编排层的典型实现。
- [[Anthropic公司]] 的 Planner 角色负责拆解任务和制定策略，Generator 按计划执行，Evaluator 检查结果，整个流程由执行编排层协调。

## 技术实现细节
1. **目标拆解**：将用户高层次的指令（如“生成一份市场分析报告”）分解为可操作的子任务：收集数据 → 分析趋势 → 撰写报告 → 审阅格式。
2. **步骤串联**：使用状态机或 DAG（有向无环图）定义步骤依赖关系。例如，必须等“收集数据”完成后才能执行“分析趋势”。
3. **自我修正轨道**：在每步执行后设置检查点，若输出不达标则回退到上一步或切换到替代路径。例如，数据分析结果不符合预期，则触发重新收集数据。

## 与近似概念的边界
- **与 Agent 框架的 Chain**：许多框架提供 `Chain` 类串联步骤，但执行编排层更强调检查点和自修正，而不仅仅是顺序调用。
- **与业务流程管理（BPM）**：BPM 关注企业级工作流自动化，而执行编排层专为模型的不确定性设计，允许动态调整计划。

## 关联概念
- [[工具系统层]] — 执行编排层中的步骤依赖调用工具系统层的工具。
- [[记忆与状态管理层]] — 执行编排层需要读取和写入当前任务状态。
- [[约束校验与恢复层]] — 修正和重试机制属于约束恢复层的职责。
