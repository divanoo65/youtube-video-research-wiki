# 约束校验与恢复层

## 定义
约束校验与恢复层是 Harness 系统最后的防御线，它通过明确约束（行为红线）、执行严格校验（输出前和输出后），并在失败时提供重试、切换路径或状态回滚等恢复机制，确保系统在真实环境中具备自愈能力。

## 在本库的具体例子
- [[Harness-Engineering-技术深度简报-从提示词到系统驾驭的范式转]] 将该层描述为“将‘失败’视为常态进行处理”。
- [[OpenAI公司]] 的自驱动治理系统：将资深工程师的模块分层规则写成系统规则，当 Agent 输出违反规则时自动拦截并反馈修改方案，这是校验与恢复的自动化实例。

## 技术实现细节
1. **约束定义**：明确 Agent 不可越界的行为。例如，“不得调用外部网络来访问私有数据”、“不得在无权限情况下删除资源”。这些约束以规则形式存储在 Harness 中，与模型提示分离。
2. **输出前校验**：在模型返回最终结果前，先进行格式、安全、逻辑等检查。例如，校验生成的 JSON 是否标准、是否有 XSS 攻击向量。
3. **输出后校验**：执行完成后，由评估观测层验证实际效果，若不达标则触发恢复机制。
4. **恢复机制**：包括重试（同路径重新调用）、路径切换（换一个模型或换一种工具组合）和状态回滚（恢复到上一个安全步骤）。例如，API 超时后自动重试 3 次，若仍失败则切换为备用模型。

## 与近似概念的边界
- **与异常处理（Try-Catch）**：异常处理是代码层面的捕获机制，而约束恢复层是系统级的设计模式，覆盖语义约束、多步回滚等更高层次的故障。
- **与错误重试中间件**：重试中间件只做简单重试，约束恢复层包含路径切换和状态回滚，更灵活。

## 关联概念
- [[评估与观测层]] — 评估结果驱动恢复触发。
- [[执行编排层]] — 恢复机制中的路径切换需要重新编排执行步骤。
- [[自驱动治理系统]] — 是约束恢复层的高级实现，包含自动拦截和修改建议。
