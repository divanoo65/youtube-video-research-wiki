# 上下文反射

## 定义
上下文反射（Context Reflection）是 [[Anthropic公司]] 发现并解决的一种现象和应对方法：当模型的上下文窗口接近填满时，模型会表现出“焦虑”，倾向于仓促结束当前任务。Anthropic 的应对策略不是压缩现有上下文，而是直接创建一个全新的 Agent 进程，将当前任务状态完整移交，类似操作系统的进程重启，从而消除上下文极限带来的心理压力。

## 在本库的具体例子
- 在 [[Harness-Engineering-技术深度简报-从提示词到系统驾驭的范式转]] 中，上下文反射被描述为 Anthropic 处理上下文极限的工程手段，属于 Harness 约束恢复层的实践。
- 视频中强调，Anthropic 不仅压缩上下文，而是“用一个全新的、干净的 Agent 承接状态”，与简单截断或摘要不同。

## 技术实现细节
1. **状态序列化**：在触发上下文反射时，将当前 Agent 的任务状态（已完成步骤、中间结果、当前目标、工具调用记录）序列化为结构化对象。
2. **新 Agent 启动**：撤销旧 Agent 实例，启动新实例，并将序列化的状态作为初始上下文注入。新 Agent 拥有干净的上下文窗口，但任务连续。
3. **交接时机**：监测上下文占用率（如 token 计数达到窗口的 80%），提前发起交接，避免模型在焦虑下做出错误决策。

## 与近似概念的边界
- **与上下文压缩（Summarization）**：压缩会丢失细节，上下文反射不压缩，而是无损转移状态，保证信息完整性。
- **与多轮对话的槽位填充**：槽位填充是预定义信息收集，上下文反射用于任意长期任务，且自动检测触发时机。

## 关联概念
- [[记忆与状态管理层]] — 上下文反射依赖于状态管理层的序列化能力。
- [[约束校验与恢复层]] — 上下文反射属于恢复层的一种机制（Agent 进程重启）。
- [[角色分离机制]] — 新 Agent 可以承担 Generator 角色，而旧的 Evaluator 继续存在。
