## 概念定义

全自动模式是 [[Web Video Presentation Skill]] 中定义的一种录制模式，核心目标是通过音频节奏驱动演示页面的自动翻页与动效播放，从而彻底解放人工操作，实现“一键录屏”。传统录制过程中，讲师需要手动切换幻灯片或步骤，即便借助自动播放功能也往往需要预设固定计时，难以适应不同内容的自然节奏。全自动模式将音频文件（TTS生成的口播音频）作为控制信号，利用音频频谱分析或时间轴标记，使页面在每段音频播放完毕时自动跳转至下一帧，同时触发预先绑定的动画、数字浮现、高亮等动效。这种模式使得从“编写脚本 → 生成音频 → 网页演示”的整条链路完全闭环，人工只需要启动录制，后续所有页面跳转、动效时机均由音频自动编排。与 [[音频模式]] 相比，全自动模式不依赖人工手动点击翻页，而是将操作权完全交给音频，适用于对节奏要求严格、希望复刻真人讲解体验的录制场景。其实质是将“人的判断”在流程中替换为基于音频的自动决策，从而进一步提升生产流程的自动化程度。结合 [[Harness 实践：Agent 全自动知识讲解视频生成技术简报]] 中提出的“反馈修复最小切片”原则，全自动模式在生成过程中如果检测到某帧动画与音频不匹配，可以自动回退到对应步骤进行修正，而不会影响已完成的其他部分。

## 技术细节

- **音频标记对齐**：在 TTS 生成阶段（调用 [[MiniMax CLI]] 批量生产音频）时，同步生成时间轴标记文件（如 JSON 格式），记录每段音频的起始、结束时间以及对应的脚本段落 ID。全自动模式通过 `requestAnimationFrame` 或 `setInterval` 轮询当前播放位置，匹配标记文件，在指定时刻触发页面状态变更。
- **动画状态机**：每个页面内部维护一个状态机，状态包括 `idle`, `entering`, `playing`, `exiting`。全自动模式将 “开始播放”、“结束播放” 等事件与音频标记绑定。当音频进入某个段落时，状态机跳转并执行对应的 CSS 动效（如渐变、位移、数字闪烁）。
- **翻页触发**：翻页不依赖用户点击，而是监听音频结束事件。为了应对音频意外中断或超时，设置了超时回退：若音频文件长度与标记不一致，则等待固定时长后强行翻页，或抛出告警由 Harness 调度器 处理。
- **隔离性**：每章采用独立的文件夹和 CSS 类名（即 [[物理隔离]] 原则），确保全自动模式切换页面时不会因样式冲突导致动效错乱。音频文件命名与开发步骤一一对应，便于快速定位和修复。

## 应用场景

- **知识讲解视频一键生成**：用户只需编写脚本，运行 Harness 流程（包括 TTS 生成和全自动录制），便可直接产出一段完美同步语音与动画的演示视频，无需后期剪辑。
- **产品演示与培训材料**：需要统一输出节奏、保证每个步骤精确时长的场景，全自动模式可以消除不同讲师操作速度差异带来的不一致性。
- **自动化 CI/CD 录屏集成**：在持续集成环境中，全自动模式可作为测试用例的录制组件，自动生成产品功能演示视频并打包到发布包中。
