# 索引化指令文件

## 定义
索引化指令文件是 [[OpenAI公司]] 提出的一种提示词工程优化方法：放弃将所有规则、约束和背景信息塞进单一庞大提示词，而是将核心提示词改造为“目录页”（通常命名为 `Agent.md`），仅保留最必要的索引，详细规范（如设计文档、质量评分标准、命名规范等）作为独立的子文档，在模型需要时通过“按需暴露”的方式动态加载。

## 在本库的具体例子
- 在 [[Harness-Engineering-技术深度简报-从提示词到系统驾驭的范式转]] 中，OpenAI 的 `Agent.md` 被描述为“目录页”，只包含核心索引，详细规范作为子文档按需暴露。这属于 Harness 信息边界层的具体实现。

## 技术实现细节
1. **目录页（`Agent.md`）**：约 10-20 行，包含标题、角色定义、关键行为规则摘要、子文档列表。例如：
   ```markdown
   # Agent 核心指令
   - 角色：资深后端工程师
   - 行为规则：见 docs/rules.md
   - 质量评分标准：见 docs/quality.md
   ```
2. **按需暴露**：当 Agent 判断需要特定规则时（如准备提交代码时触发格式校验），Harness 系统自动将 `docs/rules.md` 注入到上下文。注入后模型可在该回合参考完整细节。
3. **索引更新**：子文档更新时无需修改 `Agent.md`，只需维护目录链接，降低维护成本。

## 与近似概念的边界
- **与长提示词（Long Prompt）**：长提示词一次性将所有信息塞入，容易让模型忽略中间关键约束；索引化指令通过分页暴露保证注意力集中。
- **与 RAG 的检索**：RAG 是检索外部知识，索引化指令的“按需暴露”也是检索的一种（从预定义的文档集中检索），但检索源是固定规范集而非互联网。

## 关联概念
- [[信息边界层]] — 索引化指令是信息边界层中“结构化组织”和“信息裁剪”的典型应用。
- [[渐进式暴露]] — 两者的思想一致：逐步暴露，避免信息过载；实现方式不同（索引化指令使用子文档目录，渐进式暴露使用动态技能加载）。
- [[自驱动治理系统]] — 索引化指令的维护可以接入自驱动治理系统自动更新规则。
