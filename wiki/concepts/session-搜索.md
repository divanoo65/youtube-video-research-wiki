## 概念定义

在[[赫美斯 (Hermes) Agent 高级应用与部署指南：云端集成、界面优化与 Token 成本控制]]中，**Session 搜索**被定义为一种由[[副模型]]（辅助模型）处理的特定任务。这种架构设计旨在优化资源分配和降低运行成本。当用户或系统需要回顾、检索或管理历史会话（Session）信息时，这项任务会被分配给成本较低、响应速度更快的辅助模型来执行，而非占用昂贵的主模型资源。Session 搜索的核心目的是高效地定位和提取与特定会话相关的上下文、对话记录或关键信息，从而为后续的交互或决策提供支持。它属于辅助任务范畴，与[[批准]]、[[压缩]]、[[记忆刷新]]等任务一同，共同构成了赫美斯 Agent 智能体分工协作的重要组成部分。通过将这类相对简单的检索任务从主模型中剥离，系统能够显著提升整体运行效率并有效控制[[Token 成本控制]]。

## 技术细节

Session 搜索被归类为“简单任务”，其技术细节主要体现在以下几个方面：

1.  **任务性质：** 相比于需要复杂推理、生成或多模态理解的[[主模型]]任务，Session 搜索通常涉及对结构化或半结构化数据的快速索引、匹配和检索。这可能包括基于关键词、时间戳、用户ID或其他元数据进行查询。
2.  **模型选择：** 赫美斯 Agent 推荐使用响应速度快且成本低的[[Flash 系列模型]]（如 Jamin 2.5 Flash / Jamin 2.25 Flash）来处理此类任务。这些模型虽然在复杂推理能力上可能不如主模型，但其在处理特定、明确的检索任务时表现出色，且能大幅降低[[Token 消耗]]。
3.  **效率与成本：** 将Session 搜索任务卸载给辅助模型，避免了每次进行历史会话查询时都调用高成本的主模型，从而实现了显著的[[Token 成本控制]]。同时，由于辅助模型通常具有更快的响应速度，用户体验也能得到提升。
4.  **与[[记忆刷新]]的协同：** Session 搜索可能与[[记忆刷新]]任务协同工作。记忆刷新负责维护和更新会话记忆，而Session 搜索则负责从这些记忆中提取所需信息。

## 应用场景

Session 搜索在多种AI Agent应用场景中都扮演着重要角色：

1.  **多轮对话管理：** 在[[多轮对话管理]]中，Agent需要频繁回顾之前的对话内容以保持上下文连贯性。Session 搜索能够快速定位并提取特定轮次的对话记录或关键信息。
2.  **用户历史查询：** 当用户询问“我上次问过你什么？”或“我们之前讨论到哪里了？”时，Session 搜索能够迅速从历史会话中找到相关记录并提供给主模型进行总结或回复。
3.  **上下文填充：** 在执行复杂任务前，Agent可能需要从历史会话中获取特定的配置、偏好或用户意图，Session 搜索可以高效地完成这一上下文填充工作。
4.  **故障排查与日志分析：** 在Agent的运维和调试过程中，通过Session 搜索可以快速定位到特定时间段或特定用户的问题会话，辅助进行故障排查和行为分析。
5.  **个性化服务：** 基于用户历史会话数据，Session 搜索可以帮助Agent理解用户偏好，从而提供更个性化的推荐或服务。
