# 上下文重映

**定义**
上下文重映（Context Reflected）是 [[Anthropic]] 提出的一种上下文管理策略。当 Agent 的上下文窗口因长时间运行而积累过多信息，导致模型丢失早期细节或产生「急于收尾」行为时，系统不是对上下文做压缩摘要，而是启动一个干净的「新 Agent」实例来接手后续任务，通过「进程级重置」解决上下文退化问题。

**在本库的具体例子**
- 在 [[Harness-Engineering-深度简报-构建稳定-可落地的-AI-Agent系统]] 中，上下文重映被描述为类似程序开发中的「重启进程」以解决内存泄漏。当上下文过长时，模型可能会对中间步骤产生厌倦或忽略早期指令，此时启动一个全新 Agent 实例，将关键状态传递给新实例，实现干净的上下文延续。
- 这是 [[Anthropic]] 应对「上下文焦虑」（Context Anxiety）的核心手段之一，与 [[独立评估者模式]] 配合使用，在长链路任务中显著提升稳定性。

**技术实现细节**
1. **状态传递**：新 Agent 实例并非完全从零开始，而是接收经过筛选的关键中间结果、当前目标和已完成步骤摘要，而非全部原始上下文。
2. **触发条件**：根据上下文 token 计数（如接近窗口 80%）或行为检测（如模型开始忽略早期指令）触发上下文重映。
3. **交接协议**：旧 Agent 在重置前执行一次「状态导出」，将当前任务状态、已完成步骤、未完成任务打包为结构化数据传入新 Agent 的系统 Prompt。
4. **环境隔离**：新 Agent 运行在独立的环境中，避免旧 Agent 中积累的偏差影响新实例的判断。

**与近似概念的边界**
- **区别于 [[上下文工程]] 的上下文压缩**：上下文压缩是通过摘要减少 token 量，所有信息保留在同一个 Agent 实例中；上下文重映是通过 Agent 实例替换「刷新」上下文环境，而非压缩。
- **区别于 [[执行编排层]] 的检查点恢复**：检查点恢复是在失败后从最近的保存点继续执行，上下文重映是主动预防性的上下文「垃圾回收」，在失败发生前就切换实例。
