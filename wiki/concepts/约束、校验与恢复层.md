# 约束、校验与恢复层

**定义**
约束、校验与恢复层（Constraint, Validation & Recovery Layer）是 Harness 六层架构的第六层（最底层），负责定义 Agent 的行为边界（哪些能做、哪些不能做）、实施严格的输入输出校验，并在 API 超时、格式混乱、理解偏差等故障发生时提供重试、回滚或切换路径的恢复机制。其核心目标是确保系统在部分步骤失败后能够拉回正轨并稳定交付，而非从根本上从头开始。

**在本库的具体例子**
- 在 [[Harness-Engineering-深度简报-构建稳定-可落地的-AI-Agent系统]] 中，约束、校验与恢复层被描述为 Harness 六层架构的最后一层。视频指出，在长链路任务中失败是常态，系统必须具备状态保存和路径回滚能力。当 Agent 的 API 调用超时或输出格式与预期不符时，该层负责触发重试或切换到备选路径（如降级使用本地模型）。
- 该层与 [[执行编排层]] 配合：执行编排层负责步骤的串联，约束与恢复层负责在步骤失败时提供恢复能力。同时，约束规则（如「不允许删除生产数据库」）在校验环节拦截违规行为。

**技术实现细节**
1. **行为边界定义**：通过规则引擎定义硬约束（如禁止调用某些 API、禁止修改某些文件）和软约束（如代码必须通过 lint 检查），硬约束直接拦截，软约束记录告警。
2. **输入输出校验**：对每个步骤的输入参数进行格式和范围检查（如日期格式、数值范围），对输出结果进行 Schema 验证（如 JSON Schema），不满足条件时自动触发重试。
3. **检查点与快照**：在关键步骤执行前保存执行状态的持久化快照（包括当前上下文、已完成步骤、中间结果），以便在失败时恢复到最近的检查点。
4. **重试策略**：支持指数退避重试（Exponential Backoff）和有限次重试（如最多 3 次），超过阈值后触发失败切换。
5. **路径切换机制**：当主路径多次失败后，切换到备选执行路径，例如主模型失败后降级使用本地小模型，或主搜索 API 超时后切换备用数据源。
6. **回滚能力**：对于已产生副作用（如写入文件、修改数据库）的步骤，维护操作日志以支持逆向操作（Undo）。

**与近似概念的边界**
- **区别于 [[执行编排层]]**：执行编排层负责「如何安排步骤的顺序」，约束与恢复层负责「当步骤失败时怎么办」。编排层关心流程的正确性，恢复层关心系统的容错性。
- **区别于传统异常处理**：传统软件工程的异常处理是确定性的（try-catch 逻辑固定），而约束与恢复层需要处理模型产出的不确定性——模型可能产生格式正确但语义错误的输出，这需要结合 [[评估与观测层]] 的验证。
