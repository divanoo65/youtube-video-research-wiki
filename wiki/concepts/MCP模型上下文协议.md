# MCP 模型上下文协议

## 定义

MCP（Model Context Protocol，模型上下文协议）是一种通信协议，它允许 AI 代理绕过浏览器安全限制，通过连接到特定平台的专用客户端（如桌面客户端或移动客户端）来执行操作。当某些网站（如淘宝）设置的安全策略禁止浏览器插件访问其页面时，MCP 提供了一个替代路径：AI 转向该平台的本地客户端，通过客户端提供的 API 或界面来实现类似的操作。

## 技术实现细节

根据报告的提及，MCP 的工作原理如下：
1. AI 检测到目标网站（如淘宝）的页面安全策略阻止了 Codex 的 DOM 访问。
2. Codex 判断该平台是否支持 MCP 客户端连接。
3. 如果支持，Codex 启动本地客户端（可能是 Electron 应用或原生应用），通过 MCP 协议与该客户端建立双向通信。
4. 客户端暴露一系列工具（如搜索商品、查看订单、加入购物车），AI 通过调用这些工具来完成任务，而不再依赖浏览器渲染。
5. 这种方式根本上绕过了网页端的安全策略，因为操作发生在客户端原生环境中。

目前 MCP 仍是一种新兴方案，尚未在实测中完整演示。

## 在本库的具体例子

在 [[sources/Codex-Chrome-插件功能实测与深度解析简报]] 中，当 Codex 尝试对淘宝执行操作时，触发站点安全策略拦截，报告明确指出“某宝无法集成 Chrome 插件，对某宝触发了站点安全策略拦截，明确禁止 Codex 去使用”。随即提出替代方案：“可探索通过 MCP 连接至该平台的专用客户端”。这是本报告中 MCP 被提及的唯一具体场景，但目前尚未有成功实施的案例。

## 与近似概念的边界

- **区别于普通 API 集成**：普通 API 集成需要平台公开 API 端点，而 MCP 可以不依赖公开 API，通过客户端内部接口实现。
- **区别于浏览器插件**：MCP 是一种独立于浏览器的协议，用于与原生客户端通信，而 Codex 的主体功能依赖于浏览器插件。
- **区别于 RDP/VNC**：RDP/VNC 是远程桌面控制协议，MCP 则是客户端内部通信协议，更轻量。
