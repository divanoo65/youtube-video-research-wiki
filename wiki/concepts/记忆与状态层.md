# 记忆与状态层

**定义**
记忆与状态层（Memory & State Layer）是 Harness 六层架构的第四层，负责对 Agent 运行过程中的各类信息进行分类管理，明确区分当前任务状态、对话中间结果、长期记忆与用户偏好。其核心目标是防止系统运行越久上下文越混乱——避免短期对话内容污染长期知识库，也避免长期记忆占用宝贵的上下文窗口。

**在本库的具体例子**
- 在 [[Harness-Engineering-深度简报-构建稳定-可落地的-AI-Agent系统]] 中，记忆与状态层被描述为 Harness 架构中的重要组成部分。视频指出，Agent 需要区分三类信息：当前任务正在进行的步骤和状态、本次对话中产生的中间结果、以及跨会话的长期记忆和用户偏好。混合使用这些信息会导致系统混乱。
- [[Hermes-Agent]] 中的「Refresh Memory（重刷记忆）」辅助任务正是记忆与状态层的具体实现——副模型负责在需要时刷新或压缩会话记忆，确保主模型的上下文不会被积累的对话历史污染。
- [[约束、校验与恢复层]] 中的检查点快照机制依赖于记忆与状态层保存的状态信息，以实现在故障时恢复到最近的检查点。

**技术实现细节**
1. **三级存储体系**：
   - **短期工作记忆（Working Memory）**：当前任务的上下文和进行中的状态，保存在模型上下文窗口中，任务完成后可释放。
   - **对话记忆（Episodic Memory）**：本次对话的交互历史，以摘要或关键事件列表形式存储，供后续步骤引用。
   - **长期记忆（Long-term Memory）**：跨会话的用户偏好、知识积累和技能记忆，以结构化数据或向量形式持久化到数据库。
2. **状态序列化与快照**：在关键步骤执行时将当前工作记忆序列化为结构化状态（JSON/YAML），持久化到本地文件或外部存储，支持故障恢复和会话恢复。
3. **记忆压缩与裁剪**：当短期记忆接近上下文窗口限制时，触发压缩策略（如将多轮对话摘要为关键事实）或裁剪策略（移除不相关的对话历史）。
4. **记忆访问控制**：定义不同层级记忆的访问权限——长期记忆在初始化时加载到系统 Prompt，对话记忆按需注入，工作记忆随步骤推进动态更新。

**与近似概念的边界**
- **区别于 [[信息边界层]]**：信息边界层关注「当前任务需要什么外部信息」，属于输入筛选；记忆与状态层关注「系统已经有什么内部状态」，属于运行态管理。信息边界层是静态的上下文裁剪，记忆与状态层是动态的状态演进。
- **区别于数据库/缓存系统**：传统数据库只负责存储，不关心内容的语义层级；记忆与状态层按照任务语义区分信息的重要性和时效性，是一种面向 Agent 的智能存储设计。
