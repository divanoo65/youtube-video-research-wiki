# 分层记忆体系

## 定义
Hermes Agent 构建的一套高度优化的多层级持久化存储系统，旨在平衡 Token 效率与长时序信息检索。体系分为四层，从高优先级、小容量的核心记忆到低优先级、无限容量的程序性记忆。

## 本库的具体例子

Hermes Agent 的四层记忆体系：

1. **核心持久记忆**: 包含 MEMORY.md（精简笔记）和 USER.md（偏好配置），容量严格限制在约 1300 tokens，确保最关键信息常驻且高效
2. **会话历史记忆**: 基于 SQLite (state.db) 和 FTS5 全文索引，支持无限扩展。通过大模型对检索片段进行摘要重构
3. **扩展记忆层 (Honcho)**: 可选层级，用于构建长期用户画像，捕捉深层行为模式
4. **程序性记忆**: 即自动生成的技能（Skill），存储在 `~/.hermes/skills/`，存储"学会了怎么做"的行动模式

对比 OpenClaw 的 Markdown 记忆存储方案（单纯的 MEMORY.md 文件），Hermes 的 SQLite+FTS5 方案在检索效率上具有显著优势。
