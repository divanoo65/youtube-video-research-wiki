# 分层记忆体系

## 定义

分层记忆体系是 Hermes Agent 构建的一套比传统聊天记录更复杂的多层级记忆系统。它将记忆分为四个层级，旨在提升 Token 效率与检索精准度，使 Agent 既能在当前会话中高效运作，又能在跨会话中积累知识与经验。

## 在本库的具体例子

在 [[Hermes-Agent-与-OpenClaw-深度对比简报]] 中，详细描述了 Hermes 的四层记忆结构：
1. **核心持久记忆**：MEMORY.md（核心笔记）+ USER.md（用户偏好），约 1300 tokens，优先级最高
2. **会话历史记忆**：基于 SQLite (FTS5) 的无限扩展历史库，支持全文索引与摘要重构
3. **扩展记忆层 (Honcho)**：实现跨会话的深层用户理解，建立长期用户画像与行为模式
4. **程序性记忆**：存储自动生成的技能（可复用的工作流），即"怎么做"而非"知道什么"

## 技术实现细节

- SQLite 集成 FTS5 全文搜索引擎，支持高效的模糊搜索和相关性排序
- 每层记忆有独立的存储空间和优先级权重，按需加载到上下文
- 程序性记忆层与 `skill_manage` 工具集成，自动将执行经验转化为可复用技能

## 与近似概念的边界

- **分层记忆体系 ≠ 普通上下文管理**：普通上下文管理只关心当前窗口，分层记忆体系管理跨会话的知识积累。
- **分层记忆体系 ≠ 知识库检索**：知识库检索是外部文档的向量化搜索，分层记忆是 Agent 自身的经历和经验。
