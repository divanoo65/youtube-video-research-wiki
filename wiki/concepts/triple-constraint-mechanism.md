## 概念定义

三重约束机制是 Hermes Agent 为了在自主执行任务过程中保障系统安全、防止智能体越权或滥用工具而设计的一套分层次的安全防护体系。该机制由三个相互补充的层面构成：**工具权限控制（Permissions）**、**沙箱运行环境（Sandboxing）** 以及**指定受限工具的 C 调度**。工具权限控制负责在调用层面限制智能体对特定工具或 API 的访问权限，只有经过授权的工具才能被激活；沙箱运行环境则提供一个隔离的执行上下文，将智能体的代码执行、文件操作和网络请求限制在受控的虚拟环境中，避免对宿主系统造成破坏；而 C 调度是一种针对高风险或关键系统工具的调度策略，通过将部分工具的调用权统一交由一个专门的安全调度器（以 C 语言实现以降低性能开销）管理，从而在底层阻断未经授权的调用。这三层机制层层递进，从“谁可以用”到“在哪里用”再到“怎么用”，共同构建了一个纵深防御的智能体安全模型。

## 技术细节

三重约束机制的技术实现依赖于 Hermes Agent 的运行时架构：

- **工具权限控制（Permissions）**：在智能体启动阶段，系统读取预先定义的工具权限配置文件（JSON/YAML），该文件列出了每个工具的最小权限集合（如只读、只写、禁止网络等）。智能体在调用任何工具前，必须先经由权限管理器进行签名验证和授权检查。权限可以基于用户角色、会话 ID 或任务类型动态调整，实现细粒度的访问控制。

- **沙箱运行环境（Sandboxing）**：Hermes Agent 利用操作系统级别的容器化技术（如 Linux Namespace 和 cgroups）或轻量级虚拟机（如 Firecracker）为每次智能体执行创建独立的沙箱。沙箱内屏蔽了宿主机的敏感目录（如 `/etc`、`/proc`），限制 CPU 和内存用量，并阻断所有非授权的外网连接。智能体产生的临时文件在沙箱销毁时自动清理，防止持久化污染。

- **指定受限工具的 C 调度**：对于如文件系统写操作、进程创建、硬件调用等高危工具，Hermes Agent 将其调用集成到一个由 C 语言实现的安全调度器（Scheduler C）中。该调度器运行在独立的低权限进程内，只接受经过签名和序列化的调用请求，并强制进行白名单校验。C 调度的设计保证了即使 Python/JavaScript 主进程被攻破，攻击者也无法绕过该底层调度器直接执行危险操作。

这三种机制协同工作，将智能体的能力封闭在受控范围内，同时保留足够的灵活性用于正常任务。

## 应用场景

- **自主编程与代码执行**：当 Hermes Agent 被要求编写并运行代码时，三重约束机制确保代码只能在沙箱中执行，无法访问宿主系统文件，且写入磁盘的临时文件受 C 调度限制，从而避免恶意代码破坏开发环境。
- **多工具链调用**：在集成 MCP 协议接入数千个外部应用时，权限控制层负责对每个工具（如发送邮件、读写数据库）进行授权，防止智能体误操作或越权使用内部服务。
- **长期运行与记忆管理**：在持久记忆和技能记忆的写入过程中，C 调度和沙箱保证对 `.hermes` 目录的操作必须经过安全审查，防止记忆污染或数据泄露。
