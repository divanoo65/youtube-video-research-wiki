# 意图导向 (Intent-Oriented)

## 概念定义
意图导向（Intent-Oriented）是一种在 AI 智能体协作架构中，针对跨会话上下文传递所设计的核心机制。在传统的 AI 交互中，会话往往是孤立的，导致上下文丢失或信息碎片化。意图导向机制通过在 `handoff` 技能包中引入“未来意图”参数，将当前会话的总结、决策与明确的后续执行指令进行结构化封装。它不仅仅是信息的简单堆叠，而是通过显式的“意图声明”，为接收文档的智能体提供明确的行动指南。这种模式确保了从设计阶段到编码阶段，或者从一个 Agent 到另一个 Agent 的任务流转过程中，目标的一致性与执行的连贯性，从而避免了因上下文模糊而导致的逻辑偏差。

## 技术细节
在 `handoff` 技能包的实现中，意图导向通过以下技术手段落地：

*   **参数化传递：** 在执行 `handoff` 操作时，系统允许开发者或前序 Agent 传入特定的 `intent` 参数。该参数会被嵌入到生成的 Markdown 文档头部或元数据区。
*   **结构化上下文封装：** 意图导向不仅仅记录“发生了什么”，更强调“接下来要做什么”。它将当前会话的精华（如决策点、待办事项）与意图指令绑定，形成一个标准的交付协议。
*   **可读性优先：** 与传统的 JSONL 格式不同，意图导向生成的文档采用 Markdown 格式，将意图描述置于显眼位置，确保人类开发者与 AI 智能体都能快速解析任务目标。
*   **去重与精炼：** 意图导向机制通常配合去重逻辑，确保传递的意图不与现有的 Artifacts（如 PRD、ADR）冲突，而是作为对现有文档的补充与行动指引。

## 应用场景
1.  **跨 Agent 任务流转：** 当架构设计 Agent 完成系统方案设计后，通过意图导向将设计决策与“编码实现”的意图传递给编码 Agent，确保编码过程严格遵循设计意图。
2.  **长周期项目协作：** 在复杂的开发任务中，当一个会话无法完成所有工作时，通过意图导向记录当前进度与下一步计划，使得后续会话能够无缝接管任务，无需重新梳理上下文。
3.  **人机协作接口：** 在 AI 辅助开发中，意图导向生成的文档可作为人类开发者与 AI 之间的沟通桥梁，人类开发者通过阅读意图描述，快速理解 AI 的工作逻辑与后续建议。
