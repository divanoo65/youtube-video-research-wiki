## 概念定义

进程监控是 Hermes Agent 智能中控台（Dashboard）中的一项关键功能，旨在对系统中所有正在运行的 AI 代理进行实时跟踪与管理。这些代理包括 Hermes（主代理）、Cloud（云端代理）、Codex（代码生成代理）、Cursor（编辑器集成代理）等，它们各自承担不同的任务，共同构成一个多代理协作网络。进程监控不仅仅是一个简单的“进程列表”，它通过与 `state.db` 等底层数据文件的交互，将每个代理的活跃程度、资源占用、任务执行状态等抽象指标转化为可视化的数据面板。例如，用户可以通过中控台查看每个代理的“学习进度”（以 `state.db` 文件大小作为参考），24 小时活动热力图，以及高频 Prompt 提醒（出现三次以上的 Prompt 会标记闪电符号，提示用户是否将其自动化）。这种监控方式将抽象的 AI 行为量化，使得运维人员能够像管理传统服务器进程一样，直观地评估 AI 系统的健康度和效率。与此同时，进程监控还集成了异常检测功能，当某个代理出现响应延迟、错误率上升或意外退出时，中控台会发出告警，并联动纠错管理模块（Correction）进行来源追踪。总之，进程监控是连接底层 AI 引擎与上层管理决策的桥梁，是实现“AI 运维自动化”的基石。

## 技术细节

### 数据采集与存储
- **state.db**: 每个代理的运行时状态以 SQLite 数据库文件的形式持久化，包含历史行为、对话记录、学习参数等。进程监控通过轮询这些文件的修改时间和大小，判断代理是否活跃。
- **心跳机制**: 每隔 5 秒，各代理向中控台发送心跳包，包含 PID、CPU 占用、内存使用、当前任务 ID 等信息。若连续 3 次未收到心跳，则认为进程失联，触发告警。
- **事件流**: 代理的所有关键操作（如任务开始/结束、错误、用户交互）都会以 JSON 事件流的形式推送到中控台的 WebSocket 通道，实现毫秒级实时更新。

### 可视化组件
- **进程列表**: 以表格形式展示代理名称、状态（运行/休眠/异常）、启动时间、运行时长、最近一次活动时间。支持按名称、状态、CPU 负载排序。
- **24 小时活动热力图**: 每个代理对应一行，时间轴以 15 分钟为粒度，颜色深浅代表活动频次。鼠标悬停显示具体事件数，点击可展开详细日志。
- **高频 Prompt 提醒**: 自动聚类重复的 Prompt 文本，统计出现次数。当某条 Prompt 出现 ≥3 次时，在进程监控面板右侧以闪电图标高亮，并附带“一键自动化”按钮，引导用户创建 Cron 任务。
- **异常指标仪表盘**: 显示各代理的错误率、平均响应时间、内存泄漏趋势等，使用折线图与百分位线（P50/P95/P99）。

### 集成接口
- 进程监控通过 gRPC 与各代理通信，支持动态注册与注销。新代理启动后自动向中控台注册，无需手动添加。
- 提供 REST API 供外部系统（如 Prometheus）拉取指标，便于融入已有的监控体系。

## 应用场景

1. **多代理协作调试**  
   在复杂的 AI 工作流中，多个代理可能同时处理不同子任务（如 Hermes 负责上下文管理，Codex 生成代码，Cloud 调用外部 API）。进程监控让开发者能够一眼看清哪个代理卡住了、哪个代理资源耗尽，从而快速定位瓶颈。

2. **自动化运维与告警**  
   运维团队可将进程监控面板作为日常巡检工具。当代理异常退出或响应超时，中控台自动发送通知（邮件/Slack/钉钉），并建议执行预定义的恢复脚本（如重启代理、清理内存缓存）。结合 [[热力图分析]]，还能发现代理负载的周期性峰值，提前规划扩容。

3. **个人效率提升**  
   对于独立开发者或高级用户，进程监控中的“高频 Prompt 提醒”功能可以显著减少重复劳动。例如，每天执行的“检查 GitHub 状态”、“生成每日摘要”等操作会被自动识别并推荐转化为 Cron 任务，从而释放人力。

4. **安全审计**  
   通过查看代理的完整活动历史（由 `state.db` 记录），管理员可以追溯任何异常操作，例如某个代理在非工作时间突然访问敏感数据。结合 [[Hermes Agent 智能中控台深度分析简报]] 中的整体架构，进程监控成为审计日志的核心入口。
