# 四层解耦架构

## 概念定义
四层解耦架构（Four-Layer Decoupled Architecture）是一种针对多 Agent 协作系统设计的现代化范式，旨在解决传统 Agent 系统因与本地环境深度绑定而导致的“宠物”式脆弱性。在该架构中，Agent 不再被视为拥有长期状态的独立个体，而是被拆解为静态定义与动态执行两个维度。通过将 Agent 的身份定义、运行时会话、记忆存储以及底层基础设施进行彻底的层级分离，系统实现了“无状态化”运行。这种设计使得 Agent 能够像“牛马”一样作为消耗品存在：一旦运行环境出现污染、冲突或崩溃，系统可以立即销毁当前的执行沙盒，并基于存储的记忆状态秒级重建，从而极大地提升了复杂任务处理的容错率与系统的可扩展性。

## 技术细节
该架构通过对 Anthropic SDK 等现代 Agent 框架的逆向分析，抽象为以下四个递进层级：

1.  **Agent (静态模板)**：作为架构的最顶层，负责定义“我是谁”。它包含模型配置、系统提示词（System Prompt）、工具集定义以及 MCP（Model Context Protocol）服务器配置。这一层是静态且可复用的，不包含任何运行时数据。
2.  **Session (动态执行)**：负责定义“我在干什么”。它是 Agent 的运行时实例，承载具体的对话线程、记忆挂载点和临时文件。Session 支持并发运行，允许同一 Agent 模板在不同上下文中独立执行。
3.  **Session Store (记忆管理)**：实现记忆与环境的解耦。该层支持对记忆树进行 Fork（分支）、回滚（Rollback）和克隆（Clone）操作，使得系统能够回溯到任务的特定节点，或在多分支任务中并行探索。
4.  **Managed Infra (托管设施)**：架构的最底层，负责实现 Agent 的彻底无状态化。通过提供独立的沙盒环境，确保 Agent 的执行过程与宿主环境隔离，支持随时销毁与秒级重建。

## 应用场景
*   **高并发自动化任务**：在需要同时运行数十个 Agent 进行复杂代码开发或数据分析的场景中，利用无状态特性避免环境冲突。
*   **复杂任务容错处理**：当 Agent 在执行长链路任务（如自动化测试或系统部署）时，若遇到环境污染，系统可自动销毁当前沙盒并从 Session Store 恢复状态，无需人工干预。
*   **多 Agent 协作编排**：配合编排者（Coordinator）模式，将任务拆解为任务图（Task Graph），通过四层解耦架构实现不同 Skill Agent 的异步协作与高效调度。
