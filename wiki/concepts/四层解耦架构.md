# 四层解耦架构

## 定义
四层解耦架构是 Anthropic Managed Agents 的工程化实现模型，将 Agent 系统拆解为四个独立且可分别管理的层：**Agent 层**（静态模板定义）、**Sandbox 层**（执行环境）、**Session 层**（动态状态/记忆）、**Session Store 层**（云端持久化）。每层之间通过明确定义的接口交互，替换某一层不影响其他层。

## 在本库的具体例子
- [[Anthropic公司]] 的 SDK Beta 目录直接体现了这四层：`Agents` 模块对应第一层，`Environments` 模块对应第二层，`Sessions` 模块对应第三层，`Memory Stores` 对应第四层。
- 报告指出：2024年4月19日合并 Session Store 代码后，Agent 彻底无状态化，这正是第四层完成的结果。
- 脑图节点显示为“Anthropic 架构四层递进”，包含四个子节点。

## 技术实现细节
1. **Agent 层**：静态定义 Agent 身份，包括模型、系统提示词、可用工具列表及 MCP 服务器绑定。同一 Agent 模板可被多次实例化。
2. **Sandbox 层**：执行环境解耦。每个 Agent 实例化时分配一个独立的沙盒（如 Docker 容器），沙盒内不保存任何状态。
3. **Session 层**：动态会话容器，记录对话线程、挂载的记忆文件。支持状态克隆、回滚和分支，允许 Coordinator 精细控制。
4. **Session Store 层**：将记忆从本地存储迁移到云端（如 Redis），使得 Agent 可以在任何主机上重建会话。

## 与近似概念的边界
- **MVC 架构**：MVC 分离展示、业务和数据，而四层解耦专注于 Agent 的身份、执行、状态和持久化，维度不同。
- **容器化部署（Docker Compose）**：容器化部署通常只处理执行环境（对应第二层），而四层解耦还覆盖了 Agent 的定义、会话管理和记忆外部化，更完整。
