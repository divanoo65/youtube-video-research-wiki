# 主副模型配置

## 定义
主副模型配置是一种 AI Agent 的 Token 优化策略，核心逻辑是将任务细分，由昂贵的主模型处理复杂逻辑，由廉价的副模型（辅助模型）处理简单任务。在 [[hermes-agent]] 中，该配置通过修改配置文件中的辅助任务（Auxiliary Tasks）部分实现。

## 在本库的具体例子

- 在 [[hermes-agent-高级玩法与优化指南-深度简报]] 中，视频详细介绍了如何通过主副模型配置大幅降低 Token 消耗。推荐使用 Google [[gemini-2.0-flash]] 作为副模型，处理审批、压缩、MCP 调用等辅助任务。
- 具体配置步骤：进入配置文件，定位到辅助任务（Auxiliary Tasks）部分，将 `approval`、`compression`、`mcp` 等任务的模型 ID 修改为性价比高的模型（如 `gemini-2.0-flash`）。

## 技术实现细节

1. **任务细分**: 将 Agent 的任务分为复杂任务和简单任务，复杂任务由主模型处理，简单任务由副模型处理。
2. **可配置的辅助任务**: 包括审批（Approval）、压缩（Compression）、记忆刷新（Memory Refresh）、MCP 调用、会话搜索、技能检索（Skills）、网页/视觉相关任务等。

## 与近似概念的边界

- **模型蒸馏**: 模型蒸馏是通过训练小模型模仿大模型的行为，而主副模型配置是运行时动态分配任务，不涉及模型训练。
- **模型级联**: 模型级联通常指多个模型按顺序处理同一任务，而主副模型配置是并行或按需分配不同类型的任务。

## 关联概念

- [[token优化]] — 主副模型配置是 Token 优化的核心策略之一。
