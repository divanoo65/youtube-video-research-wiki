# 毫秒级沙盒

## 定义
毫秒级沙盒是一类轻量级容器技术，旨在将虚拟化环境创建时间压缩到毫秒量级（传统 Docker 容器启动通常需要数百毫秒到数秒）。这类技术对于 Agent 蜂群架构至关重要，因为当前每个子 Agent 执行结束后都需要销毁，新任务到来时需快速创建新沙盒。

## 在本库的具体例子
- [[Cube-Sandbox]] 是腾讯开源的毫秒级沙盒实现，本报告将其列为“高效基础设施”推荐之一。
- 在 [[Anthropic公司]] 的 Managed Agents 复刻方案中，推荐使用毫秒级沙盒来运行 `Docker 容器化 Worker`，以支持“预定义任务图”中快速切换子 Agent 执行环境。
- 报告指出：“传统 Docker 启动耗时较长，腾讯开源的 **Cube Sandbox** 等技术正致力于实现毫秒级启动”。

## 技术实现细节
- **轻量虚拟化**：利用 microVM（如 Firecracker）或基于 user-space 进程隔离（如 gVisor），跳过完整内核启动流程。
- **预冷池**：维持一个预热沙盒池，新请求到来时从池中快速分配。
- **镜像分层**：共享基础镜像层，只修改增量层，加速沙盒创建。

## 与近似概念的边界
- **Docker 容器**：Docker 启动通常需要秒级，毫秒级沙盒通过精简内核和预热机制达到更快速度。
- **无服务器函数（AWS Lambda）**：Lambda 有冷启动问题（数百毫秒），毫秒级沙盒更专注于执行环境隔离和快速销毁。

## 关联概念

- [[宠物与牛马理论]] — 牛马模式要求沙盒可快速销毁重建。
- [[编排者（Coordinator）]] — Coordinator 依赖毫秒级沙盒实现高频调度。
