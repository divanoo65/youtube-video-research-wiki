# 宠物与牛马理论

## 定义
宠物与牛马理论（Pets vs Cattle Theory）源自服务器管理领域，被 Anthropic 引入 Agent 架构。**宠物模式**：Agent 与本地环境深度绑定，需要悉心维护，一旦出错难以复现，类似宠物。**牛马模式**：Agent 被视为无状态的资源，可以随时销毁并重建，类似农场里的牛马，生病就替换。Managed Agents 架构全面采用牛马模式，将 Agent 实体与其运行状态彻底分离。

## 在本库的具体例子
- 在 [[Anthropic公司]] 的 Managed Agents 中，每个 Agent 的“双手”（沙盒）在任务结束后立即销毁；即使中间出错，Coordinator 直接启动一个新沙盒代替。
- 脑图节点显示：“宠物与牛马理论：随时销毁与重建”作为核心理念的子节点。
- 报告中提到：“若某个沙盒运行出错，直接关停并重启干净的沙盒”，这就是牛马模式的具体实践。

## 技术实现细节
- **无状态设计**：Agent 的所有记忆存储在外部的 Session Store（如 Redis）中，沙盒本身不保存任何持久数据。
- **快速重建**：依赖毫秒级沙盒技术（如 [[Cube-Sandbox]]），使得替换 Agent 执行单元的成本极低。
- **健康检查**：Coordinator 定期轮询各 Agent 的执行状态，一旦发现异常立即销毁并重建。

## 与近似概念的边界
- **容器编排（Kubernetes Pods）**：K8s 也提倡 pod 可被杀死重建，但 pod 通常配有 Persistent Volume 保留状态。宠物与牛马理论更进一步，要求完全不保留持久存储。
- **无状态微服务**：微服务无状态，但常需要外部数据库。宠物与牛马理论强调执行环境本身销毁，但记忆通过外部存储恢复，更彻底。

## 关联概念
- [[脑手分离模式]] — 实现牛马模式的前提，通过分离大脑和双手使双手可随意销毁。
- [[Session-Store]]] — 使记忆外置，让 Agent 本体变成可销毁的牛马。
- [[毫秒级沙盒]] — 牛马模式的技术可行性基础。
