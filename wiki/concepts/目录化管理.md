# 目录化管理

**定义**
目录化管理（Catalog Management）是 [[OpenAI]] 提出的一种上下文组织和暴露策略，将工程规范、技能定义、设计规则等文档组织为目录索引结构（而非单一巨大的 `.md` 文件），Agent 在需要特定能力或信息时才动态调取对应的子文档。其核心思想是「按需暴露」——避免将所有内容一次性注入上下文窗口，从而解决上下文窗口稀缺导致的 token 浪费和注意力稀释问题。

**在本库的具体例子**
- 在 [[Harness-Engineering-深度简报-构建稳定-可落地的-AI-Agent系统]] 中，目录化管理被列为 [[OpenAI]] 的一线实践。视频指出，OpenAI 放弃将所有规范塞入一个巨大的 `.md` 文件，而是将其变为 Skills 目录索引，Agent 根据需要动态调取子文档（如设计规范、安全规则等）。
- 在 [[信息边界层]] 的上下文中，目录化管理是信息边界层的一种优化实现——通过结构化组织让 Agent 只获取当前任务相关的上下文片段，避免信息污染。
- 在 [[上下文工程]] 的语境下，目录化管理是分阶段注入上下文的具体手段：先注入全局框架，当 Agent 需要特定细节时再加载子文档。

**技术实现细节**
1. **目录索引结构**：创建一个顶层索引文件，列出所有可用的技能模块或知识子文档及其简要描述、适用场景和访问路径。
2. **按需加载机制**：Agent 根据当前任务的分析结果，通过工具调用（如 `read_document(path)`）从索引中选择性地加载子文档。未加载的文档不占用上下文窗口。
3. **触发条件定义**：为每个子文档定义明确的触发条件（如「当任务涉及数据库设计时，加载 db-schema-rules.md」），由规则引擎或 Agent 自主判断。
4. **目录更新维护**：将目录索引的更新纳入 [[自动化治理]] 系统，确保子文档的描述和路径与实际保持一致。
5. **层级化组织**：目录支持多级嵌套，例如：顶层目录按领域分类（安全、设计、部署），每个领域下再分子模块。

**与近似概念的边界**
- **区别于 [[检索增强生成]]**：RAG 基于语义相似度自动检索文档片段，不确定性较高；目录化管理基于结构化索引和显式规则，Agent 按明确路径加载完整文档，可预测性更强。两者可以互补使用。
- **区别于传统文档管理**：传统文档管理是面向人类阅读的，目录化管理是面向 Agent 消费的——索引的标题和描述需要针对 Agent 的语义理解能力专门优化，而非直接使用人类习惯的文档组织方式。
