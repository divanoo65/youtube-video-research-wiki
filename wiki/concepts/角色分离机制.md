# 角色分离机制

## 定义
角色分离机制是 [[Anthropic公司]] 提出的一种 Agent 内部架构设计：将传统单一 Agent 拆分为三个独立角色——Planner（规划者）负责拆解任务和制定策略，Generator（执行者）按计划生成具体输出，Evaluator（评估者）独立验证输出是否合格。三个角色各有明确的上下文边界和工具集合。

## 在本库的具体例子
- 在 [[Harness-Engineering-技术深度简报-从提示词到系统驾驭的范式转]] 中，Anthropic 明确采用 Planner → Generator → Evaluator 的分离模式，并强调 Evaluator 必须具备独立性，能真实操作页面检查结果，而非仅审阅代码。
- 例如，在代码生成任务中：Planner 写出实现步骤，Generator 编写代码，Evaluator 实际运行代码并审查输出。

## 技术实现细节
1. **Planner（规划者）**：扮演系统分析员角色，它只看到高层目标和可用工具列表，输出是自然语言计划或伪代码计划。Planner 的上下文通常很干净，只包含目标和约束。
2. **Generator（执行者）**：接收 Planner 的计划，调用具体工具（如代码编写、API 调用）产生物理输出。Generator 可能会多次调用同一函数。
3. **Evaluator（评估者）**：完全独立于前两者，拥有自己的工具集（如浏览器、测试框架、日志系统）。Evaluator 不参与生成过程，只对最终产出进行质量检查。

## 与近似概念的边界
- **与单一 Agent 的自反思**：自反思是同一 Agent 在生成后自我检查，但可能陷入自我一致性偏见。角色分离强制使用不同的实例，避免自我欺骗。
- **与多 Agent 辩论**：辩论是两个 Agent 相互反驳，角色分离是流水线式协作，各有独立职责。

## 关联概念
- [[评估与观测层]] — Evaluator 角色是该层的核心组成部分。
- [[执行编排层]] — Planner 的输出驱动编排层的步骤串联。
- [[上下文反射]] — 当上下文接近极限时，可能只重启 Generator 或 Evaluator 角色。
