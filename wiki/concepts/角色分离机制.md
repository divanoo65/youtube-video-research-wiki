# 角色分离机制（Planner/Generator/Evaluator）

## 定义
角色分离机制是 Anthropic 提出的 Agent 架构模式，将复杂任务拆分为三个独立角色：**Planner（规划者）**、**Generator（执行者）** 和 **Evaluator（评估者）**。其中 Evaluator 拥有独立运行环境进行真实操作验证（如代码执行、UI 截图对比），确保生产与验收逻辑分离，降低模型自评偏差。

## 在本库的具体例子

- [[Harness-Engineering-从概率生成走向稳定交付]] 视频中详细介绍了该机制：Planner 负责制定子任务计划，Generator 依次执行，Evaluator 对每个产出的结果进行独立验证。
- 例如在自动代码生成场景：Planner 将需求分解为模块，Generator 编写代码，Evaluator 运行单元测试、检查语法错误，若失败则反馈给 Generator 修正或 Planner 调整策略。

## 技术实现细节

1. **角色隔离**：三个角色使用不同的模型实例（或同一模型不同提示配置），确保评估不依赖生成时的“自我感觉”。
2. **Evaluator 环境独立性**：Evaluator 不会直接在生成环境中运行，而是部署在隔离的沙盒中，防止验证行为污染原始状态。
3. **反馈闭环**：如果 Evaluator 发现结果不满足准入准则（如测试失败、格式不符），则生成修正指令回传给 Generator，并可以设置最大重试次数。
4. **Planner 动态调整**：如果多次重试仍失败，Planner 可能需要重新规划子步骤顺序或降低目标难度。

## 与近似概念的边界

- **与单一 Agent 自评**：单一 Agent 用自身输出作为输入进行自我修正（self-verification），容易出现确认偏差。角色分离机制通过独立实例和环境，显著提高验收可靠性。
- **与多 Agent 辩论**：多 Agent 辩论通常让多个 Agent 互相讨论，角色分离机制更强调角色职责的明确划分和执行-验证分离。
