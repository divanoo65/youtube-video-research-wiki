# 自动化治理

**定义**
自动化治理（Automated Governance）是 [[OpenAI]] 提出的一种系统化规则引擎方法，将资深工程师的经验和工程规范（如模块分层规则、代码风格规范、安全策略）编写为程序化的自动拦截规则。当 Agent 的行为触发违规时，治理系统不仅上报错误，还会将修复建议结构化地反馈给 Agent，驱动下一轮自我修正迭代，从而在无需人工介入的前提下持续约束 Agent 的行为符合组织标准。

**在本库的具体例子**
- 在 [[Harness-Engineering-深度简报-构建稳定-可落地的-AI-Agent系统]] 中，自动化治理被列为 [[OpenAI]] 的核心实践之一。视频指出，OpenAI 的工程师不再仅写代码，而是成为「环境设计师」——将模块分层规则等经验写成系统拦截规则，在 Agent 产生违规行为时自动拦截并给出修复建议。
- 自动化治理与 [[目录化管理]] 协同工作：目录化管理提供结构化的知识索引，自动化治理确保 Agent 在调用这些知识时遵循正确的规范和约束。两者共同构成了 [[信息边界层]] 和 [[约束、校验与恢复层]] 的工程化实现。

**技术实现细节**
1. **规则定义形式**：规则以结构化格式（如 YAML 或 DSL）编写，包含规则名称、触发条件（如正则匹配、AST 模式匹配）、违规等级（ERROR/WARNING/INFO）和修复建议模板。
2. **拦截与反馈机制**：当 Agent 生成代码或执行操作时，治理系统实时扫描输出，检测到违规后：① 拦截输出并生成违规报告；② 返回格式化的修复建议作为 Agent 的下一轮输入；③ Agent 根据建议修正后重新提交。
3. **规则库管理**：规则库按领域分类（如安全规则、架构规则、代码风格），支持动态加载和版本管理。新规则可通过 [[目录化管理]] 的索引自动注册到 Agent 的约束上下文中。
4. **渐进式治理**：治理规则分为硬性规则（必须遵守，违反则直接拦截）和建议性规则（提醒但不强制），Agent 可根据自身权限和任务上下文决定是否遵循建议。
5. **审计日志**：所有违规事件、Agent 的修正行为和最终结果记录到审计日志中，用于后续分析和规则优化。

**与近似概念的边界**
- **区别于 [[约束、校验与恢复层]]**：约束、校验与恢复层是 Harness 架构的整体层级，涵盖所有规则执行和故障恢复；自动化治理是 OpenAI 在约束与校验方面的一种具体实现方案，聚焦于「规则驱动的自修复」。
- **区别于传统 Linter/代码检查**：传统 Linter 只报错不修错，且面向人类开发者；自动化治理不仅报错，还会向 Agent 提供机器可读的修复建议，形成自动化的「报错→修复→验证」闭环。
