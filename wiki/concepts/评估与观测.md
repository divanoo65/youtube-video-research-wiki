# 评估与观测

## 概念定义

评估与观测是 Agent 系统在自动化执行过程中，用于保障输出质量与流程合规性的核心机制。其核心思想是：**避免主 Agent 既当运动员又当裁判，引入一个独立的 Reviewer Agent 对主 Agent 的产出进行逐项、逐条的硬性自检，并遵循严格的合查规则（如逐项核对清单、检查关键字段是否存在、格式是否符合预期等）**。这一机制将人工审核点（Checkpoints）内化为 Agent 自身的可控逻辑，从而在保持全自动运行的同时，将错误发现与修正前置到执行阶段内部，而非事后人工纠错。在[[Harness 实践：Agent 全自动知识讲解视频生成技术简报]]所描述的完整工作流中，评估与观测模块贯穿了内容编写、网页开发、音频生成与最终合成的每一个阶段，确保每个阶段的输出在进入下一阶段前都通过了独立、可重复的验证。这一概念与[[自检修复]]紧密相关，但更侧重于“发现”问题（评估）与“收集”状态（观测），而修复则交由[[约束与恢复]]中的“最小切片修复”策略完成。

## 技术细节

1. **Reviewer Agent 的设计**：   
   - **独立上下文**：Reviewer Agent 拥有独立的上下文窗口，不共享主 Agent 的中间状态，从而避免“认知偏差”——即主 Agent 因沉浸于自身生成过程而忽略某些缺陷。  
   - **检查规则库**：将审核规则以结构化清单（如 JSON 或纯文本清单）的形式注入 Reviewer 的系统提示。规则通常包括：  
     - 格式要求（例如`outline.md`必须包含`## 章节标题`和`- 画面描述`）。  
     - 内容完整性（例如口播稿的每一段文字是否对应大纲中的某一画面）。  
     - 逻辑一致性（例如代码中的函数名是否与大纲描述的功能匹配）。  
   - **逐项合查流程**：Reviewer Agent 接收主 Agent 的输出（如一个`index.html`文件），逐一行扫描文件中包含的每一项产出，并对照规则清单勾选。若某项不符合，则生成结构化错误报告（含位置、类型、建议修复方式）。  
   - **反馈闭环**：错误报告被提交回主 Agent 或专门的修复 Agent（见[[约束与恢复]]），并在修复后由 Reviewer 再次验证，直至所有检查项通过或人工介入。

2. **观测（Observation）的落地**：  
   - **日志与状态快照**：在主 Agent 执行每个阶段（内容编写、网页开发、音频生成、合成）时，系统自动将关键中间产物（如`outline.md`、`index.html`、音频文件列表）的快照写入共享记忆目录（例如`status/`）。  
   - **执行状态看板**：通过[[TMUX]]或自定义 Web 仪表盘，实时展示每个阶段的完成度、Reviewer 的检查结果、失败次数等指标。  
   - **人工干预点**：当 Reviewer 连续两次修复后仍未能通过检查，或检查出无法自动修复的严重错误时（例如底层模型 API 超时），系统将标记该阶段为“需人工确认”，并暂停后续流程。

3. **与上下文管理的关系**：  
   - 为了避免状态膨胀，Reviewer 仅读取当前阶段的产出与上一阶段传递下来的必要上下文（如稿件基调、视觉风格），而不访问主 Agent 的完整思考链。这与[[上下文管理]]中的“物理隔离”原则一致——仅在对应阶段读取，避免注意力稀释。

## 应用场景

- **全自动知识讲解视频生成**：在[[Harness 实践：Agent 全自动知识讲解视频生成技术简报]]中，评估与观测模块被部署在四个阶段之间：第一阶段产出口播稿和大纲后，Reviewer 检查是否已口语化、是否拆分为细化步骤、是否标注了画面建议；第二阶段产出网页后，Reviewer 验证布局是否符合大纲、交互是否正常；第三阶段音频合成后，Reviewer 检查音频时长是否与口播稿匹配、音频文件是否存在；第四阶段合成后，Reviewer 检查字幕同步与输出格式。每一次审核通过后，系统才允许进入下一阶段，否则触发[[自检修复]]流程。

- **代码生成与审查**：在采用[[代理团队]]协作开发复杂项目（如[[网页编码]]）时，可以设立一个专门的 Reviewer Agent 对生成代码进行静态分析、风格检查与功能测试。例如，在[[Harness]]框架中，Reviewer Agent 可以调用外部 CLI（如 ESLint、Pytest）并解析错误输出，与规则清单交叉验证。

- **多模态内容生产**：当系统需同时处理文字、图像、音频时，评估阶段可引入多模态 Reviewer（如调用图像识别 API 检查视觉元素是否与描述一致），确保“所见即所得”。

## 相关概念

- [[自检修复]]：评估与观测的输出驱动修复过程，两者共同构成“检测-修复”闭环。  
- [[人工审核点]]：评估与观测不能替代所有人工审核，当系统无法置信时，仍需触发人工介入点。  
- [[代理团队]]：评估与观测是团队中“质检员”角色的具体实现。  
- [[上下文管理]]：Reviewer Agent 的上下文隔离策略是避免注意力稀释的关键。
