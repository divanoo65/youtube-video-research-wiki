## 概念定义

**策划记忆**（Memory Curation）是 Hermes Agent 闭环学习循环中的第一个核心环节，指智能体在每次对话（或子任务）结束后，针对交互过程中产生的信息进行主动筛选、提炼与结构化存储的操作。它不是粗暴地将所有历史记录原封不动地保留，而是模仿人类“写日记”的方式——只有那些对后续行为有指导意义、能反映用户偏好或可复用的经验才被有选择地记录。策划记忆的质量直接影响后续的自主创建技能、自改进以及用户建模等环节的效果，是整个自进化机制的基础保障。

从认知科学角度看，人类的长期记忆并非全量存储，而是通过海马体对短期记忆进行编码和巩固，将重要信息转入长时记忆。HM 的策划记忆正是对这一过程的工程化模拟：通过预定义的评估框架（如信息的新颖性、与用户偏好的关联度、可复用程度等）让 LLM 自动决策哪些对话片段需要写入持久层，哪些可以丢弃，从而在 Token 预算与记忆容量之间取得平衡。这一机制避免了上下文窗口溢出的同时，也防止了噪声数据污染核心记忆库。

## 技术细节

1. **触发时机**：每次会话结束或用户显式要求保存时触发，位于闭环学习循环顶端。
2. **决策依据**：基于 LLM 对当前对话的总结与评估，结合用户行为信号（如纠正、重复请求、情感表达）判断信息价值。
3. **存储格式**：采用结构化 Markdown 文档，每份记忆包含元数据（时间戳、来源会话 ID）、摘要、关键实体（文件路径、命令、偏好短语）以及关联权重。
4. **存储位置**：写入本地持久化目录下的 `/memory` 文件夹，支持跨设备迁移与 FTS 全文检索。
5. **去重与合并**：若新发现与已有记忆重复或冲突，HM 会先检索历史记忆（通过 FTS 召回），再进行增量合并或覆盖更新，避免冗余。
6. **验证机制**：每次写入前由另一个 LLM 审查（双重校验），确保信息准确且不包含隐私风险。

## 应用场景

- **个人知识管理**：用户与 AI 助手长期对话后，助手自动提取关键项目进展、技术选型偏好，形成可检索的“个人经验库”。
- **企业智能客服**：客服智能体在每次服务后提炼高频问题与解决方案，生成技能模板供后续自动回答使用。
- **开发辅助**：代码助手在修复 Bug 后记录错误模式与修复策略，下次遇到类似错误时直接调用。
- **自适应学习系统**：教育机器人根据学生错题与反馈，策划记忆薄弱知识点，动态调整后续教学策略。
