# 成本拆分 (Cost Breakdown)

## 概念定义
成本拆分（Cost Breakdown）是指在 AI 代理（Agent）运行管理中，通过技术手段对 AI 执行任务所消耗的各类资源（如 Token 数量、API 调用次数、计算时长等）进行细粒度统计、分类与归因的过程。在 Hermes-Agent 智能中控台的语境下，成本拆分不仅是简单的账单汇总，更是一种将“黑盒”运行状态转化为可量化、可审计数据的管理机制。它通过对处理的 Token 总量（如 128 万 token）、技能调用频率（如 80 个技能点）以及不同交互渠道的资源占用进行拆解，帮助用户清晰地识别 AI 代理在哪些环节产生了高昂成本，从而实现从“盲目使用”到“精细化运营”的转变。

## 技术细节
成本拆分功能的实现依赖于中控台对 Agent 运行日志的实时监控与数据聚合：
*   **数据采集层：** 系统持续追踪 Agent 的底层调用，记录每一次交互的输入/输出 Token 消耗、模型推理时长以及外部工具（Tool/Skill）的调用次数。
*   **多维度归因：** 将总成本拆解为不同的维度，例如按任务类型（如自动化任务排程）、按交互渠道（如 CLI vs Telegram）、按技能模块（如特定技能点的调用频率）进行分类。
*   **可视化映射：** 将抽象的计算资源消耗转化为直观的图表或数据看板，使用户能够通过中控台的 Web 界面直接查看各项指标的占比与变化趋势。
*   **动态监控：** 结合“成长曲线”与“物理指标”监控，成本拆分数据会随时间轴更新，支持用户对比不同时间段的资源消耗效率。

## 应用场景
1.  **预算控制与优化：** 当用户发现特定技能或任务类型消耗了不成比例的 Token 时，可以通过成本拆分定位问题，进而优化 Prompt 或调整技能逻辑，降低运行成本。
2.  **性能审计：** 通过分析成本拆分数据，识别出哪些功能模块是“资源黑洞”，从而决定是否需要对该模块进行重构或限制其调用频率。
3.  **多渠道 ROI 分析：** 比较来自不同终端（如 CLI、Telegram）的请求成本，评估不同接入方式的经济效益，辅助决策资源分配策略。
