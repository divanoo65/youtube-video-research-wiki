# 提示词优化

## 定义

提示词优化（Prompt Optimization）是 **AI 模型自身具备的元认知能力**：当用户给出一个模糊或简短的指令时，模型会自动将其扩展、重写或结构化，生成一个更清晰、更具体、甚至包含示例的优化后 Prompt。这个优化后的 Prompt 再被用于实际任务执行。在 Codex Chrome 中，这一能力被用于改善子代理的任务描述，或者直接提升用户请求的完成质量。

## 在本库的具体例子

在 [[sources/Codex-Chrome-插件功能特性与实测深度简报]] 中，演示了 **AI 自动优化用户输入的简单指令**：用户可能只说“整理博客文章”，而 Codex 会将其扩展为“请浏览 Simo 博客 4 月份全部 121 条内容，提取 14 篇正式文章的要（标题、日期、摘要、关键词），输出为 Markdown 表格”。最终系统成功执行了该细化后的任务。

## 技术实现细节

- 模型内部可能采用 **自指令（self-instruction）** 模式：首先生成一个元 Prompt 来描述任务要求，再将其注入到实际执行上下文中。
- 在 Codex 中，优化后的 Prompt 会明确指定**代理数量、数据格式、输出结构**等，比人类手动编写的 prompt 更规整。
- 优化过程本身消耗一定的 token 和推理时间，但能显著提高后续任务的执行速度和正确率。

## 与近似概念的边界

- **人类手动 Prompt 工程**：人类专家编写精细 Prompt 与 AI 自动优化的区别在于——AI 优化能根据上下文动态调整，且能考虑到模型自身的理解偏好。但人类可以对非常特殊的领域进行深度定制。
- **Prompt 模板**：预定义的静态模板不能根据用户输入自适应，而提示词优化是**动态生成**的，属于通用能力。

## 关联概念

- [[子代理协作]] — 主代理为每个子代理分配任务时，使用优化后的提示词确保子代理明确任务目标。
- [[多标签并行]] — 优化后的提示词能明确指定在哪个标签页执行什么操作。
