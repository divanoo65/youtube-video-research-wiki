# 工具系统层

**定义**
工具系统层（Tool System Layer）是 Harness 六层架构的第二层，负责管理 Agent 可调用的外部工具集合。其核心职责包括：为 Agent 选择和暴露合适的工具（避免工具过多导致模型乱用）、定义工具的调用时机（明确何时该调用工具、何时该直接回答），以及对工具的原始返回结果进行精炼提取（而非直接透传原始数据）。

**在本库的具体例子**
- 在 [[Harness-Engineering-深度简报-构建稳定-可落地的-AI-Agent系统]] 中，工具系统层被描述为需要解决「工具太少能力不足，工具太多模型乱用」的矛盾。典型场景：Agent 可调用搜索、数据库、代码执行器等工具，但不应在只需要简单回答时错误调用搜索。
- OpenAI 的 [[目录化管理]] 与工具系统层配合——Agent 按需调取子文档中的工具定义和参数规范，而非将所有工具暴露在上下文窗口内。
- [[Hermes-Agent]] 中的 MCP 调用和 Web Search 辅助任务属于工具系统层的具体实现，通过副模型处理工具调用降低 Token 消耗。

**技术实现细节**
1. **工具注册机制**：每个工具以结构化接口定义（名称、描述、输入参数、输出格式）注册到工具注册表中，Agent 根据语义匹配选择合适的工具。
2. **调用时机决策**：通过规则引擎或分类器判断当前任务是否需要调用工具（例如，当 Agent 知识不足以回答时触发搜索），避免频繁误调。
3. **结果精炼管道**：工具返回的原始数据（如搜索结果的 HTML 页面）经过摘要、过滤、格式转换后，只保留与当前任务相关的关键信息注入上下文。
4. **工具权限分级**：将工具分为只读（如搜索、文档查询）和写操作（如数据库修改、文件写入）两类，写操作需要额外的审批流程（如 [[Hermes-Agent]] 的 Approval 辅助任务）。

**与近似概念的边界**
- **区别于 [[信息边界层]]**：信息边界层决定「什么信息能进入 Agent 的上下文」，工具系统层决定「Agent 能使用什么工具去获取或操作外部系统」。前者是信息筛选，后者是能力暴露。
- **区别于 [[执行编排层]]**：工具系统层定义「有哪些原子工具可用」和「何时调用它们」，执行编排层定义「如何将这些工具调用串联成有逻辑顺序的工作流」。工具是原子能力，编排是流程设计。
