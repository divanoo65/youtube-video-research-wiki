# 工具系统层

## 定义
工具系统层将文本生成模型转变为行动者，通过精心选择可用工具、精确控制调用时机，并对工具返回的原始数据进行提炼和筛选，确保模型始终基于高质量信息做出决策。

## 在本库的具体例子
- 在 [[Harness-Engineering-技术深度简报-从提示词到系统驾驭的范式转]] 中，工具系统层被描述为“解决工具太少能力不足、工具太多模型乱用的平衡问题”。
- [[Scale-AI公司]] 的渐进式暴露实践：当特定 Agent Skill 被触发时，动态加载相关脚本和参数定义，这属于工具系统层的按需暴露策略。

## 技术实现细节
1. **工具选择**：不在所有场景下暴露所有工具。例如，在财务分析场景中，仅暴露“计算器”、“数据库查询”和“报表生成”三个工具，而不暴露“搜索互联网”或“发送邮件”，防止模型误用。
2. **调用时机控制**：明确何时该查资料。例如，在“先思考再行动”范式中，模型必须先生成推理链，确认需要外部信息时才允许调用查询工具。
3. **结果炼制（Result Refinement）**：工具返回的原始数据（如 API 返回值、SQL 结果集）需经过提炼、筛选和格式化后再送还模型。例如，数据库查询返回100行数据，只提取与当前问题最相关的5行并总结后传递给模型。

## 与近似概念的边界
- **与函数调用（Function Calling）**：函数调用是模型 API 特性，工具系统层是对函数调用的治理层，解决选择、时机和结果处理等策略问题。
- **与 RAG**：RAG 是一种检索工具，工具系统层不限于 RAG，还包含所有其他工具（代码执行器、计算器、API 客户端等），并统一管理其调用策略。

## 关联概念
- [[信息边界层]] — 决定哪部分信息可以进入上下文，直接影响工具触发逻辑。
- [[执行编排层]] — 工具调用是编排步骤中的一环，执行编排层负责串联各工具调用顺序。
- [[渐进式暴露]] — 是工具系统层中“工具选择”的动态实现。
