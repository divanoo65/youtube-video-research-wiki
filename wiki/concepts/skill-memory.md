## 概念定义

技能记忆（Skill Memory）是 Hermes Agent 三层记忆体系中的最高层级，专门用于记录“怎么做”的元知识。不同于会话记忆（记录“发生了什么”）和持久记忆（记录“你是谁”），技能记忆的核心价值在于将智能体在任务执行过程中反复验证有效的方法论、操作步骤和决策模式，转化为结构化的 Markdown 文档进行持久化存储。这种设计模仿了人类认知中的程序性记忆——一旦学会某项技能（如骑车、打字），便能无需重复思考即可调用。在 Hermes Agent 中，技能记忆并非静态的指令集，而是“活的”资源：当智能体在后续任务中应用该技能时，系统会通过实际反馈自动评估其效果，并触发针对性的优化与重写。这使得技能记忆能够随环境变化和任务升级而持续进化，避免传统智能体中固定规则库的僵化问题。技能记忆的存储以本地 Markdown 文件的形式存在，支持跨设备迁移和人工审查，同时与工具系统（如 MCP 接入的 6000 余个外部应用）紧密配合，构成智能体自主提升执行效率的核心引擎。

## 技术细节

技能记忆的实现依赖几个关键技术机制：

- **自进化闭环**：智能体在执行任务时，若发现某个操作序列反复成功或失败，会主动将其提炼为技能，写入技能记忆库。后续执行时系统会对比预期结果与真实反馈，若性能下降，则触发自动重写流程。
- **Markdown 结构化**：每条技能以标准 Markdown 格式存储，包含名称、触发条件、步骤列表、预期输出、验证指标等字段。这种人类可读格式便于开发者审查和手动调整。
- **与三层记忆协同**：技能记忆并非孤立运行。它对[[session-memory]]中高频复用的模式进行抽象，并依赖[[persistent-memory]]中的用户偏好和项目背景来调整执行细节。三者构成“经验 → 个性 → 技能”的递进沉淀链路。
- **安全约束集成**：技能执行受[[triple-constraint-mechanism]]保护，包括工具权限控制（限制技能调用的 API 范围）、沙箱运行环境（隔离潜在危险操作）以及 C 调度（对受限工具的执行顺序进行编排）。
- **与工具系统联动**：技能内容常涉及对 40 余种内置工具（如搜索、多媒体处理、协作）以及通过[[model-context-protocol]]接入的外部应用的调用。技能记忆中的指令会动态绑定当前可用的工具实例。

## 应用场景

- **复杂工作流自动化**：用户可让智能体学习“每周自动整理项目周报”的技能，技能记忆会存储获取数据、分析关键指标、生成 Markdown 报告、发送至团队协作平台等完整步骤，避免每次重复配置。
- **个人知识管理**：持续记录用户处理特定类型文档（合同、技术论文）的审阅习惯，形成专属的处理流程，并在后续同类任务中自动启用。
- **企业级经验转移**：新入职的团队成员可通过导入团队积累的技能记忆库，快速掌握标准运维流程、数据清洗规范等，大幅缩短上手周期。
- **智能体自我迭代**：在长时间运行中，技能记忆会自动淘汰低效方案，引入更优策略，例如将某个频繁出错的数据爬取步骤替换为更稳定的 API 调用。
