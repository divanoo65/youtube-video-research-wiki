笔记库是 [[Hermes Agent 高级玩法与优化全指南：云端模型、交互增强与 Token 节省策略]] 在其交互界面 Open WebUI 中提供的一项核心功能，旨在扩展 Agent 的知识边界和信息处理能力。它允许用户将预先整理好的笔记、文档或其他文本资料作为外部知识源，供 Agent 在处理用户请求时进行引用、学习和整合。通过引入笔记库，Agent 不再局限于其训练数据或当前会话内容，能够访问和利用更广泛、更专业的领域知识，从而提供更准确、更深入的回答和解决方案。这极大地提升了 Agent 的实用性和多功能性，使其能够更好地适应各种复杂的任务需求，尤其是在需要特定背景知识支持的场景中。

### 技术细节

笔记库的实现通常涉及将用户提供的文本数据进行索引和向量化处理，构建一个可供检索的知识库。当用户向 Agent 提出问题时，系统会首先在笔记库中检索与问题相关的片段或信息，然后将这些检索到的内容与用户的问题一起作为上下文输入给大型语言模型（LLM）。这种机制类似于检索增强生成（RAG）架构，确保 Agent 在生成回复时能够参考最新的、最相关的外部知识。在 [[Hermes Agent 高级玩法与优化全指南：云端模型、交互增强与 Token 节省策略]] 中，笔记库与 [[引用网页]] 功能并行，共同构成了 Agent 外部信息获取的重要途径，极大地增强了其信息整合和知识应用的能力。

### 应用场景

笔记库功能在多种场景下都具有显著价值：

1.  **专业领域知识问答:** 当 Agent 需要回答特定行业、公司内部规章或项目文档相关的问题时，可以将这些资料导入笔记库，确保回答的准确性和专业性。
2.  **个人知识管理:** 用户可以将自己的学习笔记、研究资料或个人备忘录整理成笔记库，让 Agent 成为一个智能的个人知识助手，随时提供信息检索和总结服务。
3.  **项目协作与信息共享:** 团队可以将项目文档、会议纪要、技术规范等共享到笔记库中，使所有成员都能通过 Agent 快速获取所需信息，提高协作效率。
4.  **内容创作辅助:** 在撰写报告、文章或代码时，Agent 可以通过引用笔记库中的资料，提供相关背景信息、数据支持或代码片段，加速创作过程。
5.  **避免重复提问与信息遗漏:** 结合 [[历史会话]] 功能，笔记库可以帮助 Agent 更好地理解长期上下文，避免重复提供已知信息，并确保关键信息不会被遗漏。
