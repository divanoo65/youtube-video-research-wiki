# 编排者 (Coordinator)

## 概念定义
编排者（Coordinator）是 Managed Agents 体系架构中的顶层指挥官，其核心职能并非直接参与具体的业务执行，而是专注于任务的宏观拆解、逻辑规划与智能体调度。在复杂的 Agent 系统中，编排者充当了“大脑”的角色，它维护着一个包含多达 20 个具备独立 Prompt 和工具集的智能体“花名册”。通过这种架构设计，编排者有效地将决策逻辑与底层执行逻辑分离，实现了“脑手解耦”。它不仅解决了单一智能体在处理长程、复杂任务时因上下文窗口限制而导致的记忆丢失与逻辑混乱问题，还通过动态的任务分发，确保了系统在面对多变需求时具备极高的灵活性与鲁棒性。

## 技术细节
*   **职能定位：** 编排者属于元智能体（Meta-Agent），其核心逻辑在于任务流的编排（Orchestration）。它不直接处理数据，而是根据任务目标生成任务图（Task Graph），并将子任务指派给最合适的执行者。
*   **管理机制：** 编排者维护一个动态的智能体池，每个智能体拥有独立的 Prompt 模板和 MCP（Model Context Protocol）工具集。编排者通过监控任务状态，决定何时调用、挂起或销毁特定的执行智能体。
*   **无状态化协作：** 编排者与执行智能体之间通过无状态的接口进行交互。执行智能体被视为“无状态的独立沙盒”，编排者负责在必要时通过 Session Store 挂载上下文，确保执行过程的连贯性，而无需让底层智能体承担过重的记忆负担。
*   **架构适应性：** 遵循“架构优于补丁”的原则，编排者层级的设计允许底层模型快速迭代。即使更换底层的 LLM 模型，编排者的调度逻辑依然保持稳定，从而避免了因模型升级导致的大量修补代码失效。

## 应用场景
*   **复杂长程任务处理：** 在需要跨越数小时甚至数天、涉及多个子步骤的复杂业务流程中（如自动化软件开发、深度数据分析），编排者负责维持整体目标，防止因上下文压缩导致的逻辑漂移。
*   **多智能体协作系统：** 当系统需要同时调用不同领域的专家智能体（如一个负责代码编写，一个负责测试，一个负责文档撰写）时，编排者作为唯一的接口人，协调各方资源，确保输出的一致性。
*   **高并发任务调度：** 在需要同时处理多个独立会话（Session）的场景下，编排者能够根据当前负载和任务优先级，动态分配计算资源，实现高效的并发处理。
