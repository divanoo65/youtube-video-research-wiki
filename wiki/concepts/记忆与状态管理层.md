# 记忆与状态管理层

## 定义
记忆与状态管理层负责区分并持久化三类数据：当前任务状态（正在执行的步骤）、中间结果（阶段性结论）和长期记忆（用户偏好、历史交互模式），确保 Agent 在多步交互中保持一致的上下文认知。

## 在本库的具体例子
- [[Harness-Engineering-技术深度简报-从提示词到系统驾驭的范式转]] 明确指出，Agent 需要管理“当前任务状态”和“中间结果”，以防止在长对话中遗忘前提。
- [[Anthropic公司]] 的上下文反射机制本质上是状态管理的一种极端形式：当当前 Agent 状态因上下文接近极限不稳定时，将状态快照转移给新 Agent 继续执行。

## 技术实现细节
1. **当前任务状态**：使用 JSON 或数据库记录当前步骤索引、已完成的子任务、等待依赖等信息。例如，在生成报告的任务中，状态中标记“数据收集已完成，等待分析结果”。
2. **中间结果**：每一步的输出作为结构化缓存存储，后续步骤可引用。例如，“分析结果摘要”字段可在报告生成步骤中直接插入。
3. **长期记忆**：用户偏好（如“喜欢简洁报告”）保存在外部向量数据库或键值存储中，在任务开始时加载；历史交互模式可用于个性化响应。

## 与近似概念的边界
- **与对话历史（Conversation History）**：对话历史是简单的消息序列，而状态管理层包含结构化状态（如任务进度、中间结果），并且支持长期记忆和跨会话持久化。
- **与缓存系统**：缓存通常只存储计算结果，而状态管理层存储任务上下文（如当前步骤、已完成依赖），用于指导下一步执行。

## 关联概念
- [[执行编排层]] — 编排层需要读取状态来决定下一步，并更新状态。
- [[信息边界层]] — 状态内容可以影响信息边界层对上下文的裁剪。
- [[上下文反射]] — 是状态管理在 Agent 实例迁移时的特殊应用。
