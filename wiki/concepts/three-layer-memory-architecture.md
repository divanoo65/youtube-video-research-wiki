## 定义

三层记忆架构（Three‑Layer Memory Architecture）是 Hermes Agent 为实现“自进化”而设计的记忆管理系统。传统 AI 智能体通常采用单一记忆池或简单的内存/持久化分离，而 HM 将记忆划分为三个逻辑层次——会话记忆（Conversational Memory）、持久记忆（Persistent Memory）与技能记忆（Skill Memory），分别承担不同的功能角色，并通过一套统一的学习循环协调三者的读写、更新与同步。这种分层设计使得智能体能够在短期交互中保持上下文连贯性，在长期使用中不断积累关于用户偏好、行为习惯的模型化知识，并自动提炼可复用的任务解决模式，从而实现“用中学、学中用”的自我迭代能力。三层记忆架构不仅是数据存储的层次化，更是一种认知架构，模拟了生物从瞬时经验到长期知识再到程序性技能的记忆转化过程，是 HM 区别于配置驱动型 Agent 的关键突破。

## 技术细节

### 1. 会话记忆
- **功能**：保存当前对话或任务执行过程中的上下文，包括用户输入、中间推理步骤、系统输出等。
- **存储方式**：通常使用滑动窗口或基于 token 数的压缩策略，保留最近 N 轮交互，超出阈值后对旧信息进行摘要或丢弃。
- **更新机制**：每次交互结束后自动追加，并触发持久记忆的异步提炼（通过 `策划记忆` 流程）。

### 2. 持久记忆
- **功能**：存储跨会话的长期知识，包括用户个人档案、偏好设置、历史行为统计、重要事实和关系等。
- **存储方式**：采用向量数据库（如 ChromaDB）进行 FTS（全文本搜索）与语义检索结合的索引，每条记录包含内容向量、元数据（时间戳、重要性权重）和原始文本。
- **更新机制**：通过 `闭环学习循环` 中的“策划记忆”阶段，从会话记忆中提取高价值信息，经过去重、抽象后写入持久记忆。同时支持用户显式反馈修正。

### 3. 技能记忆
- **功能**：存储智能体通过学习获得的可复用行为模式、工作流模板、工具调用序列等，相当于“程序性记忆”。
- **存储方式**：以结构化的 `技能` 单元存储，每个技能包含触发条件、执行步骤、所需工具、预期结果和成功/失败案例。
- **更新机制**：智能体在成功完成某一任务后，通过 `自主创建技能` 流程将任务模式抽象为技能；当同类型任务多次失败时，自动从技能记忆中移除或替换。技能记忆支持版本管理，可回退至旧版。

三大层次之间通过 `三重约束机制` 相互连接：会话记忆的内容约束持久记忆的提炼方向，持久记忆的用户模型约束技能记忆的适用场景，技能记忆的执行结果反过来更新会话记忆中的上下文。这种闭环交互使得记忆系统具有自洽性和动态平衡性。

## 应用场景

- **个性化虚拟助手**：借助持久记忆中的用户偏好（如时间管理习惯、饮食禁忌），结合会话记忆中的当前需求，提供高度定制化的建议；技能记忆自动积累高频操作流程（如“订机票+酒店+租车”），逐步实现一键执行。
- **自适应开发者工具**：在编码环境中，会话记忆记录当前项目上下文，持久记忆学习开发者的代码风格和常用库，技能记忆自动生成代码片段模板或 commit 消息规范，并随项目演进自我修正。
- **长期部署的客服机器人**：通过三层记忆架构，机器人能记住用户历史问题、投诉倾向和已尝试的解决方案（持久记忆），同时掌握该行业常见问题解决模式（技能记忆），并在每次对话中动态调整沟通策略。
- **自进化的研究型 Agent**：用于文献综述或实验数据处理，会话记忆维护当前论文阅读链，持久记忆存储已读论文的关键发现与关系图，技能记忆学会特定领域的数据清洗与可视化流水线，大幅减少重复劳动。
