## 定义

推理引擎是运行大语言模型、接收输入并生成输出的软件层。在本地部署场景中，推理引擎负责加载模型权重、执行前向传播、管理显存和提供 API 接口。Llama-cpp 是其中一种基于 C++ 的轻量级实现。

## 在本库的具体例子

- 在 [[零成本本地-AI-Agent-部署与应用简报-Hermes-与-Qwen-3-6]] 中，通过编译 Llama-cpp 并启动服务，为 Qwen 3.6 27B 模型提供兼容 OpenAI API 的接口，供 Hermes Agent 调用。

## 技术实现细节

- Llama-cpp 使用 mmap 加载模型权重，支持 CPU 和 CUDA 推理。
- 通过重新编译启用 CUDA 后端后，推理速度可提升数倍。
- 提供 `/v1/chat/completions` 端点，与 OpenAI 的 API 格式兼容，方便 Agent 框架无缝对接。
- 支持多种量化格式（GGUF），在显存和精度之间取得平衡。

## 与近似概念的边界

- **vLLM**：vLLM 使用 PagedAttention 等技术优化吞吐，适合高并发场景，但对显存要求更高；Llama-cpp 更适合单用户低并发场景。
- **Ollama**：Ollama 是更高级的封装，提供开箱即用的体验；Llama-cpp 是底层引擎，Ollama 也基于它。
