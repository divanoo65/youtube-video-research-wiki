## 定义

五层纵深防御是 Hermes Agent 内建的安全体系，从用户授权、危险命令审批、容器隔离、MCP 凭证过滤到上下文扫描，形成五道防线。该模型旨在使智能体能够在 Telegram/Discord 等公开消息平台中安全运行。

## 在本库的具体例子

- 在 [[Hermes-Agent-与-OpenClaw-深度对比-下一代自进化智能体的崛起]] 中，提到当 Hermes Agent 收到一条删除系统文件的命令时，危险命令审批层会拦截并要求用户二次确认。

## 技术实现细节

1. **用户授权**：只允许预设用户 ID 或群组发送指令。
2. **危险命令审批**：对 `rm`、`sudo` 等高风险操作弹出确认对话框。
3. **容器隔离**：敏感任务在 Docker 容器内运行，隔离系统资源。
4. **MCP 凭证过滤**：在调用外部 MCP 服务时过滤 API Key 等敏感信息。
5. **上下文扫描**：检测提示注入、SSRF 攻击等，并实施黑名单过滤。

## 与近似概念的边界

- **OpenClaw 安全**：OpenClaw 早期版本依赖用户自行配置防火墙，无系统级防御层。
- **沙箱**：沙箱通常只隔离执行环境，五层防御还包括输入输出端的审查。
