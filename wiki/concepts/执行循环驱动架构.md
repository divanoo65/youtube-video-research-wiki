# 执行循环驱动架构

## 定义

执行循环驱动架构是 Hermes Agent 的核心设计哲学——将 AI Agent 自身的**执行循环（Execution Loop）**作为系统的同步编排引擎，而非依赖外部的 Gateway 网关。所有模块（工具运行时、定时任务、通信协议 ACP、SQLite 持久化等）均围绕此循环集成，使系统天生支持自我进化，每一次执行都能反过来优化决策逻辑。

## 在本库的具体例子

在 [[Hermes-Agent-与-OpenClaw-深度对比简报]] 中，Hermes Agent 的执行循环驱动架构与 OpenClaw 的 Gateway 中心化架构形成了鲜明对比。Hermes 的模块围绕闭环集成，支持自我进化逻辑；而 OpenClaw 以 Gateway 网关为控制中枢，负责统一分发与调度。

## 技术实现细节

- Agent 的每次执行循环包含：感知 → 思考 → 行动 → 观察 → 循环
- 执行结果自动记录到 SQLite 持久化存储中
- 循环过程中检测到可复用的工作流模式时，自动触发 skill_manage 创建新技能
- 与通信协议 ACP 深度集成，支持消息平台（Telegram/Discord/CLI）的无缝接入

## 与近似概念的边界

- **执行循环驱动 ≠ Gateway 网关架构**：执行循环将编排逻辑内置于 Agent 自身，而非委托给外部网关。前者更灵活和支持进化，后者更稳定和可审计。
- **执行循环驱动 ≠ 传统编排器**：传统编排器是外部调度系统，而执行循环是 Agent 自身的元认知能力。
