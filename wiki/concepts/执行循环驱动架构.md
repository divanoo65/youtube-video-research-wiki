---\ntitle: 执行循环驱动架构\ntype: concept\ntags: [hermes-agent, 架构]\nsummary: Hermes Agent 的核心架构，将智能体自身的执行循环定义为同步编排引擎，使各模块围绕该循环集成。\nsources: [raw/notebooklm-analysis/NbldZVdusKo_d24ec914-e7e1-4097-b8ee-574598890508_20260509T010454Z_report.md]\ncreated: 2026-05-09\nupdated: 2026-05-09\nlayer: L1\n---\n\n## 定义与核心含义\n执行循环驱动架构是 Hermes Agent 的核心架构理念。与传统的中心化网关控制不同，Hermes Agent 将智能体自身的执行循环 (Execution Loop) 定义为同步编排引擎。所有模块（网关、工具运行时、ACP协议等）均围绕这个执行循环进行集成和通信。\n\n## 应用场景与步骤\n- 在 Hermes Agent 的运行中，执行循环不断地从输入中获取任务，通过工具调用执行，然后根据结果更新内部状态和记忆，进入下一轮循环。\n- 每次循环都可以优化决策逻辑，形成闭环学习和自我改进。\n- 开发者可以通过观察执行循环的日志来理解智能体的决策过程。\n\n## 在本库视频中的具体例子\n- 在视频《智能体架构的范式转移：Hermes Agent 与 OpenClaw 深度技术简报》中，对比了 OpenClaw 的 Gateway 网关中心化控制与 Hermes Agent 的执行循环驱动。\n- 报告指出：网关、工具运行时、ACP协议等所有模块均围绕执行循环集成。\n\n## 关联概念\n- [[ACP协议]]：执行循环中用于模块间通信的协议。\n- [[程序化知识生成]]：执行循环产生的经验可以被抽象为可复用的Skills。\n- [[四层结构化记忆体系]]：执行循环中的状态和经验被存储在结构化记忆中。\n\n## 关联实体\n- [[Hermes Agent]]：采用此架构的智能体框架。\n- [[OpenClaw]]：对比架构，使用中心化Gateway网关。\n\n> 本页面内容基于 raw/notebooklm-analysis/NbldZVdusKo_d24ec914-e7e1-4097-b8ee-574598890508_20260509T010454Z_report.md，仅作 L1 事实层整理。
