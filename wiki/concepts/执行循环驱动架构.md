## 定义与核心含义
执行循环驱动架构是 Hermes Agent 的核心架构理念。与传统的中心化网关控制不同，Hermes Agent 将智能体自身的执行循环 (Execution Loop) 定义为同步编排引擎。所有模块（网关、工具运行时、ACP协议等）均围绕这个执行循环进行集成和通信。

## 应用场景与步骤
- 在 Hermes Agent 的运行中，执行循环不断地从输入中获取任务，通过工具调用执行，然后根据结果更新内部状态和记忆，进入下一轮循环。
- 每次循环都可以优化决策逻辑，形成闭环学习和自我改进。
- 开发者可以通过观察执行循环的日志来理解智能体的决策过程。

## 在本库视频中的具体例子
- 在视频《智能体架构的范式转移：Hermes Agent 与 OpenClaw 深度技术简报》中，对比了 OpenClaw 的 Gateway 网关中心化控制与 Hermes Agent 的执行循环驱动。
- 报告指出：网关、工具运行时、ACP协议等所有模块均围绕执行循环集成。

## 关联概念
- [[ACP协议]]：执行循环中用于模块间通信的协议。
- [[程序化知识生成]]：执行循环产生的经验可以被抽象为可复用的Skills。
- [[四层结构化记忆体系]]：执行循环中的状态和经验被存储在结构化记忆中。
