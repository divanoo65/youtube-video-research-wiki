# 执行循环驱动架构

## 定义

执行循环（Execution Loop）驱动架构是将 Agent 自身的执行闭环作为同步编排引擎的架构模式。不同于传统的中心化 Gateway 模式，它将网关、调度、工具、环境全部集成于执行循环，使每次执行都能反向优化决策，天然支持自我进化。

## 在本库的具体例子

- [[hermes-agent]] 采用执行循环驱动架构，将控制核心从中心化 Gateway 转变为执行循环。在 [[Hermes-Agent-与-OpenClaw-深度对比简报]] 中，这种架构被描述为"重新定义了本地智能体应该如何运行、如何学习、如何成长"。
- 相比之下，[[openclaw]] 采用 Gateway 中心化架构，侧重稳定性和可审计性。两种架构模式决定了不同的运行表现和扩展能力。

## 与其他概念的对比

- **vs 中心化 Gateway 架构：** Gateway 架构以模块化指挥为核心，侧重稳定性与行为追踪；执行循环架构以闭环驱动为核心，天生支持自我进化和程序化知识生成。
- **关联 [[程序化知识生成]]：** 执行循环的每次执行都能产生反馈，这些反馈被 [[程序化知识生成]] 机制捕获并转化为可复用的技能。

## 关联概念

- [[程序化知识生成]] — 在执行循环基础上实现的自动技能生成
- [[分层记忆体系]] — 执行循环依赖的记忆基础设施
