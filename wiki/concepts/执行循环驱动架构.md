# 执行循环驱动架构

## 定义

执行循环驱动架构（Execution Loop Architecture）是 [[hermes-agent]] 的核心设计模式，以智能体自身的执行循环为编排引擎，将网关、调度器、工具调用和环境集成到一个闭环中，天然支持自我进化。

## 在本库的具体例子

- [[hermes-agent]] 采用此架构，与 [[openclaw]] 的中心化 Gateway 模式形成直接对比
- 在 Execution Loop 模式下，每次任务执行同时也是决策逻辑的优化过程
- "执行过程即是决策逻辑的优化过程"——这是执行循环区别于传统 Agent 架构的关键特征
- 该架构为[[程序化知识生成]]提供了运行环境，使技能能自动从执行经验中提取

## 与其他概念的关系

- [[程序化知识生成]] 依赖于执行循环提供的持续优化环境
- [[分层记忆体系]] 是执行循环中记忆模块的具体实现
- 与 [[openclaw]] 的 Gateway 中心化架构形成根本性对比：Gateway 侧重稳定可控，Execution Loop 侧重自我进化
