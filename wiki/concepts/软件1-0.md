软件 1.0 是 Andrej Karpathy 在其关于软件发展阶段的论述中提出的第一个范式，代表了传统的、基于显式规则的编程方法。在这个阶段，软件的开发核心在于编写具体的、指令式的逻辑代码。程序员通过明确的算法、数据结构和控制流来构建程序，每一个功能和行为都需要被精确地定义和实现。这种模式下，软件的运行路径和结果是高度可预测和可控的，但同时也要求开发者对所有可能的场景和细节进行预先编码。例如，一个用于安装复杂软件的 Bash 脚本，其内部包含了大量针对不同系统环境和依赖的显式判断和操作指令，这正是软件 1.0 的典型体现。这种编程方式强调精确性和确定性，是早期计算机科学和软件工程的基础，也是后续 [[软件 2.0]] 和 [[软件 3.0]] 发展的基础。

### 技术细节

软件 1.0 的核心在于其“显式规则”的编程范式。这意味着开发者需要手动编写所有的业务逻辑、控制流和数据处理代码。例如，在传统的 OpenClaw 软件安装过程中，为了适配不同的操作系统和硬件环境，开发者需要编写复杂的 Shell 脚本。这些脚本包含了大量的条件判断（if/else）、循环（for/while）以及系统调用，以确保软件能够在目标环境中正确地编译、配置和运行。这种模式下，程序的行为完全由开发者预先设定的代码决定，不具备自主学习或适应能力。任何功能的变更或新需求的实现，都需要通过修改或添加源代码来完成。这种编程方式的优点在于其可解释性强、调试相对直接，但缺点是开发效率受限于人工编码的复杂度和维护成本。

### 应用场景

软件 1.0 的应用场景广泛存在于各种传统软件开发领域，尤其是在对确定性、可控性和性能有严格要求的系统中：

1.  **系统级编程和脚本：** 如操作系统内核、设备驱动程序、以及各种自动化运维的 Bash 或 Python 脚本。
2.  **企业级应用：** 传统的 ERP、CRM 系统，其业务逻辑通常通过显式编码实现。
3.  **嵌入式系统：** 资源受限且需要高度确定性行为的嵌入式设备固件，例如微控制器编程。
4.  **游戏开发：** 早期游戏引擎和游戏逻辑的实现，通常依赖于大量的显式代码来定义角色行为、物理规则等。
5.  **科学计算：** 各种数值模拟和数据分析工具，其算法和流程都是通过显式代码实现的。

尽管 [[软件 3.0]] 正在兴起，但软件 1.0 的编程范式在许多领域仍然是不可或缺的，并且是理解 [[Andrej Karpathy：从“氛围编程”到“智能体工程”的演进]] 这一演变过程的起点。
