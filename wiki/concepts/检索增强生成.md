# 检索增强生成

**定义**
检索增强生成（Retrieval Augmented Generation, RAG）是 [[上下文工程]] 的核心技术范式，通过在模型生成回答前从外部知识库或搜索引擎中检索与当前查询相关的文档片段，将这些片段作为补充上下文注入模型输入，使模型能够基于最新、最相关的信息生成答案。RAG 解决了模型训练数据截止导致的知识过时问题，以及模型在缺乏足够信息时易产生的幻觉问题。

**在本库的具体例子**
- 在 [[Harness-Engineering-深度简报-构建稳定-可落地的-AI-Agent系统]] 中，RAG 被列为 [[上下文工程]] 的核心技术之一，用于在正确时机向模型送入正确且适量的信息。视频指出，许多 Agent 失败的原因不是模型不够好，而是上下文欠缺或污染——RAG 正是解决上下文欠缺的核心手段。
- OpenAI 的 [[目录化管理]] 与 RAG 理念一脉相承：将大量文档组织为可索引的目录结构，Agent 在需要时通过检索机制动态加载相关子文档，而非将所有知识一次性注入上下文窗口。
- [[Hermes-Agent]] 中的 Web Search 辅助任务本质上是 RAG 的一种实时检索实现——Agent 在需要最新信息时调用搜索引擎，将检索结果压缩后注入上下文。

**技术实现细节**
1. **向量检索**：将文档切分为片段（Chunking），通过嵌入模型（Embedding Model）转为向量存入向量数据库。用户查询也转为向量，通过余弦相似度或内积检索 top-k 最相关片段。
2. **检索增强流程**：用户查询 → 向量检索获取相关文档片段 → 将片段与原始查询拼接为 Prompt → 送入大模型生成最终回答。
3. **高级 RAG 策略**：包括 HyDE（假设文档嵌入，先让模型生成假设答案再检索）、RAG-Fusion（多查询扩展后融合结果）、Self-RAG（让模型自行判断是否需要检索以及检索结果的可靠性）等。
4. **上下文窗口管理**：检索到的片段需要按相关性排序，通过摘要压缩或切片选择控制在上下文窗口限制内，避免信息过载。

**与近似概念的边界**
- **区别于 [[上下文工程]]**：RAG 是上下文工程的技术手段之一，上下文工程的概念更广，还包括上下文压缩、分阶段注入、窗口管理等策略。RAG 聚焦于「从外部知识库检索信息」这一特定路径。
- **区别于传统搜索**：传统搜索只返回文档列表，RAG 将检索结果与生成能力结合——模型「阅读」检索到的内容后生成综合答案，而非仅仅列出结果。
