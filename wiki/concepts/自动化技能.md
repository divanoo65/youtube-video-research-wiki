## 概念定义

自动化技能（Automation Skill）是 [[Hermes Agent 网页版中控台深度分析报告]] 中提出的一种智能效率提升机制，旨在将用户与 AI 交互过程中出现频率较高的重复性 Prompt 自动识别并转化为可永久保存、一键触发的自动化任务。当中控台通过模式识别观察到某个 Prompt 被用户主动提交三次以上时，会在该 Prompt 旁标注一个闪电图标，提示用户可将其生成为一个自动化技能。一旦用户确认，该技能便会以独立条目的形式出现在中控台的技能管理区，支持后续的编辑、启用、禁用和删除操作。

自动化技能的核心价值在于将零散的、重复的 AI 交互行为从手动输入中解放出来，使其变为系统级的背景任务。与传统的 Cron 定时任务不同，自动化技能更侧重于对“行为模式”的固化——它不要求固定的时间触发，而是模拟用户手动发起 Prompt 的动作，可在需要时随时调用，也可挂载到特定的事件（如新文件监听、Git 状态变化等）下自动执行。这本质上是将用户的“经验”以结构化数据存储于本地 state.db 数据库中，随着技能数量的积累，AI 系统对用户需求的响应速度与准确度会逐步提升。

## 技术细节

自动化技能的底层实现依赖于中控台的模式识别引擎。该引擎持续分析用户在 Web Chat 或其他前端界面中输入的自然语言 Prompt 历史记录，通过语义相似度计算和频率统计，筛选出符合“重复三次以上且具有通用价值”的 Prompt 片段。被选中的 Prompt 经过模板化处理（例如将具体文件路径替换为变量占位符），生成一个带有唯一标识符的技能对象。该对象包含技能名称、原始 Prompt、参数列表、执行环境配置（如是否需要上下文窗口）、以及触发条件（手动/事件）。所有技能对象存储在 Hermes Agent 的本地数据库 state.db 中，并通过中控台的 UI 层提供可视化管理界面。当技能被触发时，系统会提取对应的 Prompt 模板并填充当前上下文变量，然后调用 `hermes chat` CLI 指令执行，结果回显到中控台或定向到指定输出通道。

## 应用场景

- **日常开发摘要生成**：用户每天早晨输入“总结昨日 Git 仓库的所有变更并生成 Markdown 报告”，重复三次后系统建议生成自动化技能。后续只需点击技能卡片即可自动拉取最新提交并生成摘要，无需再手动输入长 Prompt。
- **依赖安全检查**：用户定期使用“扫描 package.json 中的已知漏洞” Prompt，系统识别后将其转化为定时触发或手动执行的技能，配合 Cron 任务管理可做到每周自动检查并告警。
- **智能纠错与反馈闭环**：在 Correction Tab（AI 的“错题本”）中，用户可针对同一类错误多次使用类似修正 Prompt，系统从中提炼出标准化“校正技能”，下次遇到同类问题时自动应用修正策略，大幅降低重复劳动。

另外，自动化技能与 [[自动化技能]] 的概念本身互为关联——本页面所定义的技能正是通过中控台“闪电图标”特性从用户行为中提炼而成，可视为对 [[Hermes Agent 网页版中控台深度分析报告]] 中模式识别能力的具体实例化描述。
