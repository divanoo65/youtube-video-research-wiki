# 自驱动治理系统

## 定义
自驱动治理系统是 [[OpenAI公司]] 提出的一种自动化质量控制机制：将资深工程师的模块划分、代码分层等经验性规则形式化为可在 Harness 中编程执行的规则，当 Agent 输出违反规则时，系统自动拦截并同时提供修改方案，形成“检测 → 拦截 → 建议修正 → 重新生成”的治理循环。

## 在本库的具体例子
- 在 [[Harness-Engineering-技术深度简报-从提示词到系统驾驭的范式转]] 中，OpenAI 将模块分层规则写成系统规则，当 Agent 生成的代码试图跨层直接调用时，系统拦截并在日志中提示“请通过 Service 层调用，建议使用 `Service.userService.getUserById()` 替代直接访问 DAO”。

## 技术实现细节
1. **规则编写**：使用 DSL（领域特定语言）或 YAML 定义约束，例如：
   ```yaml
   - rule: "禁止 Controller 直接调用 Repository"
     pattern: "Controller -> Repository"
     suggestion: "请通过 Service 层调用"
   ```
2. **自动拦截**：Harness 在 Agent 输出代码后、交付前，对代码的 AST 做静态分析，匹配规则模式。若匹配则中止交付，并将拦截消息连同建议反写回 Agent 的上下文，要求 Agent 修改。
3. **治理循环**：Agent 收到反馈后修正输出，再次提交，Harness 再次校验，直到通过或达到最大重试次数。

## 与近似概念的边界
- **与 CI/CD 的 Lint**：Lint 只是报告错误，不自纠正；自驱动治理系统会反馈修改建议并触发重生成。
- **与 Human-in-the-loop**：人环需要人类介入，自驱动治理系统完全自动化，适合高频重复场景。
