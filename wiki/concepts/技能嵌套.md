# 技能嵌套 (Skill Nesting)

### 概念定义
技能嵌套（Skill Nesting）是一种在自动化工作流中，将基础的、单一功能的AI指令（原子技能）按照逻辑顺序进行封装与组合，形成更高层级、更具复杂解决能力的复合技能的架构模式。在 Claude Code 与 NotebookLM 的集成场景中，技能嵌套表现为通过 MCP（模型上下文协议）将底层的 API 调用（如上传文档、提取摘要、生成音频概览）封装为可被 Claude Code 直接调用的“研究动作”。这种模式的核心价值在于降低了用户在多工具间切换的认知负荷，使得 AI 能够通过单一的自然语言指令，触发一系列预定义的、具有依赖关系的自动化任务链，从而实现从资料搜集到内容生成的全流程闭环。

### 技术细节
1. **原子化封装**：将 NotebookLM 的各项功能（如 Source 管理、查询、音频生成）抽象为独立的 MCP 工具函数。
2. **层级调用逻辑**：通过 Claude Code 的推理能力，将复杂任务拆解为多个子任务，并按顺序调用已封装的原子技能。
3. **上下文传递**：在嵌套过程中，确保前置技能的输出（如提取的文本摘要）能够无缝作为后置技能的输入（如作为生成音频概览的 Prompt 基础），实现数据流的自动化流转。
4. **错误处理与反馈**：在嵌套层级中加入校验机制，当某一子技能执行失败时，能够通过反馈循环触发重试或人工干预。

### 应用场景
* **自动化研究工作流**：用户只需输入“分析该视频并生成报告”，系统自动嵌套“下载视频”、“提取字幕”、“上传至 NotebookLM”、“执行深度分析”、“生成总结”等一系列技能。
* **跨平台资料整合**：将不同来源（PDF、网页、视频）的资料通过嵌套技能统一导入至 [[NotebookLM]] 的知识库中，并自动触发知识关联分析。
* **定制化内容生产**：通过嵌套“资料溯源”与“内容生成”技能，确保 AI 生成的每一段结论都能精准对应原始资料，提升输出的专业度与可信度。
