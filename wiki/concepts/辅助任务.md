# 辅助任务

## 定义
辅助任务（Auxiliary Tasks）是 [[主副模型配置]] 中由副模型处理的简单任务类型。在 [[hermes-agent]] 中，这些任务包括审批（Approval）、压缩（Compression）、记忆刷新（Memory Refresh）、MCP 调用、会话搜索、技能检索（Skills）、网页/视觉相关任务等。通过将辅助任务分配给廉价模型，可以大幅降低 Token 消耗。

## 在本库的具体例子

- 在 [[hermes-agent-高级玩法与优化指南-深度简报]] 中，视频详细列出了辅助任务的具体类型，并建议将 `approval`、`compression`、`mcp` 等任务的模型 ID 修改为性价比高的模型（如 `gemini-2.0-flash`）。
- 具体配置：进入配置文件，定位到辅助任务（Auxiliary Tasks）部分，将相关任务的模型 ID 修改为 `gemini-2.0-flash`。

## 技术实现细节

1. **任务类型**: 包括审批、压缩、记忆刷新、MCP 调用、会话搜索、技能检索、网页/视觉相关任务。
2. **配置方式**: 在配置文件中定位到辅助任务部分，修改每个任务的模型 ID。
3. **推荐模型**: Google Gemini 2.0 Flash 被推荐为处理辅助任务的副模型。

## 与近似概念的边界

- **主模型任务**: 主模型处理复杂逻辑任务，如推理、规划、代码生成等，与辅助任务的简单性质形成对比。
- **Agent 技能**: Agent 技能是 Agent 自身具备的能力，而辅助任务是运行时动态分配的任务类型。
