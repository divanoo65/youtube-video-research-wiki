# 辅助任务

辅助任务是指在大模型 Agent 系统中，为了优化整体运行成本和响应速度，将那些逻辑简单、重复性强、对模型能力要求不高的工作划分给独立的、性价比更高的模型去完成的一种设计模式。在 [[Hermes Agent 高级玩法与优化指南简报]] 所描述的架构中，辅助任务构成了“主-副模型协作机制”的核心组成部分：主模型负责处理复杂的逻辑推理与最终决策，通常调用性能最强但价格昂贵的模型（如 GPT-4 级别）；而副模型则专门承担批准（Approval）、上下文压缩（Compression）、记忆重刷、MCP 调用、Session 搜索、技能相关操作、视觉解析以及网页工具调用等琐碎但高频的辅助性工作。这种分工不仅能够大幅节省 token 消耗（因为副模型处理的是简单任务，无需使用顶级模型），还能释放主模型的上下文窗口和推理能力，使其专注于真正需要深度思考的环节。视频作者特别推荐将所有的辅助任务统一指派给性价比极高的模型，例如 Google 的 Gemini 1.5 Flash，因为它在保证足够任务质量的同时，成本仅为高端模型的数十分之一。通过编辑配置文件，用户可以为不同的子任务（如 `tasks.compression`、`tasks.memory` 等）分别指定 API Key、Base URL 及模型 ID，从而实现灵活的任务卸载与成本控制。

## 技术细节

- **职能分工**：主模型承担复杂逻辑与最终决策；副模型承担批准、压缩、记忆重刷、MCP 调用、Session 搜索、技能相关、视觉解析及网页工具等简单重复任务。
- **推荐方案**：所有辅助任务均使用性价比极高的模型，如 Google Gemini 1.5 Flash。
- **配置实现**：通过修改配置文件，为不同子任务（例如 `tasks.compression`、`tasks.memory`）分别设置独立的 API Key、Base URL 及模型 ID。
- **优势**：大幅节省 token 费用，降低主模型负载，提升系统整体吞吐量与稳定性。

## 应用场景

- **成本敏感型部署**：个人开发者或中小团队在运行 [[主模型]] 时，希望控制 API 调用成本，将辅助任务分流到便宜模型。
- **高并发聊天机器人**：在需要同时处理大量用户请求的场景下，辅助任务可快速响应简单查询，避免主模型被琐事阻塞。
- **多模态 Agent 系统**：视觉解析、OCR 等任务单独交由辅助模型处理，不影响主模型的推理连续性。
- **记忆与状态管理**：定期重刷、压缩会话历史等维护性工作由副模型后台执行，保持主模型上下文窗口清洁。
