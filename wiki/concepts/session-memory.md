# 会话记忆 (Session Memory)

## 概念定义

会话记忆（Session Memory）是智能体（Agent）系统内部用于记录和管理当前对话过程中所有交互历史的一种结构化记忆机制。它的核心任务不是简单地保存原始对话日志，而是将对话中的关键信息——包括用户输入、系统回复、工具调用结果以及用户隐式表达的偏好——以可检索、可压缩的方式组织起来，使得智能体能够在长周期对话中保持对上下文的敏感度，而不会因Token数量膨胀导致性能下降或模型“忘记”早期内容。与传统的滑动窗口或简单拼接历史文本不同，会话记忆通过引入信息论中的相关性评分和本地数据库的全文本搜索（FTS）技术，实现了按需加载：只有与当前话题紧密相关的历史片段才会被注入到模型的上下文中。这种方法不仅大幅降低了API调用成本（Token消耗可减少30%–60%），还避免了无关干扰，提升了回复精准度。在[[three-layer-memory-architecture]]中，会话记忆被定位为“发生了什么”的实时记录层，与记录“你是谁”的[[persistent-memory]]、记录“怎么做”的[[skill-memory]]形成互补，共同支撑智能体从单次问答进化到具有学习能力的长期交互伙伴。

## 技术细节

会话记忆的实现通常依赖以下几个关键技术组件：

- **FTS全文索引**：利用嵌入式数据库（如SQLite的FTS5扩展）对每条历史消息建立倒排索引，支持按单词、短语或通配符快速匹配。每次新用户输入到来时，系统将该输入分词后作为查询条件，从索引中召回Top-K条最相关历史记录。
- **相关性评分与优先级队列**：召回结果并非简单按时间排序，而是基于TF-IDF、BM25等算法计算每条历史与当前输入的相关性分数，然后选择分数超过阈值的条目放入上下文。同时，系统维护一个优先级队列，确保最重要的交互（如用户明确指定过的偏好、工具执行错误）始终被保留。
- **Token预算控制**：会话记忆模块会预先设定一个上下文窗口的Token上限（例如模型上下文长度的80%），当召回内容的总Token数超过该上限时，自动裁剪低分条目或对长文本进行摘要压缩。
- **事件驱动更新**：每次用户输入或系统输出后，会话记忆自动将新交互写入索引库；如果用户明确要求“忘记之前的某个话题”，则通过删除索引中对应文档的方式高效清理。

## 应用场景

1. **长期编程助手**：在代码审查或逐步调试的对话中，会话记忆能记住用户之前提到的变量命名偏好、测试框架选择等，甚至能回忆起几轮之前的错误日志，避免重复提问。
2. **多轮任务型对话**：例如旅行规划助手，用户可能先讨论目的地，五轮后又回到酒店选择。会话记忆能自动将之前提到的预算限制和日期约束带入当前轮次，无需用户重复说明。
3. **个性化学习辅导**：当学生分多次提问时，系统通过[[user-modeling]]捕捉其薄弱知识点，而会话记忆则负责记录每次辅导的具体讨论内容（如某道题的解题思路），确保后续讲解与历史一致，形成连贯的教学闭环。
