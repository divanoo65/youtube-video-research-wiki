# 用户画像 (User Profile)

### 概念定义
用户画像（User Profile）在 Hermes-Agent 的架构中扮演着“长期记忆库”的角色。与仅记录运行中临时数据的“工作记忆”不同，用户画像旨在捕获并持久化存储用户的偏好、习惯、特定背景信息以及历史交互中沉淀下来的关键特征。通过构建结构化的用户画像，AI 能够超越单次会话的局限，在后续的交互中主动调用这些长期信息，从而提供更具个性化、更符合用户预期且无需重复解释背景的智能服务。它是实现 AI 从“工具”向“助手”进化的关键基础设施，确保了系统对用户认知的一致性与连贯性。

### 技术细节
*   **存储机制：** 用户画像通常作为 Agent 长期记忆的一部分，与 [[工作记忆]] 形成互补。在技术实现上，它可能通过向量数据库或结构化的 JSON/Markdown 文件进行持久化存储，以便于检索增强生成（RAG）流程调用。
*   **交互界面：** 中控台提供了“直接编辑”功能，允许用户无需翻阅底层的本地 Markdown 文件，即可在 UI 界面上直接对记忆条目进行增删改操作。为了防止误操作，系统在删除条目时设计了二次确认机制。
*   **数据来源：** 数据的填充不仅依赖于用户的显式输入，还可能通过系统对交互历史的自动分析（如任务聚类、模式识别）进行动态更新。

### 应用场景
*   **个性化偏好设置：** 用户可以记录自己的编程习惯（如偏好的代码风格）、语言偏好或特定的工作流程，AI 在后续任务中会自动应用这些偏好，无需用户反复提醒。
*   **上下文连续性：** 当用户在不同时间段发起任务时，AI 可以通过读取用户画像，快速恢复之前的讨论背景或项目状态，减少沟通成本。
*   **纠错与优化：** 结合 [[纠错记录]]，用户画像可以记录 AI 曾经犯过的错误类型，从而在未来的任务中主动规避类似的逻辑陷阱或编程遗漏。
