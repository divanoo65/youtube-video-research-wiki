# 评估与观测层

**定义**
评估与观测层（Evaluation & Observation Layer）是 Harness 六层架构的第五层，负责建立独立于 Agent 生成过程的验收与监控机制。它包括环境验证（如代码是否能真实运行）、自动化测试（如单元测试、集成测试）、日志与指标监控（如成功率、延迟、token 消耗），让系统不仅能「做」，还能通过反馈知道自己做对了没有、做得好不好，从而形成自我改进的闭环。

**在本库的具体例子**
- 在 [[Harness-Engineering-深度简报-构建稳定-可落地的-AI-Agent系统]] 中，评估与观测层被描述为「让系统不仅有行动能力，还有自我意识」。视频强调必须建立生产与验收分离的机制，确保评估者具备在真实环境中运行和验证结果的能力。
- [[Anthropic]] 的 [[独立评估者模式]] 是该层的直接实现——Evaluator 角色在隔离环境中运行生成的代码或操作，输出通过/失败结果及具体问题列表。
- [[自动化治理]] 中的规则拦截和审计日志也属于评估与观测层的一部分，但侧重于规则驱动的检测而非独立评估。

**技术实现细节**
1. **环境验证**：在沙箱或隔离环境（Docker 容器、VM）中实际执行 Agent 生成的代码、查询或操作，验证其语义正确性和安全性，而非仅做静态分析。
2. **自动化测试框架**：为 Agent 的输出结果定义可量化的测试标准，包括单元测试（验证单个函数正确性）、集成测试（验证多步骤协作）、端到端测试（验证完整任务）。
3. **指标采集与监控**：实时采集关键指标如任务成功率、平均完成时间、各步骤耗时分布、重试次数、API 成本等，通过仪表盘可视化呈现。
4. **日志追溯**：记录 Agent 每次执行的全链路日志（输入 Prompt、各步输出、工具调用记录、错误信息），支持事后回溯分析和问题定位。
5. **评分与反馈循环**：将评估结果量化为评分（如 0-100），当评分低于阈值时自动触发 [[约束、校验与恢复层]] 的重试或回滚，或触发 [[上下文重映]] 重置 Agent 上下文。
6. **A/B 评估**：在生产环境中并行运行多个 Agent 配置或策略，对比评估指标，选择最优方案持续部署。

**与近似概念的边界**
- **区别于 [[约束、校验与恢复层]]**：评估与观测层关注「如何知道做得对不对」（检测和度量），约束与恢复层关注「做错了怎么办」（纠错和恢复）。两者是检测与执行的关系——评估层发现问题后触发恢复层的机制。
- **区别于传统软件监控（APM）**：传统 APM 监控的是系统级指标（CPU、内存、请求延迟），评估与观测层除了监控这些基础指标外，还需监控 Agent 特有的语义指标（如输出质量、任务完成度、幻觉率）。
