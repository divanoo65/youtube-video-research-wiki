# 评估与观测层

## 定义
评估与观测层是独立于生成过程的验证平面，通过自省（输出是否合规）、工程监控（日志、指标归因）和独立环境验证（真实操作检查）等手段，为 Harness 提供反馈信号，帮助系统自我纠正。

## 在本库的具体例子
- [[Anthropic公司]] 的 Evaluator 角色是本层的具体实现：Evaluator 必须具备独立性，能真实操作页面、检查交互结果，而非仅审阅代码。
- [[OpenAI公司]] 的自驱动治理系统包含日志指标归因和错误拦截，属于工程监控范畴。

## 技术实现细节
1. **自省能力（Introspection）**：在输出前后嵌入评判规则，要求模型自我判断输出是否满足格式、逻辑和约束条件。例如，生成 SQL 时要求模型先评估自己写的 SQL 是否可能包含注入漏洞。
2. **独立环境验证**：构建仿真环境（如 Playwright 浏览器实例）让 Evaluator 实际执行操作，对比预期结果与真实结果。例如，评估一个网页抓取 Agent 时，Evaluator 实际访问页面检查抓取内容。
3. **日志与指标监控**：记录每次 Agent 执行的成功/失败次数、平均步骤数、工具调用频率等，归因到 Harness 各层，定位瓶颈。例如，若工具系统层调用超时率高，则优化工具可用性。

## 与近似概念的边界
- **与单元测试**：单元测试是开发阶段验证，评估观测层是运行时持续验证，且模型输出具有随机性，需要更灵活的评判标准（如 AI-based evaluation）。
- **与日志系统（Logging）**：日志系统只是记录事件，评估观测层在此基础上增加了验证逻辑和自动反馈。

## 关联概念
- [[角色分离机制]] — Evaluator 角色是评估观测层的组织实现。
- [[约束校验与恢复层]] — 评估结果可触发校验和恢复操作。
- [[自驱动治理系统]] — 是评估观测层与约束恢复层结合的高级形式。
