## 概念定义

IP 访问，在信息技术领域通常指通过设备的网络协议地址（IP地址）直接访问网络服务或资源的方式。在 [[Hermes Agent 高级玩法与优化全指南：云端模型、交互增强与 Token 节省策略]] 的语境中，IP 访问特指一种实现移动端交互的手段。当用户希望通过智能手机等移动设备上的 [[手机浏览器]] 访问部署在本地或私有网络中的 Hermes Agent 服务时，可以直接利用服务器的IP地址进行连接。这种方式允许用户在不依赖复杂域名解析或公共网络服务的情况下，直接通过IP地址与Agent进行通信，从而实现跨平台、移动化的操作体验。它为用户提供了一种灵活且直接的访问途径，尤其适用于内部网络环境或对访问控制有特定要求的场景，是实现便捷移动访问的重要选项之一。

## 技术细节

IP 访问的核心在于直接使用服务器的IP地址作为访问入口。对于部署在本地局域网内的 Hermes Agent，用户可以通过在移动设备的 [[手机浏览器]] 中输入服务器的局域网IP地址和对应的端口号来访问。例如，如果 Hermes Agent 服务运行在 IP 地址为 `192.168.1.100` 的服务器上，并且监听 `8000` 端口，用户便可在手机浏览器中输入 `http://192.168.1.100:8000` 进行访问。

然而，如果用户希望在外部网络（如公共Wi-Fi或移动数据网络）访问内部网络中的 Hermes Agent，则通常需要配合其他网络技术。最常见的辅助技术是 [[内网穿透]]。内网穿透技术能够将内部网络的特定端口映射到公网，使得外部用户可以通过公网IP或域名访问到内部服务。在没有内网穿透的情况下，IP 访问通常仅限于同一局域网内，无法直接从外部网络访问。因此，IP 访问与内网穿透常常是实现移动端远程访问的两种互补或替代方案。

## 应用场景

*   **移动端交互:** 用户在离开电脑后，仍能通过手机或平板电脑上的 [[手机浏览器]] 随时随地与 Hermes Agent 进行交互，追溯历史会话、执行任务或查看结果，极大地提升了使用的灵活性和便捷性。
*   **内部测试与演示:** 在企业或团队内部，开发者和测试人员可以方便地通过IP地址在不同设备上访问和测试 Hermes Agent 的功能，无需进行复杂的部署或配置，加快了开发和测试周期。
*   **私有化部署环境:** 对于将 Hermes Agent 部署在私有网络或本地服务器上的用户，IP 访问提供了一种直接且安全的访问方式，避免了数据通过公共云服务传输的潜在风险，确保了数据隐私和安全性。
*   **快速部署与访问:** 在某些场景下，为了快速验证功能或进行临时性使用，直接通过IP地址访问比配置域名等方式更为便捷，省去了域名注册、解析和SSL证书配置等繁琐步骤。
