# 会话存储 (Session Storage)

## 概念定义
在现代多 Agent 协作架构中，会话存储（Session Storage）是指将 Agent 的交互历史、短期记忆、任务状态以及中间变量从 Agent 的执行逻辑中剥离，并持久化存储于外部系统（如数据库或缓存层）的一种机制。在“牛马”范式下，Agent 本身被设计为无状态的执行单元，这意味着 Agent 实例本身不应保留任何长期记忆或环境状态。会话存储充当了 Agent 的“外部大脑”，使得 Agent 在处理任务时，能够通过唯一的会话 ID 随时拉取上下文，并在任务完成后安全地销毁执行环境，而不会丢失任何关键信息。这种解耦不仅解决了传统“宠物”模式下因环境污染导致的系统崩溃问题，还极大地提升了系统的鲁棒性与可扩展性。

## 技术细节
*   **无状态设计原则**：Agent 在执行任务时，仅通过 API 调用获取当前会话的上下文快照，执行完毕后将更新后的状态回写至存储层。
*   **存储介质**：通常采用高性能的键值存储系统（如 [[Redis]]）或分布式数据库，以支持高并发的读写请求，确保 Agent 在毫秒级时间内完成上下文加载。
*   **上下文管理**：会话存储不仅包含原始对话文本，还应包含结构化的任务状态（如 [[任务图]] 的当前进度、已完成的子任务列表、中间计算结果等）。
*   **序列化与反序列化**：为了适配不同类型的 Agent，会话数据通常以 JSON 或 Protobuf 格式进行序列化，确保跨语言、跨环境的兼容性。

## 应用场景
*   **大规模 Agent 集群**：在需要同时运行成百上千个 Agent 的企业级场景中，会话存储允许系统动态调度 Agent 实例，无需绑定特定的计算资源。
*   **故障恢复与重试**：当某个 Agent 实例因沙盒环境冲突或内存溢出而崩溃时，系统可以立即拉起一个新的实例，并从会话存储中恢复之前的任务状态，实现无缝衔接。
*   **多轮复杂任务编排**：在涉及长链路的任务中，会话存储确保了不同阶段的 Agent 能够共享统一的上下文，避免因上下文丢失导致的逻辑断层。
