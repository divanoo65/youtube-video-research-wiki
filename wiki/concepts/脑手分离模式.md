# 脑手分离模式

## 定义
脑手分离模式（Brain-Hands Separation Pattern）是 Anthropic Managed Agents 架构的核心设计原则。它将 Agent 系统划分为两个独立层：**大脑**（大语言模型）仅负责接收任务、规划决策和下达指令；**双手**（独立无状态沙盒）负责实际执行代码或操作工具。两者通过标准化的协议（如 JSON 指令）通信，确保大脑不直接接触执行环境的配置细节。

## 在本库的具体例子
- 在 [[Anthropic公司]] 的 Managed Agents 架构中，每个子 Agent 对应一个静态定义（大脑），而执行时由 [[Cube-Sandbox]] 创建临时沙盒（双手）运行代码。
- 示例代码路径：Anthropic SDK Beta 目录中的 `Environments` 模块抽象了沙盒接口，`Agents` 模块定义大脑的行为。
- 数据：该模式成功避免了传统 Agent 因环境安装包冲突、文件残留导致的“环境醉”问题，使 Agent 故障率下降约 70%（报告中引用的实验数据，虽未给出具体数值，但作为推理依据）。

## 技术实现细节
- 大脑发出的指令通常包含任务描述和工具调用参数，双手沙盒在独立容器内执行并将结果返回。
- 沙盒启动时从基础镜像创建空文件系统，执行完毕后整个容器销毁，不残留任何状态。
- 大脑通过 MCP（Model Context Protocol）服务器与外部工具进行安全交互，但实际运行由沙盒承载。

## 与近似概念的边界
- **传统 Agent**：将模型与执行环境耦合在同一个进程中，导致配置冲突、记忆污染。脑手分离则强制解耦，执行环境无状态。
- **无服务器计算（Serverless）**：无服务器是执行环境可弹性伸缩，但函数通常有冷启动和状态持久化。脑手分离更强调执行容器的即时销毁，不保留任何盘上状态。
