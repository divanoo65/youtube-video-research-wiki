## 概念定义

Managed Agents 架构是 Anthropic 提出的一种企业级人工智能 Agent 设计范式，其核心思想在于将模型的**规划决策能力（大脑）**与**执行代码的沙盒环境（双手）**彻底解耦，并通过引入 Session 与记忆存储的独立管理，构建高扩展性、高鲁棒性的 Agent 操作系统。传统 Agent 开发往往依赖大量补丁代码来弥补模型记忆缺陷，这些代码在模型升级后迅速沦为垃圾。Managed Agents 架构的破局点在于：允许模型专注于指令发出，而底层的执行环境则是无状态、可随时销毁并重建的独立容器。这种从“养宠物”（手写复杂逻辑维护 Agent）到“用牛马”（将 Agent 视为可动态调度的容器集群）的范式转变，使得计算资源分配向动态、高并发的 Agent 容器化协作方向演进。该架构不仅解耦了“脑”与“手”，还进一步拆解了会话层、代理层、记忆层与执行层，形成四层解耦结构，从而支持多 Agent 蜂群协作、毫秒级沙盒创建、以及前线部署工程师的实时介入。Managed Agents 架构代表了当前 Agent 发展的标准答案——**脑手解耦**与**无状态化**，是构建复杂、可扩展、可维护的 AI 系统的关键基础设施。

## 技术细节

Managed Agents 架构的技术实现包含以下关键组件：

- **大脑（Brain）**：由大语言模型（如 [[Claude]]）负责高层规划、任务拆分与决策。模型通过分析用户意图，生成一系列原子操作指令，并传递给执行层。
- **双手（Hands）**：一个轻量级、无状态的沙盒执行环境，如 [[Cube Sandbox]]。该环境接收大脑发出的代码片段，立即执行并返回结果，执行完毕后可被销毁，不保留任何状态。
- **会话与代理分离**：将 Session（用户会话上下文）与 Agent（执行逻辑的实例）彻底分离。Session 负责持久化对话历史、用户偏好和中间结果，而 Agent 实例仅在执行期间存在，完成后释放资源。这实现了无状态化，允许水平扩展和容错。
- **记忆存储独立管理**：记忆（Memory）不再混入 Agent 实例，而是作为独立的外部存储系统（如向量数据库或键值存储）。Agent 在启动时动态加载相关记忆，执行后更新，从而避免状态膨胀。
- **毫秒级沙盒**：通过容器化技术（如 Docker + 预热镜像）实现 50-100ms 内创建和销毁沙盒，使得高频、并行任务成为可能。
- **协调者模式**：引入一个中央协调器（Orchestrator）负责接收大脑的任务，调度多个沙盒实例并行执行，并汇总结果。协调器也负责错误重试、超时管理和资源池动态扩容。

## 应用场景

- **复杂长程任务分解**：例如撰写一份包含多章节、多数据源的研究报告。大脑将报告拆分为大纲、各章节初稿、数据查询、图表生成等子任务，交给多个沙盒并行执行，最终由协调者合并输出。[[脑手解耦]]使得模型无需记忆中间状态，只需专注推理。
- **企业级自动化工作流**：在 [[腾讯]] 等公司的内部基建中，Agent 需要频繁调用外部 API（如数据库查询、发送邮件、操作工单系统）。Managed Agents 架构允许每个 API 调用在独立沙盒中执行，避免污染主 Agent 状态，同时支持回滚与审计。
- **多 Agent 蜂群协作**：多个大脑（不同模型或同一模型的不同实例）基于共享的记忆池和任务队列协同工作，例如金融领域的实时风险监控，其中一个 Agent 负责分析新闻，另一个负责交易信号，第三个负责风控规则校验。[[无状态化]]保证了各 Agent 互不干扰且能水平扩展。
- **前线部署工程师的实时介入**：当沙盒执行出现异常或需要人工决策时，系统可暂停任务，将控制权临时交给前线工程师，工程师在沙盒内直接调试或修改代码，完成后继续执行。这种设计降低了自动化系统的风险敞口。
