## 概念定义

托管代理架构（Managed Agent Architecture）是 Anthropic 提出的一种面向 AI Agent 的系统设计范式，其核心哲学在于通过“脑手分离”与“四层解耦”，彻底改变传统 Agent 开发中模型、记忆与本地环境深度耦合的“宠物模式”。在该架构下，Agent 被拆解为两个独立层次：大脑层负责规划决策与指令下发，执行层则由一组无状态、可随时销毁重建的沙盒（Sandbox）组成，每个沙盒作为“双手”仅负责具体任务的原子执行。大脑与执行环境之间通过标准化的任务图（Task Graph）通信，所有状态信息（如对话历史、中间结果）被外部存储（如会话存储层）统一管理，从而实现 Agent 的全生命周期可观测、可恢复与水平扩展。这种设计使得 Agent 不再是环境绑定的脆弱单体，而像“牛马”一样，每时每刻都可以在干净的环境中从头开始执行任务，极大提升了系统的鲁棒性、资源利用率和运维便利性。托管代理架构被视为 AI Agent 从原型走向大规模生产部署的关键基础设施，也是“蜂群模式”和“编排者模式”的基础支撑。

## 技术细节

托管代理架构的技术实现依赖于多层解耦设计：

- **大脑与双手的协议层**：大脑（通常由大语言模型驱动）生成高层次任务指令，通过结构化的“动作原语”（如 `run_command`、`write_file`、`read_file`）传递给执行沙盒。沙盒运行在隔离的容器（如 [[Docker]] 容器）或轻量级虚拟机中，每次任务执行后即被销毁。
- **无状态化执行**：沙盒内部不持久化任何状态，所有环境变量、文件系统、网络连接在每次启动时从零构建。这消除了长期运行导致的配置漂移和缓存污染问题。
- **记忆树状管理**：大脑的推理上下文通过外部存储（如向量数据库或键值存储）以树形结构组织，允许回溯、分支与合并。每次规划时，大脑从记忆树中加载必要的上下文片段，而不是依赖连续对话窗口。
- **四层解耦架构**：由编排层（Orchestrator）、大脑层（Reasoner）、执行层（Sandbox）和存储层（Store）构成。编排层负责任务拆分与调度，大脑层负责逻辑推理，执行层负责具体操作，存储层负责持久化与共享状态。
- **错误容错策略**：当执行沙盒发生崩溃、超时或逻辑错误时，编排层自动销毁当前沙盒并重新创建一个干净沙盒，同时通知大脑调整下一步计划。这种“熔断-重建”机制保证了系统在不可预测环境下的稳定性。

## 应用场景

托管代理架构适用于需要高可靠、长周期、多步骤自动化的复杂场景：

- **企业级自动化流程**：如 ERP 系统中的审批流、数据迁移、报告生成，Agent 可以独立运行数小时甚至数天，期间遇到任何环境问题都能自动恢复。
- **软件测试与 CI/CD**：Agent 在隔离沙盒中执行代码编译、单元测试、集成测试及其修复，每阶段使用独立环境，避免测试数据污染。
- **安全审计与分析**：Agent 在临时沙盒中分析可疑文件、执行渗透测试或复现漏洞，完成后销毁环境，不留痕迹。
- **多 Agent 协同（蜂群模式）**：多个托管 Agent 并行处理大型任务的各个子模块，由编排者统一调度，整体架构具备弹性伸缩能力。例如在 [[Palantir AIP]] 的军事分析场景中，数百个 Agent 同时分析卫星图像、文本报告和信号情报，每个 Agent 仅在需要时创建并执行单一动作后即被回收。
- **云端 Agent 操作系统**：作为未来 FDE（Full Desktop Environment）的底层支撑，托管代理架构将计算与决策彻底分离，使得 Agent 可以随时在任何云端节点上重建自己的“工作环境”，真正实现“Agent 即基础设施”。
