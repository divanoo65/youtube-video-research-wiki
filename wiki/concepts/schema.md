# Schema

## 概念定义

Schema（规则层）是 LLM-Wiki 三层架构中的最高层级，位于原始数据层（Raw Data）和 Wiki 产物层（Wiki Assets）之上。它本质上是一组由用户预先定义或动态调整的“说明书”，规定了大型语言模型（LLM）在知识处理各阶段应遵循的解读逻辑、分类标准、输出格式以及关联规则。与传统的固定模板不同，Schema 是一套可编程的元规则，它允许用户无需修改模型本身，就能让通用大模型表现出特定业务领域的专业行为。

在 LLM-Wiki 中，Schema 承担着“翻译器”和“过滤器”的双重角色。当用户向系统投喂一篇论文或笔记时，大模型不再仅凭自身预训练知识进行通用摘要，而是首先检索当前 Wiki 中已存在的 Schema 配置，了解该领域特有的概念层级（例如“核心概念”与“衍生概念”的区分）、信息提取优先级（如实验方法优先于背景介绍）以及术语映射（如将“ML”自动统一为“机器学习”）。这种机制确保了知识产出的专业一致性，避免了因模型语料库偏差导致的概念混沌。

此外，Schema 还支持版本化管理。随着用户对领域理解的深化，可以迭代更新规则文件，并通过 LLM-Wiki 的健康检查（Health Check）模块自动校验新旧内容是否符合最新规则。这种设计使知识库具备了自适应的进化能力，真正实现了从“存储知识的容器”到“理解知识的助手”的质变。

## 技术细节

Schema 在实现上采用纯文本的规则描述格式（通常为 YAML 或 JSON），存储于 LLM-Wiki 的指定目录（如 `schemas/`）中。每个规则文件包含以下核心模块：

- **实体识别规则**：定义哪些类型的术语应被识别为实体并创建独立页面（例如 `type: concept | tool | person`），以及实体之间的默认关系（如子类、引用、冲突）。
- **分类策略**：指定大模型在摄入内容时应使用的分类标签体系，包括标签的层级结构和互斥规则（例如一篇文章不能同时属于“理论”和“实践”两个类别）。
- **输出模板**：规定摘要、概念页、双向链接等生成物的格式要求，如摘要长度范围、必须包含的字段（作者、发表年份）、链接格式（`wikilink` 或 `[markdown]`）。
- **断言与约束**：允许用户定义简单的逻辑断言，例如“如果某篇论文的发表日期晚于 2023 年，则自动添加 #前沿 标签”，用于自动化标记。

在运行时，LLM-Wiki 的摄入（Ingestion）流程会首先加载当前项目下的所有 Schema 文件，构建一个内部规则引擎。大模型在解析新素材时，系统会将该规则引擎的上下文注入到提示词（Prompt）中，确保输出严格遵循 Schema。由于规则是结构化文本，而非嵌入模型参数，用户可以在不重训模型的情况下，通过编辑文件快速调整知识库的行为。

## 应用场景

1. **学术研究团队**：通过 Schema 定义本领域特有的概念分类（如“算法”、“数据集”、“评估指标”），使 LLM 在阅读新论文时自动提取实验设置和结果表格，并统一链接到已有的 概念页面，形成领域知识图谱。
2. **企业知识管理**：为不同部门定制 Schema，规定技术文档必须包含“决策背景”、“实施步骤”、“风险说明”等章节，确保知识库输出的结构一致，便于后续的 RAG 查询。
3. **个人学习系统**：用户可将自己的学习阶段（如“入门”、“进阶”、“实战”）写进 Schema，让 LLM 根据当前阶段调整内容摘要的难度和重点，避免信息过载。
