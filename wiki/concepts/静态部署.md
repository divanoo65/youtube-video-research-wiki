## 静态部署

**静态部署**是指将由纯 HTML、CSS、JavaScript 以及图片、字体等资源组成的无需服务器端动态处理的网页或应用程序，托管到 Web 服务器上，使用户能够通过稳定的 URL 直接访问。在 [[Hugging Face Spaces 永久免费云服务器使用与 AI 项目部署指南]] 的语境下，静态部署特指利用 Hugging Face Spaces 平台的 **Static Spaces** 功能，将基于 Gradio、Streamlit 或纯前端的演示项目以静态资源的形式发布到免费云服务器上。这种部署方式不涉及后端运行时（如 Python 或 Node.js），因此资源消耗极低、访问速度更快，且完全符合 Hugging Face 永久免费套餐的使用限制。

静态部署的核心优势在于**零维护成本**和**高可用性**。开发者只需将编译后的静态文件（如 HTML 文件夹、单页应用构建产物等）上传至 Spaces 仓库，平台会自动提供 CDN 加速与 HTTPS 证书，同时保证 100% 的在线时间。对于 AI 项目而言，许多模型的推理前端（例如基于 Transformers.js 的浏览器端推理）完全可以在客户端运行，后端仅需提供静态资源托管，这种情况下静态部署是最优选择。

### 技术细节

Hugging Face Spaces 的静态部署支持两种模式：

1. **标准静态空间**：仓库根目录包含 `index.html` 或其他静态文件，系统自动识别并启用静态服务器。用户可通过 `https://huggingface.co/spaces/{用户名}/{空间名}` 访问。
2. **自定义构建输出**：若项目使用构建工具（如 Vite、Webpack），可以在空间配置中指定 `build` 命令和输出目录，Spaces 会自动执行构建并将结果部署为静态站点。

静态空间不提供服务器端环境变量或数据库，所有动态功能必须通过浏览器端的 JavaScript 调用外部 API（如 Hugging Face Inference API）实现。此外，静态空间支持自定义域名、重写规则（.htaccess 等效配置）和 gzip 压缩，但禁止使用 `app.py` 或 `Dockerfile` 等动态启动文件。

### 应用场景

- **AI 模型演示页面**：将训练好的模型封装为 Gradio 或 Streamlit 应用后，若仅需展示界面而无需后端推理，可将前端构建产物静态部署，同时通过 JavaScript 调用远程推理 API。
- **个人作品集 / 技术博客**：利用静态空间托管纯静态网站，结合 Markdown 或 Hugo 等静态生成器，实现零成本的个人官网或项目文档。
- **浏览器端 AI 应用**：如使用 [[Transformer.js]] 在浏览器中运行 NLP 或图像模型，整个应用无需任何后端服务，直接静态部署即可。这种模式与 [[一键抠图]] 等轻量级前端工具结合，可以快速打造免安装的在线工具。

静态部署是拥抱云原生、降低运维成本的重要实践，特别适合个人开发者和小型团队快速验证想法、分享成果。
