# 自动化研究工作流

### 概念定义
自动化研究工作流是指通过集成先进的 AI 代理工具（如 Claude Code）与知识管理平台（如 NotebookLM），将传统研究过程中繁琐的手动操作（如资料上传、格式转换、跨平台数据迁移及手动提问）转化为由自然语言驱动的自动化执行序列。该工作流的核心逻辑在于利用模型上下文协议（MCP）打破不同 AI 工具间的壁垒，使研究者能够通过单一指令触发跨平台的知识处理任务。这种工作流不仅极大地降低了研究人员在工具切换上的认知负荷，还通过标准化的自动化脚本确保了研究过程的连贯性与可复现性。它将 AI 从单纯的“问答助手”提升为“研究执行代理”，能够自主完成从多源数据采集到深度知识汇总的全链路任务，同时严格遵循 NotebookLM 基于特定源资料回答的“忠实度”原则，确保研究结论的准确性与可溯源性。

### 技术细节
1. **协议层（MCP）**：利用模型上下文协议（Model Context Protocol）作为底层连接器，实现 Claude Code 对 NotebookLM 内部功能的 API 调用与指令下发。
2. **指令驱动**：通过自然语言编写的自动化脚本，替代传统的 GUI 点击交互，实现对 YouTube 视频、PDF 文档及网页内容的批量处理。
3. **上下文管理**：利用 Claude Code 的代码执行能力，将外部数据源预处理后注入 NotebookLM 的知识库，确保模型在处理复杂研究任务时具备充足的上下文支撑。
4. **忠实度保障**：工作流设计中强制执行“源资料绑定”，确保所有生成的结论均能通过 [[资料溯源]] 机制追溯至原始输入，防止 AI 幻觉。

### 应用场景
* **学术文献综述**：自动抓取指定领域的 YouTube 讲座视频与 PDF 论文，通过工作流自动汇总核心观点并生成对比分析报告。
* **市场情报监控**：定期自动收集行业网页资料，利用 NotebookLM 的分析能力生成周报，并由 Claude Code 自动整理格式并归档。
* **深度知识库构建**：将零散的个人笔记与在线资源自动同步至 NotebookLM，构建私有的、可实时交互的知识资产库。
