## 概念定义

沙盒环境（Sandbox Environment）在智能体（Agent）架构中指一个完全隔离、无状态、可随时销毁与重建的**代码执行运行时**。它是“脑手解耦”模式中“双手”的具体体现：大模型负责规划决策（大脑），沙盒环境负责执行具体的代码调用、工具操作和外部交互（双手）。与传统将模型记忆与本地环境深度绑定的“宠物模式”不同，沙盒环境遵循“牛马模式”哲学——Agent被视为无状态的“牛马”，每个沙盒实例都是独立、封闭且短暂的。一旦沙盒内发生污染、配置错误或执行失败，系统不会试图修复它，而是直接销毁该沙盒并立即拉起一个全新的干净环境。这种设计彻底消除了环境脆弱性，避免了模型进化速度远超手动修复代码速度的痛点，使得Agent能够在大规模集群中实现毫秒级弹性扩缩容。沙盒环境通常由容器化技术（如Docker、Firecracker）实现，通过轻量级虚拟化提供底层资源隔离，同时借助标准化文件系统镜像（如分层存储、快照技术）确保每次启动时环境的一致性。此外，沙盒对外暴露统一的API接口（如文件读写、网络请求、子进程调用），大模型通过标准化的工具调用（如Function Calling）间接控制沙盒行为，从而将规划逻辑与执行细节彻底解耦。沙盒环境内无任何持久化状态，所有会话数据（如上下文记忆）由外部存储（如Redis、数据库）统一管理，沙盒仅作为瞬态计算单元存在。这一模式不仅大幅降低了运维复杂度，还实现了资源复用最大化：单台物理机可以并发运行数千个沙盒实例，每个实例仅占用数十兆内存，执行完任务后立即回收。

## 技术细节

### 1. 隔离机制
- **进程级隔离**：使用 Linux Namespace（PID、Network、Mount、UTS）和 Cgroups 限制 CPU、内存、I/O 资源。
- **文件系统隔离**：基于 OverlayFS 创建读写层，基础镜像为只读，每次写入均在上层进行，销毁时直接丢弃上层。
- **网络隔离**：每个沙盒分配独立虚拟网卡（veth pair），通过 iptables/nftables 策略严格限制出站/入站流量。
- **超时与安全策略**：设置最大执行时长（如 30 秒），超出则强制终止；禁止访问宿主机敏感目录（如 `/proc`、`/sys`）。

### 2. 无状态设计
- **环境快照**：沙盒启动时从预配置的 OCI 镜像（如 Docker Image）加载初始状态，执行过程中不保留任何用户数据。
- **外部状态存储**：对话历史、Agent 记忆、工具调用结果统一写入外部键值存储（如 Redis），沙盒仅通过 API 读取/写入。
- **销毁与重建**：沙盒完成后立即终止并释放资源；新任务到来时从镜像池中快速克隆（Copy-on-Write 技术，冷启动 < 100ms）。

### 3. 编排与调度
- **沙盒池化**：维持一批预热沙盒（已启动但未分配任务），任务到达时直接绑定，减少冷启动延迟。
- **健康检查**：定期向沙盒发送心跳探测，异常实例自动回收并补充。
- **任务图协同**：在 [[四层解耦架构]] 中，沙盒作为执行层节点与 [[协调者]] 通信，协调者根据 [[任务图]] 依赖关系并行分发子任务到多个沙盒实例。

## 应用场景

- **大规模 Agent 编排**：在蜂群架构中，每个 Agent 对应多个沙盒，同时执行不同子任务（如数据分析、API 调用、文档生成），实现任务并行与负载均衡。
- **多租户安全执行**：SaaS 平台为不同用户或客户提供隔离的 Agent 执行环境，防止恶意代码跨租户传播。
- **模型迭代兼容**：当基础模型升级（如 [[Claude]] 3.5 → 4.0）时，无需重写沙盒内补丁代码，仅需更新系统提示词和工具定义即可适配新模型。
- **自动化测试与沙箱**：在 CI/CD 流水线中使用沙盒运行 Agent 生成的代码脚本，验证正确性后自动部署到生产环境。
- **科学计算与数据处理**：沙盒内可运行 Python/R 脚本，配合外部存储（如 S3）读写大规模数据集，任务完成后环境立即清除，避免数据残留。
