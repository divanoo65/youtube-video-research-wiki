## 概念定义

主副模型策略是针对大型语言模型（LLM）Agent 在运行过程中 Token 消耗过快问题而提出的一种任务分级处理方案。其核心思想是将一个原本由单一高性能大模型全权处理的 Agent 工作流，拆解为两个或多个角色不同的模型共同协作：由参数规模更大、推理能力更强但价格昂贵的高性能模型担任“主模型”，专门负责处理复杂的逻辑推理、决策制定和核心任务执行；同时，引入一个或多个参数较小、推理速度更快且成本低廉的模型担任“副模型”，用于承担那些对智能要求不高但频繁出现的辅助性工作。这些辅助任务包括任务执行前的批准确认（Approval）、对话上下文的压缩与总结（Compression）、长期记忆的提取与维护（Refresher）、外部工具或插件的调用（MCP/Skills），以及图像信息或网页内容的处理（Vision/Web）。通过这种分工，可以避免高性能模型被低价值任务浪费算力，在保证整体系统智能水平的前提下，大幅降低 Token 开销，从而延长单次会话时长、提高 Agent 的响应速度，并降低运营成本。该策略尤其适用于需要长时间运行、多轮对话或频繁调用外部工具的 Agent 场景，是构建高效、可扩展的 AI 系统的重要方法论。

## 技术细节

主副模型策略的实现依赖于对 Agent 工作流的精细化拆解和模型路由机制。具体技术要点如下：

1.  **任务分级与路由**：在 Agent 的调度层预先定义一套规则，将用户的请求或系统内部事件按复杂度分类。主模型负责处理需要深度推理、多步规划或创造性输出的任务；副模型则处理那些规则明确、逻辑简单或仅需关键词匹配的任务。
2.  **副模型任务清单**：根据常见场景，副模型可以承接以下五类典型任务：
    - **批准 (Approval)**：在执行关键操作（如发送邮件、修改文件）前，由副模型快速判断是否符合预设条件，降低主模型被误唤的概率。
    - **压缩 (Compression)**：在对话上下文接近 Token 上限时，由副模型对历史记录进行归纳摘要，保留核心信息，减少主模型的输入长度。
    - **记忆刷新 (Refresher)**：定期从长期记忆库中提取相关信息并刷新到当前上下文，保证 Agent 对用户偏好和历史状态的感知。
    - **工具调用 (MCP/Skills)**：通过 MCP（Model Context Protocol）调用外部技能或插件时，由副模型解析参数、校验格式，减轻主模型负担。
    - **视觉/网页 (Vision/Web)**：对传入的图片、截图或网页内容进行初步解析（如 OCR、摘要），将结构化结果交给主模型进一步处理。
3.  **模型选择建议**：教程中推荐以 Gemini 1.5 Flash 作为副模型的典型实例，因其具有低延迟、高吞吐量和极低 Token 成本；主模型则可根据任务复杂度选用 Gemini 1.5 Pro、GPT-4 或 Claude 3 等旗舰模型。
4.  **工作流编排**：Agent 框架（如 Hermes Agent）需内置针对不同任务的调用链，支持动态切换主副模型，并可配置优先级、超时重试和 Fallback 机制。

## 应用场景

- **智能客服系统**：在对话早期，用副模型处理用户身份验证、常见 FAQ 匹配；当问题升级为复杂投诉或需要多步操作时，将对话转交主模型。可显著降低常规咨询的推理成本。
- **代码开发助手**：主模型负责代码生成与调试，副模型负责代码格式检查、文档检索、编译错误预检等非推理密集型任务，提升整体响应速度。
- **多工具协同 Agent**：在需要频繁调用搜索引擎、数据库查询、文件读写等外部工具的 Agent 中，副模型预处理查询语句、过滤无效结果，主模型仅对最终结果进行综合判断，极大减少主模型的 Token 消耗。
- **长期记忆维护**：在持续性对话系统（如虚拟伴侣、个人知识管理助手）中，副模型定时压缩旧对话、刷新记忆摘要，使主模型每次接收的都是精简后的高价值上下文，从而支持超长会话。
