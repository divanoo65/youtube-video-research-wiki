# 提示工程

**定义**
提示工程（Prompt Engineering）是 AI 工程化的第一阶段，核心是设计符合模型理解的指令文本（包括角色设定、任务描述、输出格式、示例等），以塑造局部概率空间，使模型更准确地回答问题或执行任务。其本质是“语言设计”。

**在本库的具体例子**
- 在 [[Harness-Engineering-深度简报-构建稳定-可落地的-AI-Agent系统]] 中，提示工程被描述为「告诉新入职员工任务流程」，即通过一段 Prompt 告知 Agent 要执行的步骤，例如“先搜索资料，再总结，最后输出 Markdown 格式”。但这在长链路任务中容易失效，因此需要更高级的 Harness 工程。
- 报告中指出，当任务成功率停滞时，不应仅仅修改 Prompt，而应检查执行编排和故障恢复机制。许多早期 Agent 系统仅依赖长 Prompt 就尝试完成多步任务，结果因上下文过载而失败。

**技术实现细节**
1. **模板化**：将 Prompt 分为系统角色、任务描述、上下文输入、输出格式等固定部分，动态填入变量。
2. **少样本示例**：在 Prompt 中插入输入-输出对，帮助模型理解期望的推理模式。
3. **思维链（Chain-of-Thought）**：引导模型逐步推理，分解复杂问题。

**与近似概念的边界**
- **对比 [[上下文工程]]**：提示工程只优化指令本身，不涉及外部信息的动态注入；上下文工程则关注在何时给模型送入哪些外部信息（如数据库查询结果、网页内容）。提示工程是上下文工程的基础组成部分。
- **对比 [[约束工程]]**：提示工程停留在“告诉”模型，而约束工程通过系统机制“强制”模型执行正确路径，包括错误重试、输出校验等。
