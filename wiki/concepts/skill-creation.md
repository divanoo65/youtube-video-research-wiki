# 自主创建技能

## 概念定义

自主创建技能是 Hermes Agent 核心闭环学习循环中的重要组成部分，体现了智能体在长期运行过程中的自成长能力。当智能体执行复杂任务并成功完成后，系统不会简单地丢弃任务过程，而是会对整个方案进行复用价值评估。若判断该解决方案属于高频需求或具有通用性，智能体会自动将其中经过验证的方法论、步骤、参数配置和最佳实践提炼出来，生成结构化的 Markdown 技能文档。这些文档不仅记录了操作流程，还包含了上下文约束、可调参数和失败案例，使得技能可以在后续类似场景中被精准调用或微调。自主创建技能避免了人工手动编写技能库的繁琐，让智能体能够从每次交互中自动积累经验，不断扩充其能力边界。该机制与人类通过实践总结经验的学习方式高度相似，是智能体实现持续进化的基础保障。

## 技术细节

自主创建技能的实现依赖于以下几项关键技术：

- **技能模板生成引擎**：系统内置一套 Markdown 技能文档模板，包含元数据（技能名称、描述、输入/输出参数、适用条件、版本号）、步骤列表、代码示例、错误处理策略和性能指标。自主创建技能时，引擎根据当前任务的关键要素填充模板。
- **复用价值评估模块**：该模块基于三个维度进行评分：（1）任务执行频次（同一任务类型在近期会话中出现的次数）；（2）解决方案的通用性（是否依赖特定环境或用户偏好）；（3）用户明确反馈（如点赞、纠正或要求保存）。只有当综合评分超过阈值时，才会触发技能创建。
- **技能存储与索引**：生成的 Markdown 文档被保存到本地的技能记忆库中，同时建立全文搜索（FTS）索引，以便在后续会话中快速检索。每个技能文档都附带语义向量表示，支持基于相似度的召回。
- **技能版本管理与回溯**：当同一技能的实践经验被更新时，系统不会直接覆盖，而是保留历史版本，并在文档头部标注变更日志。这允许用户或智能体在必要时回滚到旧版本。
- **与自改进机制的联动**：自主创建技能并非一次性完成。若用户在后续使用中纠正了技能文档中的错误（例如指出某个参数推荐值需要调整），[[self-improvement]]机制会定位到对应的 Markdown 文件并直接修改，实现技能的持续优化。

## 应用场景

自主创建技能在多种实际场景中发挥关键作用：

- **代码开发辅助**：当用户要求智能体编写某种特定模式的代码（如 React 组件、API 调用封装）时，若该模式在项目中反复出现，智能体会自动创建一份“代码生成技能”文档，记录参数、依赖和注意事项。后续同类请求可直接调用该技能，提高响应速度和一致性。
- **数据分析流水线**：在数据处理任务中，如果用户经常要求对某个格式的 CSV 文件进行清洗和可视化，智能体可自主创建“数据预处理技能”，将清洗规则、可视化模板和异常处理逻辑固化，避免重复工作。
- **系统运维与监控**：对于服务器异常排查场景，智能体通过多次成功解决特定故障类型后，会生成“故障诊断技能”，包含检查清单、命令序列和常见原因分析，帮助快速复现解决方案。
- **知识管理与企业培训**：将专业领域（如法律合规、金融分析）中的反复性问答转化为技能文档，新员工或新会话可直接参考，降低学习成本。
