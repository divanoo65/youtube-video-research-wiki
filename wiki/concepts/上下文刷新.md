# 上下文刷新（Context Refresh）

## 定义
上下文刷新（Context Refresh）是 Anthropic 提出的应对长上下文导致模型性能下降的方法。当上下文过长导致模型丢失细节、产生收尾焦虑或注意力涣散时，不采用压缩或摘要方式，而是更换一个干净的 Agent 实例来承接当前任务状态，类似于操作系统的进程重启。

## 在本库的具体例子

- [[Harness-Engineering-从概率生成走向稳定交付]] 视频中，Context Refresh 被列为 Anthropic 的两大实践之一，用于解决上下文工程中“信息过载”问题。
- 具体做法：当 Agent A 执行一段时间后上下文窗口接近极限，系统创建一个新的 Agent B（相同模型），将必要状态（如任务目标、关键中间结果）以结构化方式传递给 B，然后 B 继续执行，A 被回收。

## 技术实现细节

1. **状态传递机制**：不是简单复制整个上下文，而是提取关键信息（当前任务进度、已决策、未完成的子目标）形成精简状态摘要，传递给新 Agent。
2. **与环境上下文解耦**：新 Agent 的上下文仅包含状态摘要+当前指令，不包含历史对话的冗余信息，避免上下文污染。
3. **与“Context Compress”区别**：常见做法是用模型压缩上下文（如摘要），但压缩本身会丢失细节。Context Refresh 通过更换 Agent 实例避免了压缩损失，但需要额外的状态序列化与反序列化成本。
4. **适用于长链路任务**：在需要多步骤、长时间执行的 Agent 任务中，周期性地进行 Context Refresh 可保持高成功率。

## 与近似概念的边界

- **与上下文压缩**：上下文压缩是对现有上下文进行摘要或选择性保留，仍然在同一个 Agent 的上下文中。Context Refresh 则切换到新 Agent，丢弃历史冗余，只保留关键状态。
- **与上下文窗口管理**：上下文窗口管理通常指通过滑动窗口或优先级队列丢弃部分内容。Context Refresh 是更激进的重启策略，适用于上下文严重退化的情况。
