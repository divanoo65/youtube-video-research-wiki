# 任务图 (Task Graph)

### 概念定义
任务图（Task Graph）是现代智能体协作架构中的核心编排逻辑，它将复杂的业务目标抽象为由多个节点（Node）和连接关系（Edge）组成的有向图结构。在这一模型中，每个节点代表一个具备特定能力（如编码、审查、测试、搜索）的智能体 Worker，而边则定义了节点间的交互规则、数据流向与触发条件。通过任务图，主控服务器能够将单一的宏观指令转化为一系列可执行的微观步骤，从而实现从“固定容器”向“动态 Agent 蜂群”的范式转变。这种架构不仅增强了系统的可预测性，还通过明确的逻辑流转，解决了单一智能体在处理长链路、高复杂度任务时的上下文丢失与逻辑混乱问题。

### 技术细节
任务图的实现通常依赖于以下关键技术组件：

*   **节点定义与封装：** 每个节点通常被封装在轻量级容器（如 Docker 或 [[Cube Sandbox]]）中，具备独立的运行环境、模型配置（如 Claude Code 或 Codex）以及特定的工具集。
*   **编排逻辑：** 主控服务器（[[编排者]]）负责解析任务图，根据当前状态动态调度 Worker。例如，在“编码 -> 审查 -> 测试”的流水线中，任务图确保了前置任务的输出作为后置任务的输入，并处理异常回滚逻辑。
*   **动态调度：** 区别于静态脚本，任务图支持根据任务负载动态扩缩容。当任务复杂度增加时，系统可并行启动多个 Worker 节点进行并发搜索或写作，最后由汇总节点进行结果聚合。
*   **状态管理：** 任务图通常结合 [[无状态化]] 设计，通过外部存储（如 [[Redis]]）维护全局状态，确保在节点销毁与重建过程中，任务上下文能够无缝衔接。

### 应用场景
1.  **复杂软件工程自动化：** 在代码开发流水线中，任务图可编排多个智能体分别负责需求分析、代码编写、单元测试与安全审计，实现端到端的自动化交付。
2.  **大规模数据分析与报告生成：** 针对需要多源数据检索的任务，任务图可调度多个 Worker 并行执行搜索，随后通过汇总节点进行逻辑推理与报告撰写，显著提升处理效率。
3.  **企业级智能体基础设施：** 在类似 [[Palantir AIP]] 的企业级平台中，任务图作为底层调度协议，支撑着跨部门、跨系统的智能体协作，将智能体从个人工具升级为企业级生产力基础设施。
