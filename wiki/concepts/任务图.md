# 任务图 (Task Graph)

### 概念定义
任务图（Task Graph）是一种在多 Agent 协作系统中用于描述、规划和执行复杂工作流的逻辑结构。它将宏观目标拆解为一系列具有依赖关系的原子任务节点，并通过有向图（Directed Graph）的形式定义任务间的执行顺序、并发逻辑及数据流转路径。在现代 Agent 编排架构中，任务图不仅是静态的流程图，更是动态的调度指令集。它允许编排者（Coordinator）将复杂的业务逻辑解耦为可管理的单元，从而实现从“单点智能”向“群体协作”的范式演进。通过任务图，系统能够精确控制 Agent 的协作边界，确保在处理如“编码-评审-测试”等复杂流水线时，各环节能够高效衔接，避免资源浪费与逻辑冲突。

### 技术细节
*   **结构化编排**：任务图通常采用有向无环图（DAG）或更复杂的动态图结构，定义节点（任务）与边（依赖关系）。
*   **并发流水线**：支持并行执行机制，例如在“编码”任务完成后，系统可同时触发“静态代码分析”与“单元测试生成”两个分支，显著缩短整体交付周期。
*   **状态流转**：每个任务节点包含输入上下文、执行逻辑（Skill）及输出结果，通过状态机管理任务的生命周期（Pending, Running, Completed, Failed）。
*   **动态调整**：高级的任务图实现支持运行时动态修改，即根据前序任务的输出结果，动态调整后续的执行路径或重试策略。

### 应用场景
*   **软件开发流水线**：在自动化编程场景中，任务图定义了从需求分析、代码生成、代码审查到自动化测试的完整闭环，确保 Agent 能够按序执行。
*   **复杂决策支持**：在需要多维度数据分析的场景中，任务图可调度不同的 Agent 分别负责数据清洗、模型推理和报告撰写，最后由编排者汇总输出。
*   **Agent 蜂群协作**：在 Agent 操作系统中，任务图作为调度算法的输入，指导计算资源如何分配给不同功能的 Agent 实例。
