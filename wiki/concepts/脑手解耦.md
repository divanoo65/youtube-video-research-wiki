# 脑手解耦 (Brain-Hand Decoupling)

## 概念定义
“脑手解耦”是现代 AI Agent 架构设计中的核心范式，其核心思想是将大模型的逻辑推理、规划决策能力（即“大脑”）与具体的代码执行、环境交互操作（即“双手”）进行物理与逻辑上的彻底分离。在传统的 Agent 开发中，模型往往与执行环境深度绑定，导致环境污染、状态漂移及维护困难。通过脑手解耦，系统将执行端抽象为无状态的独立沙盒，使得 Agent 的决策过程不再依赖于特定执行环境的长期存续。这种设计不仅极大地提升了系统的鲁棒性，还使得 Agent 能够像“牛马”一样被随时销毁、重建或横向扩展，从而彻底摆脱了传统“宠物式”运维带来的脆弱性。

## 技术细节
脑手解耦的实现依赖于以下关键技术架构：

1.  **无状态执行沙盒**：底层执行环境（如 [[Cube Sandbox]]）被设计为瞬态的。当任务开始时，系统拉起一个干净的沙盒；当任务完成或出现配置冲突时，系统直接销毁该沙盒，而非尝试修复。这消除了环境污染带来的不确定性。
2.  **状态分离机制**：通过将 Agent 的会话状态（Session）、长期记忆（Memory Store）与上下文（Context）从执行环境中剥离，存储在外部的持久化层中。执行端仅负责接收指令并返回结果，不保留任何业务逻辑状态。
3.  **编排层调度**：引入 [[协调者]]（Coordinator）角色，负责将复杂的长程任务拆解为原子化的子任务，并根据任务需求动态分配计算资源。这种架构允许在不同任务间复用执行环境，极大提升了资源利用率。
4.  **通信协议标准化**：脑与手之间通过标准化的 API 或 RPC 协议进行通信，确保决策层（LLM）与执行层（Sandbox）的解耦，使得更换底层模型或升级执行环境互不影响。

## 应用场景
*   **复杂长程任务自动化**：在需要多步骤、跨环境操作的任务中，利用脑手解耦确保每一步操作都在纯净环境中执行，避免因前序步骤导致的环境污染。
*   **企业级多 Agent 协作**：在 [[Multi-Agent 协作架构深度解析：Anthropic Managed Agents 与蜂群模式实践简报]] 场景下，通过解耦使得多个 Agent 可以共享底层的执行资源池，实现高效的并发处理。
*   **高可靠性系统构建**：对于需要 24/7 运行的 AI 服务，通过“宠物与牛马模型”的转换，利用无状态化特性实现系统的自我修复与快速恢复。
