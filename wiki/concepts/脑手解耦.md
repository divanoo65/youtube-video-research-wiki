## 概述

**脑手解耦**（Brain-Hand Decoupling）是[[多Agent协作架构]]中的核心设计原则，指将模型的**规划决策能力**（大脑）与**代码执行环境**（双手）彻底分离。这一原则由 Anthropic 在 2025 年 4 月的博客中正式提出，并作为其 [[Managed Agents]] 架构的基础。

## 核心机制

### 1. 从“宠物”到“牛马”的范式转变
传统 Agent 开发将模型逻辑、上下文记忆与本地环境深度绑定，导致系统脆弱——环境配置出错或记忆冗余时容易崩溃。脑手解耦通过以下方式解决：
- **无状态沙盒**：每项执行任务在独立的沙盒（如 Docker 或高性能 [[Cube Sandbox]]）中运行。
- **故障隔离**：任务出错或环境污染时，直接销毁沙盒并重新拉起干净环境，无需修复或调试。

### 2. 四层解耦架构
参考报告中描述的架构包含四个层级：

| 层级 | 职责 | 实现方式 |
|------|------|----------|
| **沙盒环境** | 代码执行 | 毫秒级启动的独立容器 |
| **Session管理** | 动态执行状态 | 承载对话线程、记忆挂载 |
| **动态编排器** | 任务拆解与派发 | Coordinator 模式 |
| **云端上下文存储** | 记忆持久化 | Redis 缓存实现 Session Store |

### 3. Session 与 Agent 的实体分离
- **Agent（静态模板）**：定义身份、工具、模型及系统提示词，不存储动态数据。
- **Session（动态执行）**：承载对话线程、记忆挂载和文件操作。同一 Agent 模板可同时启动多个独立 Session。

## 技术实现细节

### 编排者（Coordinator）模式
在 Anthropic SDK 中通过 `coordinator` 字段实现：
- 编排者负责拆解复杂任务、派发指令给下属 Agent。
- 一个编排者可管理多达 20 个拥有独立 Prompt 和 [[Model Context Protocol]] 服务器的 Agent。
- 解决单一 Agent 在长程任务中上下文不足、记忆压缩导致“变笨”的问题。

### 上下文解耦（Session Store）
- 将 Context 从本地搬到云端（如 Redis 缓存），实现 Agent 彻底无状态化。
- Session 可作为“记忆树”进行 Fork（分支）、回滚、克隆或恢复。
- 编排者可以时刻掌握 Agent 最聪明的状态，并在需要时回滚或复制该状态。

## 实践意义

### 多模型组合下的 Token 效率
通过任务图（Task Graph）预定义规则，实现大小模型的异构组合：
- 多个节点并发执行搜索、编码或写作任务。
- 根据任务阶段复杂程度灵活调用不同性能的模型，优化 Token 使用效率。

### 计算资源变革
Agent 架构向“毫秒级启动沙盒”演进，云端计算资源分配模式发生变化：
- 从固定服务转向动态容器。
- 未来可能是无数个根据需求瞬间创建并销毁的 Agent 容器组成的“蜂群”。
