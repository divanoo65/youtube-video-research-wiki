# 内网穿透 (NAT Traversal)

### 概念定义
内网穿透（NAT Traversal）是一种网络通信技术，主要用于解决处于私有局域网（LAN）内的设备无法被公网直接访问的问题。由于大多数家庭或企业网络通过 NAT（网络地址转换）接入互联网，外部请求无法直接寻址到内部设备的特定端口。内网穿透通过在公网服务器与本地设备之间建立一条隧道，将公网的请求转发至本地指定的端口（如 8080），从而使得部署在本地的 Agent 服务、Web 应用或 API 接口能够像部署在云端一样，随时随地被远程访问。对于 Hermes Agent 等本地化部署的智能体而言，该技术是实现移动端交互与跨设备协作的关键基础设施。

### 技术细节
1. **端口映射机制：** 内网穿透工具（如 FRP, Ngrok, Cloudflare Tunnel 等）通过在本地运行客户端，与公网中转服务器建立长连接。当公网请求到达中转服务器时，流量会被封装并通过隧道转发至本地的指定端口（例如将本地的 8080 端口映射至公网域名）。
2. **安全性考量：** 由于内网服务直接暴露于公网，必须配合身份验证（如 Token 校验、HTTPS 加密）以防止未经授权的访问。
3. **性能影响：** 穿透性能受限于中转服务器的带宽与延迟。在 Hermes Agent 的应用场景中，若需进行高频的 API 交互，建议选择线路质量较好的中转节点，以减少请求响应的延迟。

### 应用场景
* **移动端交互：** 当用户在室外需要通过手机访问部署在本地电脑上的 Hermes Agent 时，利用内网穿透将本地服务映射至公网，即可实现远程提问与技能调用。
* **远程调试与演示：** 开发者可将本地正在开发的 Agent 服务临时暴露给外部测试人员，无需将代码部署至云端服务器即可完成功能验证。
* **API 服务化：** 将本地运行的 Agent 封装为 API 服务，通过内网穿透提供给外部应用调用，实现本地计算资源的灵活复用。
