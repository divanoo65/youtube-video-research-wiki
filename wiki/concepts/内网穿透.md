# 内网穿透

**定义**
内网穿透（Intranet Penetration）是一种网络技术，通过建立隧道连接，使位于内网（如家庭或企业局域网）中的服务能够通过公网 IP 或域名从外部网络访问。在 AI Agent 部署场景中，它解决了本地运行的 Agent 服务无法在户外通过手机等移动设备直接访问的问题。

**在本库的具体例子**
- 在 [[Hermes-Agent-高级玩法与部署优化简报]] 中，内网穿透被推荐为实现移动端远程访问的关键技术。用户部署 [[Hermes-Agent]] 并通过 [[Open-WebUI]] 提供交互界面后，使用内网穿透工具（如 frp）将 Open WebUI 的 8080 端口暴露到公网，即可在户外通过手机浏览器访问家中的 Agent。
- 在 [[Hermes-Agent]] 的典型应用场景中，内网穿透使个人本地 AI 助手从「仅限居家使用」扩展为「随时随地访问」。

**技术实现细节**
1. **常用工具**：frp（Fast Reverse Proxy）是最常用的内网穿透工具，由服务端（公网服务器）和客户端（内网机器）组成。其他方案包括 Ngrok、Cloudflare Tunnel、ZeroTier 等。
2. **典型配置**：在内网机器上运行 frp 客户端，将本地 Open WebUI 的 8080 端口映射到公网服务器的指定端口。用户通过 `http://公网IP:映射端口` 访问。
3. **安全措施**：必须设置 API 密码认证（如 [[Hermes-Agent]] 配置文件中的自定义密码），推荐结合 HTTPS 和 IP 白名单，防止未经授权的访问。
4. **Heremss Agent 实践**：先确保 Agent 配置文件中启用了 API 服务（端口 8642），Open WebUI 运行在 8080 端口，然后通过 frp 将 8080 端口暴露到公网。

**与近似概念的边界**
- **区别于 VPN**：VPN 是将远程设备接入内网，能访问所有内网资源；内网穿透是精准暴露特定端口，仅映射指定服务，暴露面更小。
- **区别于端口转发**：端口转发通常需要公网 IP 且需在路由器上配置；内网穿透不需要公网 IP，仅需一台公网服务器即可实现。
