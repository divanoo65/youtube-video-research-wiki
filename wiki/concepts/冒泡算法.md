冒泡算法（Bubble Sort）是一种简单直观的排序算法。它重复地遍历待排序的列表，一次比较两个相邻的元素，如果它们的顺序不正确（例如，前一个比后一个大），就交换它们的位置。这个过程会重复进行，直到没有更多的元素需要交换，这意味着列表已经排序完成。由于较小的元素会像气泡一样逐渐“浮”到列表的顶端（或一端），因此得名冒泡排序。尽管冒泡算法在概念上非常容易理解和实现，但其时间复杂度在最坏情况下和平均情况下都是 O(n^2)，这使得它在大规模数据集上的效率非常低下，通常不适用于生产环境中的高性能排序需求。然而，它在教学和理解排序算法基本原理方面具有重要价值。在 [[Hermes Agent 高级玩法与优化全指南：云端模型、交互增强与 Token 节省策略]] 中，冒泡算法被提及作为展示其“在界面内直接运行 Python 代码”能力的一个具体示例，突出了该 Agent 的 [[功能扩展]] 性。

### 技术细节
冒泡算法的核心在于相邻元素的比较与交换。其基本步骤如下：
1.  **比较相邻元素：** 从列表的第一个元素开始，比较它与下一个相邻的元素。
2.  **条件交换：** 如果它们的顺序不正确（例如，升序排列时，前一个元素大于后一个元素），则交换它们的位置。
3.  **遍历一轮：** 对每一对相邻元素重复上述步骤，从列表的开头到结尾。完成一轮遍历后，最大的（或最小的）元素会“冒泡”到列表的末尾。
4.  **缩小范围：** 针对除了已排序的最后一个元素之外的所有元素重复步骤1-3。每次遍历都减少一个比较范围，因为末尾的元素已经确定是最大（或最小）的。
5.  **循环终止：** 重复进行上述过程，直到整个列表排序完成，即在一轮遍历中没有发生任何交换。

**性能特点：**
*   **时间复杂度：**
    *   最好情况（列表已排序）：O(n)
    *   最坏情况（列表逆序）：O(n^2)
    *   平均情况：O(n^2)
*   **空间复杂度：** O(1)（原地排序，不需要额外的存储空间）
*   **稳定性：** 冒泡排序是一种稳定的排序算法，即相等元素的相对顺序在排序后不会改变。

### 应用场景
尽管冒泡算法的效率相对较低，但在以下特定场景中仍具有其价值：
*   **教学和学习：** 作为最简单的排序算法之一，它非常适合初学者理解排序算法的基本原理和实现逻辑。
*   **小规模数据排序：** 对于数据量非常小（例如几十个元素）的列表，冒泡算法的性能开销可以忽略不计，且实现简单。
*   **功能演示：** 如在 [[Hermes Agent 高级玩法与优化全指南：云端模型、交互增强与 Token 节省策略]] 中，它被用作展示 Agent 运行代码能力的示例，强调了其 [[上传文件]] 等多样的功能扩展性。
*   **作为其他算法的子模块：** 在某些特定场景下，冒泡排序的变种或部分逻辑可能被集成到更复杂的算法中。
