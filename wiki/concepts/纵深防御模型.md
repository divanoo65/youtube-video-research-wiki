# 纵深防御模型

## 定义

纵深防御模型是 Hermes Agent 为安全运行而设计的五层防护体系，包括用户授权、危险命令审批、容器隔离、MCP 凭证过滤及上下文扫描。这套机制使 Hermes Agent 能够安全地部署在暴露的互联网环境中，而无需牺牲功能完整性。

## 在本库的具体例子

在 [[Hermes-Agent-与-OpenClaw-深度对比简报]] 中，Hermes 的五层纵深防御模型被描述为包含：
1. **用户授权**：操作必须经过用户确认
2. **危险命令审批**：高风险 Shell 命令需要二次确认
3. **容器隔离**：关键操作在隔离容器中执行
4. **MCP 凭证过滤**：防止 Agent 误用敏感凭证
5. **上下文扫描**：检测输入输出中的安全隐患

## 技术实现细节

- 每层防御有独立的触发条件和降级处理逻辑
- 用户可配置各层防御的严格程度
- 危险命令审批层会维护一个已知安全命令的白名单
- 容器隔离层使用 Docker 或类似技术创建沙箱环境

## 与近似概念的边界

- **纵深防御 ≠ 简单权限控制**：单一权限控制只检查"能不能做"，纵深防御在多个维度上检查"该不该做"。
- **纵深防御 ≠ 网络安全防护**：网络安全防护关注外部威胁，纵深防御关注 Agent 自身的操作安全。
