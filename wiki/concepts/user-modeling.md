# 用户建模

## 概念定义

用户建模是 [[hermes-agent-self-evolving-ai-agent]] 中核心的智能行为之一，指系统通过对用户历史行为、显式反馈以及隐性信号的持续分析，构建并维护一个动态更新的用户偏好画像。它超越了简单的用户输入记录，深入挖掘用户的潜在偏好——例如对简洁代码风格的偏爱、对特定编程库（如 PyTorch 而非 TensorFlow）的习惯性选择，甚至能够从用户的消极表达（如“这个实现太冗长”、“我不喜欢这种风格”）中提取出有价值的建模信号。用户建模与 [[learning-cycle]] 紧密耦合：每一次交互产生的反馈都被用于修正或丰富用户模型，使得智能体对用户的理解随时间演进，最终实现高度个性化的服务。该模型不仅包含短期会话中的临时推断，更以 [[persistent-memory]] 的形式跨会话保存，确保长期一致性。用户建模是智能体从“通用助手”向“个人专属代理”跃迁的关键能力，也是其自进化架构中不可或缺的组成部分。

## 技术细节

用户建模的实现依赖 Hermes Agent 的三层记忆架构：

- **输入信号处理**：系统从多个维度搜集信号，包括显式指令、代码修改请求、对历史输出的评价（点赞/踩/直接纠正）、操作模式（如频繁调用某工具）、以及对话中流露的偏好短语。即使否定式表达（“不要使用Matplotlib”）也被编码为负向权重，更新用户模型中的对应维度。
- **特征提取与向量化**：利用嵌入模型将用户偏好转化为低维特征向量，存入持久记忆数据库。同时，[[fts-retrieval]] 机制通过全文索引对用户历史行为进行按需加载，避免窗口溢出，并使得模型能基于当前上下文动态召回相关历史。
- **模型更新策略**：采用增量学习方式，每次交互后通过轻量级权重调整更新用户画像。当新反馈与旧模型冲突时（如用户改变偏好），系统会启动冲突检测与优先级覆盖机制，确保模型能够适应行为漂移，而非僵化。
- **多模态用户画像**：用户建模不仅存储偏好标签，还维护一个“技能偏好图”，例如用户喜欢哪种代码风格、常用哪些工具、对细节注释的需求程度等。这些画像信息通过 [[skill-creation]] 转化为可复用的技能文档，从而实现经验的沉淀与自动化。

## 应用场景

- **个性化代码生成**：当开发者要求“写一个数据处理函数”时，用户建模会根据之前对该用户喜欢 Pandas 而非 NumPy、偏爱类型注解、习惯使用 docstring 格式的认知，自动生成符合风格的代码，无需重复显式说明。
- **智能推荐与自动化调度**：系统根据用户建模预测用户下一步可能需要的工具或模板，提前预加载相关技能文档，减少响应延迟。例如，用户经常在周报前调用图表生成工具，模型会自动优先缓存该工具。
- **跨会话一致性保持**：用户在上一会话中设置过“项目 A 使用 Django，项目 B 使用 FastAPI”，用户建模将这些偏好与项目上下文绑定，当新对话涉及项目 A 时，自动应用对应框架约束，无需用户重述。
- **行为异常检测与主动纠偏**：若模型检测到用户当前行为（如突然要求使用不安全的命令）与历史画像严重偏离，可主动触发确认机制，避免误操作或安全风险。

用户建模不仅是静态缓存，更是动态演进的“第二大脑”，使智能体能够像助理一样理解用户深层需求，是 [[hermes-agent-self-evolving-ai-agent]] 区别于传统助手的重要特征。
