# Harness 实践：让 Agent 全自动制作知识讲解视频简报

## 执行摘要

本简报旨在分析“code秘密花园”分享的 AI Agent 视频制作实战流程。该方案摒弃了传统的视频模型或 NotebookLM，转而采用 **Web Coding（网页驱动）** 的方式生成视频。其核心逻辑是通过一套名为 `web-video-presentation` 的 **Skill（技能协议）** 来约束 AI Agent（如 Cline/Cloud Code），将原始技术文章自动化地转化为具备精美视觉、口播脚本、同步音频和动态演示效果的视频素材。

该实践不仅展示了如何制作视频，更深入探讨了 **Harness（驾驭工程）** 的理念：即将模型能力、工具能力与人工判断组织成一条稳定、可控且可重复的生产流程。

---

## 核心主题深入分析

### 1. 网页驱动 (Web-based) 的技术选型逻辑
视频制作未采用视频生成模型，而是通过编写网页代码实现。其核心优势在于：
*   **高度可控：** 字体、配色、每帧的停留时间以及精确到数值的显示，均可通过修改代码完成，规避了视频模型的“抽卡”随机性。
*   **成本极低：** 相比视频模型，网页渲染的计算成本几乎可以忽略。
*   **动态演示：** 弥补了 NotebookLM 只能产出静态图、无法进行动态动画演示的缺陷。

### 2. Harness 驾驭工程的六大核心维度
为了保证 Agent 的产出稳定性，该 Skill 遵循了成熟 Harness 的设计框架：
*   **执行与编排：** 将流程切分为四个阶段，并设置人工检查点（Checkpoint）。
*   **上下文管理：** 按需加载文档。口播规范、开发指引等仅在对应阶段读取，避免模型注意力被稀释。
*   **状态与记忆：** 利用 `outline.md`（开发计划）作为持续记忆，确保长达百余个步骤的开发中风格和结构的一致性。
*   **工具系统：** 深度利用 Agent 的文件读写工具，并配合 MiniMax CLI 进行语音合成。
*   **约束与恢复：** 提出“反馈修复的最小切片”原则，纠错时仅修改受影响的代码片段，而非重做整章。
*   **评估与观测：** 建立硬性自检规则，甚至引入独立审查 Agent（Reviewer Agent）进行逐项检查。

### 3. 多 Agent 协同模式
在处理复杂的大型项目（如 100 多个步骤的视频）时，该方案支持两种协作方式：
*   **Subagent（子进程）：** 适合结果导向任务。主 Agent 将不同章节分给多个子 Agent 并行编写，最后回收结果。
*   **Agent Teams（项目小组）：** 适合需要反复反馈的任务（如开发与测试的对抗）。各成员可互相发消息、共享任务列表，配合 `tmux` 可在多个终端面板实时观测进展。

## 全自动化生产流程指南

根据视频实践，可将文章转化为视频的过程概括为以下四个阶段：

### 阶段一：内容编写（Content Creation）
*   **任务：** 原始文章转化为口播稿（短句、口语化、第二人称）和开发计划（细粒度的章节与步骤）。
*   **人工确认点：** 确认脚本风格、选择视觉主题、确定素材准备方式（AI 生成或手动提供）。

### 阶段二：网页开发（Development）
*   **先行验证：** 优先开发并交付第一章节，确保视觉气质和动效节奏符合预期。
*   **并行推进：** 第一章确认后，利用 Subagents 并行开发剩余章节。每章需进行物理隔离（独立文件夹、独立 CSS），确保视觉统一且互不干扰。

### 阶段三：音频合成（Audio Synthesis）
*   **流程：** 提取口播文本清单，调用 MiniMax CLI 批量生成音频文件。
*   **容错：** 记录报错文件并支持断点续传。

### 阶段四：录制与产出（Recording）
网页提供三种录制模式：
1.  **手动模式：** 不播放音频，手动切换步骤，适合人工配音。
2.  **音频模式 (`?audio=1`)：** 自动播放对应步骤的音频，手动切换画面。
3.  **全自动模式 (`?auto=1`)：** 网页根据音频节奏自动播放所有动画。

## 行动洞察

1.  **工程化胜过原生能力：** 不要仅依赖 Agent 的原始智慧，而应通过 Skill 级协议建立“自检-修复-汇报”的闭环机制，解决自评失真问题。
2.  **建立“文档化记忆”：** 在长链路任务中，将关键决策写入文件（如 `outline.md`），让 Agent 在每个阶段强制读取，能有效解决 Context 稀释导致的节奏混乱。
3.  **最小化干预原则：** 在反馈循环中，引导 Agent 定位问题层次（节奏、视觉、内容或代码），实施最小切片修复，以维持系统的整体稳定性。
