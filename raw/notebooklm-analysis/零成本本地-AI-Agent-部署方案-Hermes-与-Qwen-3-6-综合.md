# 零成本本地 AI Agent 部署方案：Hermes 与 Qwen 3.6 综合简报

本报告旨在详细分析并总结基于 Hermes Agent 与 Qwen 3.6 模型构建的本地 AI Agent 系统的部署流程、核心优势及其在实际应用中的表现。

## 1. 执行摘要 (Executive Summary)

当前 AI 领域正朝着本地化、私有化和代理化方向快速演进。通过结合最新开源的 **Qwen 3.6 模型** 与 **Hermes Agent**，用户可以在无须支付任何月费或 Token 费用的前提下，构建一个 24 小时在线、完全受控且高度隐私安全的本地 AI 助手。该系统不仅在代码编写、中文理解和逻辑推理方面表现卓越，还支持通过 Telegram 等第三方工具进行远程调度，实现了“Token 自由”与移动端 AI 控制的结合。

## 2. 核心主题分析

### 2.1 技术栈：稳定性与性能的平衡
为了确保系统在不同硬件配置下的稳定性，该方案选择了 **Llama-cpp** 作为推理后端，而非 VLLM 或 DeepSpeed。
*   **兼容性优先：** Llama-cpp 能有效防止显存溢出（OOM），适合显存容量有限的个人电脑。
*   **跨平台支持：** 利用 Windows 的 Linux 子系统 (WSL) 和 Ubuntu 24.04 环境，实现 Linux 原生性能与 Windows 日常使用的无缝切换。

### 2.2 模型选择：灵活适配硬件
该方案的核心在于 Qwen 系列模型（包括 3.5 和 3.6 版本）的灵活性：
*   **高配方案：** 针对 24GB 显存用户，推荐使用 **Qwen 3.6 27B** 模型，其性能可满足复杂的自动化任务。
*   **中低配方案：** 提供 0.8B、2B、4B、9B 等多种参数规模，适配不同显存需求的电脑，确保人人皆可参与本地部署。

### 2.3 Hermes Agent：从对话到代理
Hermes Agent 的引入将单一的大模型转变为具备执行力的智能代理：
*   **自定义对接：** 支持通过 V1 接口对接本地模型，API Key 可自定义，极大提升了灵活性。
*   **远程操控：** 通过集成 Telegram Bot，用户可以在手机上随时随地给家中的本地模型下达指令。
*   **自动化：** 支持定时任务执行，是实现“24小时在线 AI 助手”的关键。

### 2.4 运行效能
在未经过度优化的环境下，该系统在处理复杂任务（如编写带前端、后台及数据库的 PHP 程序）时，生成速度可达 **39.51 Token/秒**。在录屏占用显卡资源的情况下，依然保持了“秒出”的响应水平，优化后预期可达 50-60 Token/秒。

## 4. 部署流程指南 (Actionable Insights)

### 第一阶段：基础环境搭建
1.  **启用 WSL：** 在管理员模式的 PowerShell 中运行 Linux 子系统安装命令。
2.  **安装 Ubuntu：** 部署 Ubuntu 24.04，并设置用户凭据。
3.  **驱动直通：** 确保 NVIDIA 显卡驱动已正确直通至 WSL 环境，必要时更新至最新版本显卡驱动。
4.  **依赖安装：** 安装 Python、pip 及 CUDA 工具包（约 2G）。

### 第二阶段：模型推理层部署
1.  **编译 Llama-cpp：** 下载并根据本地硬件路径重新编译 Llama-cpp。
2.  **下载模型：** 根据显存大小选择 Qwen 3.6/3.5 模型（如 27B 版本约 17G）。
3.  **启动服务：** 运行模型并开启 8080 端口服务。可选禁用“深度思考模式”以加速 Agent 响应。

### 第三阶段：Hermes Agent 配置与集成
1.  **代理安装：** 部署最新版 Hermes Agent。
2.  **自定义对接：** 将 Base URL 指向本地地址 `http://localhost:8080/v1`。
3.  **Telegram 集成：** 
    *   通过 BotFather 创建机器人并获取 Token。
    *   获取个人 Telegram ID 以确保私密使用。
4.  **自动化设置：** 设置随系统自动启动模式。

### 第四阶段：持久化与自动化
*   **启动脚本：** 创建 Linux 启动脚本，实现开机自动加载模型与 Agent 服务，无需手动干预即可实现 24/7 在线服务。
