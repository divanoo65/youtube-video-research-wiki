# 零成本本地 AI Agent 部署指南：Hermes + Qwen 3.6 深度集成报告

## 执行摘要

本报告详细分析了基于 Hermes Agent 与 Qwen 3.6 模型的本地 AI 助手部署方案。该方案的核心价值在于通过“零成本”和“本地化”实现“Token 自由”，在保障数据隐私安全的同时，提供强大的自动化任务处理、代码编写、中文理解及逻辑推理能力。通过 WSL（Windows Subsystem for Linux）环境集成 Llama-cpp、Qwen 系列模型以及 Hermes Agent，用户可以构建一个 24 小时在线、支持手机远程访问且开机自动运行的个性化 AI 助手。

---

## 核心主题深度分析

### 1. 硬件要求与底层环境构建
部署本地最强 Agent 组合的基础是 Windows 系统下的 Linux 子系统（WSL）。
*   **操作系统环境**：推荐使用 Ubuntu 24.04 版本。
*   **显卡驱动与加速**：方案高度依赖 NVIDIA 显卡（N 卡）。需确保显卡驱动为最新版本，并安装 CUDA 工具包（约 2G 大小）以支持硬件加速。
*   **运行框架选择**：为了防止在显存不足时出现“爆显存”现象，该方案放弃了 vLLM 或 DeepSpeed，转而采用更具稳定性的 **Llama-cpp** 方案。

### 2. 模型矩阵与性能表现
Qwen 3.6 作为核心推理模型，展现了极高的灵活性和响应速度。
*   **模型规格适配**：
    | 模型版本 | 适用场景/显存建议 |
    | :--- | :--- |
    | **Qwen 3.6 27B** | 适合 24G 显存（如 3090/4090），模型大小约 17G。 |
    | **Qwen 3.5 0.8B/2B/4B/9B** | 适合显存较小的设备，通过替换名称即可快速适配。 |
*   **生成速度**：在 24G 显存设备上，Qwen 3.6 27B 模型在未优化状态下可达到约 **40 tokens/s** 的生成速度。若关闭录屏等额外负载并进行优化，速度可提升至 **50-60 tokens/s**。
*   **模式优化**：默认开启的“深度思考模式”虽然增强了逻辑，但会降低响应速度。为更好地对接 Hermes Agent，建议关闭此模式以获得“秒出”体验。

### 3. Hermes Agent 的集成与功能扩展
Hermes Agent 是将纯模型转化为实用助手的关键层。
*   **多端访问**：通过对接 Telegram 等第三方工具，用户可以突破内网限制，在手机端随时随地进行远程对话。
*   **自定义配置**：支持自定义 Base URL 和 API Key（本地模型可随意填写），并能自动识别已安装的本地模型。
*   **自动化能力**：支持创建定时任务，使本地模型能够执行自主的 Agent 逻辑，而非仅仅是简单的问答。

## 关键步骤详解

### 第一阶段：基础环境配置
1.  **安装 WSL**：通过管理员权限的 PowerShell 执行一键安装命令，并安装 Ubuntu 24.04 镜像。
2.  **显卡直通校验**：在 Linux 子系统中使用命令确认 N 卡驱动已成功直通。
3.  **依赖安装**：安装 Python、pip 及重新编译 Llama-cpp。

### 第二阶段：模型部署与服务化
1.  **模型下载**：根据显存大小选择合适的 Qwen 版本。国内用户可使用 ModelScope 镜像加速。
2.  **启动推理服务**：通过 Llama-cpp 开启 API 服务，默认端口为 8080。

### 第三阶段：Agent 与 Telegram 对接
1.  **安装 Hermes**：执行 Hermes Agent 的一键安装脚本。
2.  **配置机器人**：通过 Telegram 的 BotFather 创建机器人并获取 Token。
3.  **权限设置**：配置个人用户 ID 限制，确保 Agent 仅供个人使用。
4.  **持久化运行**：创建启动脚本并配置 Linux 子系统随系统自启，实现 24 小时在线。

## 可执行洞察与建议

*   **性能优化策略**：如果追求更快的响应速度，建议在启动命令中明确关闭“深度思考模式”，这对于 Agent 调用场景下的任务执行尤为重要。
*   **硬件适配建议**：
    *   **高端配置（24G 显存）**：首选 Qwen 3.6 27B，能够平衡处理能力与生成速度。
    *   **中低端配置**：应灵活切换至 Qwen 3.5 的 4B 或 9B 版本，确保 Llama-cpp 运行稳定而不爆显存。
*   **网络环境方案**：在国内部署时，若直接从原地址下载模型速度受限，务必切换至 ModelScope 镜像以保证部署效率。
*   **自动化管理**：务必完成“开机自启脚本”的设置。Agent 的价值在于随时可用，手动运行命令会极大降低工具的使用频率。
