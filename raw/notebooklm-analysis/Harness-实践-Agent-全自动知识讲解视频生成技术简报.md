# Harness 实践：Agent 全自动知识讲解视频生成技术简报

本报告旨在详细解析利用 Agent 全自动制作知识讲解视频的技术实现方案。该方案基于 Harness 架构思想，通过将复杂的视频生产任务封装为特定的 "Skill"（技能），实现从原始技术文章到高质量视频的标准化、自动化产出。

## 一、 执行摘要 (Executive Summary)

当前 AI 视频生成领域虽发展迅速，但在精细化、可控性及成本方面仍存在挑战。本方案避开了直接使用视频生成模型或 NotebookLM 等工具，转而采用 **Web Coding（网页编码）** 的方式生成视频内容。

通过在 **Harness** 框架下设计的 **"Web Video Presentation" Skill**，开发者可以指挥 AI Agent（如 Claude Code）完成口播稿改写、开发大纲制定、网页动态效果编写及语音合成。该流程的核心价值在于通过工程化手段（流程控制、状态管理、自检修复）驾驭模型能力，确保不同主题、不同背景下的视频产出具有高度的稳定性和一致性。

## 二、 核心架构分析：Harness 驱动的生产系统

该方案并非简单的 Prompt 工程，而是一套完整的 **Harness（驾驭）工程系统**。一个成熟的 Agent 技能（Skill）被拆解为以下六个核心部分：

### 1. 核心组件分布
| 组件 | 说明 |
| :--- | :--- |
| **上下文管理** | 将文档拆分为多个子文档（口播规范、开发指引等），仅在对应阶段读取，避免注意力稀释。 |
| **工具系统** | 利用 Agent 自带的文件读写工具，结合外部 CLI（如 MiniMax CLI）进行语音合成。 |
| **执行与编排** | 将流程切分为四个阶段，并设置人工检查点（Checkpoints）。 |
| **状态与记忆** | 通过 `outline.md`（开发计划）保持长链路开发的连续性，记录用户修改偏好。 |
| **评估与观测** | 引入独立的 Reviewer Agent 进行硬性自检，遵循逐项合查规则。 |
| **约束与恢复** | 遵循“最小切片修复”原则，出错时仅修改受影响的代码层级，不重作整个章节。 |

### 2. 技术栈构成
*   **核心 Agent:** Claude Code (亦可兼容 Cursor/Codex)。
*   **底层模型:** MiniMax (推荐 2.7 版本，因其性价比高、速度快且包含多模态套餐)。
*   **配置工具:** CC Switch (用于切换 Agent 底层模型)。
*   **语音合成:** MiniMax CLI (实现口播稿到音频的批量转换)。
*   **环境管理:** TMUX (多终端管理，用于可视化展示 Agent Teams 的协作过程)。

## 三、 视频生产的四个关键阶段

### 第一阶段：内容编写与规划
*   **口播稿改写：** 将书面化的技术文章转变为短句、口语化、第二人称的叙述风格。
*   **开发大纲 (Outline)：** 将稿件拆分为章节和细粒度的视觉步骤，标注每一步对应的画面内容。
*   **人工审核点 1：** 确认稿件风格、大纲节奏、视觉主题及素材准备方式（AI 生成或手动提供）。

### 第二阶段：网页开发与验证
*   **基准章节开发：** 优先完成第一章开发，由人工验证视觉气质、动效节奏和信息密度。
*   **并行开发：** 在基准确定后，利用 Subagents 或 Agent Teams 并行编写后续章节。
*   **物理隔离：** 每章采用独立的文件夹和 CSS 类名，确保视觉风格一致且互不干扰。

### 第三阶段：语音合成 (TTS)
*   从口播稿中提取音频清单。
*   调用 MiniMax CLI 批量生成音频文件，文件名与开发步骤一一对应。

### 第四阶段：录制与产出
*   **手动模式：** 自由切换步骤，手动录制。
*   **音频模式：** 自动播放音频，辅助录制。
*   **全自动模式：** 页面根据音频节奏自动翻页和展示动效，实现“一键录屏”。

## 四、 关键洞察与核心语录

### 1. 关于技术路线的选择
> "为什么还要折腾网页呢？答案就两个字：**可控**。字体配色、每一步停几秒、某一帧要不要出现一个精确的数字，这些东西在网页里改几行代码就可以直接搞定，比用视频模型直接‘抽卡’要稳定得多。"

### 2. 关于 Harness 的本质
> "做 Harness 不是非得从零去做一个 Agent，能用一个 Skill 约束 Agent 把一个垂直的工作做好，也是在做 Harness。其核心价值在于把模型能力、工具能力和人的判断组织成一条稳定可控的生产流程。"

### 3. 关于修复机制的哲学
> "Skill 里面有一条关键的原则叫：**反馈修复的最小切片**。……直接重作整个章节的问题是它会把已经做对的部分也改掉，用户可能反而更不满意。"

## 五、 行动建议与实践指南

1.  **环境搭建：**
    *   安装 `claude-code` 并配置 `cc-switch`。
    *   注册 MiniMax 并获取 API Key，安装 `minimax-cli`。
    *   下载并部署 `Web Video Presentation` 技能包至 Agent 的工作目录。

2.  **执行策略：**
    *   **从小样开始：** 务必严格执行“第一章验收”制度，在基准未跑通前不要开启大规模并行开发。
    *   **善用并行：** 对于 10 章节以上的长视频，建议启动 `agent-teams` 配合 `tmux`，同时运行 3 个左右的 Agent 以平衡效率与 Token 开销。
    *   **严格自检：** 即使不使用多 Agent 团队，也要求当前 Agent 必须对照 Checklist 逐项核查，严禁“目测一遍就放行”。

3.  **模式选择建议：**
    *   追求极致效率时使用“全自动模式” (`out2=1`) 进行录屏。
    *   追求表达感染力时，建议使用“音频模式” (`audio=1`) 配合真人配音或手动节奏控制。
