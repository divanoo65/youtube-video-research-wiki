# LLM-Wiki（小Wiki）：基于大模型的个人知识库构建指南

## 执行摘要

LLM-Wiki（又称“小Wiki”）是由知名人工智能专家安德烈·卡帕西（Andrej Karpathy）提出的一种新型个人知识库构建思想。该方法旨在解决传统知识管理中存在的“收藏即吃灰”和信息孤岛问题。与传统的检索增强生成（RAG）将大模型视为“解释器”不同，LLM-Wiki 提倡将大模型作为“编译器”，将零散的原始信息转化为结构化、高关联且具有长远价值的数字资产。通过三层架构设计（原始数据层、Wiki 产物层、规则层）和闭环工作流，LLM-Wiki 能够实现知识的“复利增长”，使知识库在持续使用中变得越来越智能。

## 核心理念深度分析

### 1. 从“解释器”向“编译器”的范式转移
传统的 RAG 模式被视为**解释器模式**，其特点是临时性和一次性的：
*   **线性叠加：** 知识库被视为静态的数据片，每次查询都是从零开始，无法对历史知识进行有效积累。
*   **缺乏复利：** 所有的学习是线性的，第一篇笔记无法让第二篇笔记变得更聪明。

相比之下，LLM-Wiki 采用**编译器模式**：
*   **知识转化：** 将原始、零散的文本“编译”为结构化、高关联的 Wiki 页面。
*   **指数级进化：** 类似维基百科（Wikipedia），新知识的输入会自动更新旧的关联页面，形成持续迭代生长的知识网络。

### 2. 三层架构设计
LLM-Wiki 采用严谨的三层结构，确保知识的可靠性与可维护性：

| 层面 | 名称 | 核心职能 | 关键策略 |
| :--- | :--- | :--- | :--- |
| **第一层** | **原始数据 (Raw Data)** | 存放论文、文章、剪藏等原始素材。 | **只读不写：** 防止原始事实被篡改，确保来源可追溯。 |
| **第二层** | **Wiki 产物 (Wiki Assets)** | 由大模型自动生成的实体、概念、摘要和双向链接。 | **结构化复用：** 核心知识载体，实现网络化呈现。 |
| **第三层** | **规则 (Schema)** | 规定大模型如何解读、分类和处理内容的“说明书”。 | **专业化对齐：** 将通用模型调教为懂业务的知识管理员。 |

## 工作流程详解

LLM-Wiki 的运行由三个核心步骤驱动：

1.  **摄入 (Ingestion)：**
    *   将新素材（如 PDF 或笔记）放入原始文件夹。
    *   触发大模型进行内容解析。大模型不仅撰写摘要，还会通过检索现有 Wiki 主动挖掘关联。
    *   **效应：** 一篇新素材通常能触发 10 到 15 个相关概念页面的自动更新，避免信息孤岛。

2.  **查询与回填 (Query & Refinement)：**
    *   **低延迟检索：** 不依赖复杂的向量数据库，而是通过简单的 Markdown 索引（Index.md）实现毫秒级全域定位。
    *   **资产增值：** 每次对话不再是一次性的，高质量的回答会被自动存回 Wiki 文件夹，实现知识生产的持续积累。

3.  **健康检查 (Health Check)：**
    *   定期执行自动化脚本（类似代码审计）。
    *   发现并解决内容矛盾、过时信息、孤立页面和缺失引用等问题，维护知识库的有序性。

## 传统 RAG 与 LLM-Wiki 的对比分析

| 维度 | 传统 RAG (解释器) | LLM-Wiki (编译器) |
| :--- | :--- | :--- |
| **存储/记忆** | 静态切片、瞬时记忆。 | 长期的、结构化的知识沉淀。 |
| **关联方式** | 即时检索，缺乏深层逻辑关联。 | 预建双向链接，形成可漫游的知识网。 |
| **积累效应** | 交互独立，无历史优化。 | 随使用不断增值，具有复利增长性。 |
| **可维护性** | 黑盒不可控，难以人工干预。 | 白盒可追溯，支持人工编辑与微调。 |

## 实践路径与工具链建议

### 1. 目录结构配置
建议在本地或云端创建以下标准目录：
*   `/raw/`：下设 `articles`, `clips`, `pdf`, `notes`, `personal` 等子目录。
*   `/wiki/`：下设 `source` (摘要), `concept` (概念), `entity` (实体), `synthesis` (综合分析)。
*   `/output/`：存放查询结果和分析报告。
*   `/scripts/`：存放维护脚本。
*   `claud.md`：存放规则文件。

### 2. 工具选择
*   **大模型后端：** Claude 或 DeepSeek（通过 API 接入）。
*   **编辑器：** **Obsidian**（用于查看和管理生成的双向链接网络）。
*   **开发环境：** VS Code (及 Cloud Code 插件)、Python 解释器。
*   **浏览器插件：** Obsidian Web Clipper（用于网页剪藏）。
*   **功能插件：** 
    *   `Dataview`：用于查询。
    *   `Graph View`：用于可视化概念关系。

### 3. 操作步骤建议
1.  **环境初始化：** 使用大模型（如通过 Prompt）自动创建上述标准文件夹结构。
2.  **规则配置：** 将 `claud.md` 作为契约文件，明确大模型的处理规范。
3.  **素材处埋：** 将文件放入 `raw` 后，运行大模型脚本进行分类、摘要并自动生成 `wiki` 目录下的 Markdown 文档。
4.  **可视化管理：** 使用 Obsidian 打开知识库文件夹，利用其双向链接功能进行知识漫游。
