# Codex Hooks 入门：实现任务自动化与音频通知简报

本简报旨在总结如何利用 Codex Hooks（钩子）功能实现任务执行过程中的音频自动化提醒。通过在任务的关键节点配置脚本，用户可以有效解决长时间运行任务中的授权等待问题，并以此为切入点构建更复杂的自动化工作流。

## 执行摘要

在执行耗时较长的 Codex 任务（如超过一小时的任务）时，程序往往会在中间阶段（如需要删除文件或访问权限时）暂停并等待用户授权。如果用户未实时盯守屏幕，任务将长时间停滞，导致效率低下。通过配置 Codex Hooks，可以在任务开始、需要授权及任务结束等关键生命周期节点自动播放音频，提醒用户及时处理。此方案不仅限于音频提醒，更是通往自动提交、日志检查等高级自动化操作的入口。

---

## 核心主题分析

### 1. Codex Hooks 的生命周期
Codex 任务的运行包含六个关键的生命周期节点（Lifecycle Hooks），通过对这些节点进行配置，可以触发特定的自动化脚本。本方案主要涉及以下四个节点：

| 节点名称 | 触发时机 | 本方案中的应用 |
| :--- | :--- | :--- |
| `session_start` | 打开新窗口时 | - |
| `user_prompt_submit` | 提交任务提示词时 | 播放任务开始音频 |
| `permission_request` | 需要用户授权时 | 播放请求授权音频 |
| `stop` | 任务处理结束时 | 播放任务完成音频 |
| 其他（工具类） | 工具使用过程中 | - |

### 2. 部署架构：项目级与全局级
Codex Hooks 支持两种作用域的配置，提供了极大的灵活性：

*   **项目级（Project-level）：** 在项目的根目录下新建 `.codex` 文件夹。此配置仅对当前项目生效。
*   **全局级/用户级（Global/User-level）：** 在操作系统用户根目录下新建 `.codex` 文件夹。此配置将对该用户下的所有项目生效。

### 3. 安全信任机制
为了确保安全性，Codex 引入了人工确认机制。当用户配置了新的 Hooks 后，系统会拦截这些操作。用户必须在 Codex App 中点击 "Review Hooks" 并手动选择“信任”（Trust）该项目或全局配置，脚本才能正常运行。

## 实施指南

### 1. 文件结构准备
在 `.codex` 文件夹内，需要建立以下目录结构：
*   `audio/`：存放 MP3 音频文件。
*   `scripts/`：存放针对不同操作系统的执行脚本。

### 2. 关键配置文件
*   **`hooks.json`：** 这是核心配置文件，定义了在哪个生命周期阶段运行什么命令。其结构包含 `hooks` 数组，每个 hook 指定 `type`（如 `command`）、`command` 内容及 `timeout`（超时时间）。
*   **`confirm` 脚本：** 用于调用具体播放逻辑的中间脚本。
*   **操作系统专用脚本：**
    *   **macOS:** 使用 `.sh` 脚本。
    *   **Windows:** 使用 `.ps1` 或相关脚本。
    *   **Linux:** 使用对应的 shell 脚本。

### 3. 操作流程
1.  **资源获取：** 从 `dpt.lab.com` 或相关资源链接下载脚本模板和音频文件。
2.  **文件配置：** 将脚本放入 `scripts` 文件夹，音频放入 `audio` 文件夹。
3.  **赋予权限：** 在 macOS 等系统中，需确保脚本具有可执行权限。
4.  **激活 Hooks：** 在 Codex App 中进行人工审查并授权信任。

## 重要引注与背景

> **关于效率损失的描述：**
> “任务运行了10分钟，来到了需要你给一个授权的位置。这时你恰巧没看见，过了两个小时你回来一看，好家伙他只干了10分钟的活，剩下的时间都在那干等。”

*   **背景：** 这突出了自动化通知的紧迫性，特别是在用户离开座位（如喝咖啡或处理其他事务）时。

> **关于自动化入口的洞察：**
> “我们做的音频播放是为了帮助大家快速地理解和入门，但实质上这是自动化的一个入口。脚本可以实现无限多你想要的内容。”

*   **背景：** 指出音频提醒只是基础应用，Hooks 的核心价值在于其扩展性。

> **关于扩展场景的建议：**
> “大家其实可以拓展到扩展通知、日志检查，甚至我们常用的 Git Autocommit 自动提交，这些都是可以做到的。”

## 行动建议与进阶思考

1.  **即刻部署音频提醒：** 按照上述文件结构配置三个核心音频点（开始、授权、结束），以显著减少长时间任务的空闲等待时间。
2.  **权限核查：** 如果脚本无法运行，应首先检查操作系统的文件运行权限（尤其是 Unix 类系统）。
3.  **探索高级自动化：**
    *   **集成通知系统：** 将音频播放替换或增加为发送手机推送通知。
    *   **开发流程自动化：** 在 `stop` 钩子中配置脚本，实现任务完成后自动进行 Git 提交或将结果记录到特定日志文件中。
4.  **全局配置优化：** 对于常用的自动化逻辑，建议部署在用户根目录，避免在每个新项目中重复配置。
