## Entities
- [[entities/codex-chrome|codex-chrome]] — Codex Chrome 是由 OpenAI 推出的浏览器插件，基于 GPT-5.5 模型实现多标签并行后台自主运行，能自动化完成数据抓取、财务报销、社交媒体发布等复杂工作流。
- [[entities/gpt-5-5|gpt-5-5]] — GPT-5.5 是 OpenAI 开发的最新大语言模型，被公认为当前最强模型之一，驱动 Codex Chrome 插件实现多标签并行、复杂工作流自动化等能力。

## Concepts
- [[concepts/多标签并行|多标签并行]] — 多标签并行是 Codex Chrome 插件的核心运行特性，允许 AI 在浏览器多个独立标签页中同时运行不同代理，互不干扰，实现任务并发处理。
- [[concepts/子代理协作|子代理协作]] — 子代理协作指 Codex Chrome 将复杂任务拆解为多个子任务，分派给多个独立 AI 代理在不同标签页中并行执行，最后汇总结果的工作模式。
- [[concepts/DOM渲染抓取|DOM渲染抓取]] — DOM渲染抓取是指 AI 代理通过读取浏览器渲染后的 DOM 树来获取数据，从而绕过服务器端防护手段。
- [[concepts/Chrome DevTools Protocol|Chrome DevTools Protocol]] — Chrome DevTools Protocol (CDP) 是 Chrome 浏览器提供的一套调试协议，允许外部工具连接并操作浏览器页面。
- [[concepts/提示词优化|提示词优化]] — 提示词优化是指 AI 代理能够自动改进用户输入的简单指令，生成更结构化、更有效的 Prompt。
